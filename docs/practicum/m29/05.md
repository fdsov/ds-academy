# M29.5 · Как встроить это в курс

Платформы подключаются не сразу, а по мере роста.

- **С Фазы 1 (основы Python и данных):** проходи микрокурсы Kaggle Learn (Python, Pandas, Intro to ML) и тренируйся на Kaggle Datasets — это закрепляет каждую теоретическую тему практикой на реальных данных.
- **После Фазы 4 (классический ML освоен):** заходи в соревнования. Начни с Titanic и House Prices, потом бери Featured-соревнование, близкое к табличным бизнес-данным.
- **С Фазы 5 (нейросети и NLP):** подключай Hugging Face — `pipeline`, готовые модели, эмбеддинги, затем Spaces для демо и постепенно дообучение моделей.

!!! tip "AI-копилот: где нейросети помогают"

    LLM-ассистент ускоряет работу с обеими платформами, если использовать его как помощника, а не как оракула.

    Где реально помогает: сгенерировать baseline-код под соревнование («напиши скелет RandomForest для Titanic с кросс-валидацией»); объяснить чужой топовый ноутбук построчно, когда непонятно, зачем сделан тот или иной признак; разобрать карточку модели и подсказать, на что смотреть в лицензии; накидать черновик Gradio-демо или `requirements.txt` для Spaces.

    Где врёт. LLM уверенно выдумывает несуществующие модели и датасеты Hugging Face, придумывает названия аргументов API и методов, которых нет в текущей версии библиотеки, ссылается на устаревшие функции. Железное правило: любую модель, датасет и аргумент проверяй в живом Hub (huggingface.co) и в официальной документации перед запуском. Если ассистент назвал модель — открой её страницу. Нет страницы — модели нет.

!!! note "Стек 2026"

    Актуально на июнь 2026: `kaggle` CLI для работы с соревнованиями и датасетами из терминала; `uv` как быстрый менеджер окружений и установки пакетов вместо голого pip и venv; `transformers` и `datasets` текущих версий для моделей и данных; `huggingface_hub` с командой `hf auth login` для аутентификации; `gradio` для демо на Spaces; `sentence-transformers` для эмбеддингов.

    Что устарело: старая команда `huggingface-cli login` заменена на `hf auth login`; ручное создание окружений через `conda` для простых задач избыточно — `uv` быстрее; устаревшие токены без гранулярных прав стоит пересоздать с минимальными scope (read/write раздельно).

!!! warning "Ловушки"

    - **Переобучение на public leaderboard.** Выбор финальной модели по public score — путь к падению на private после закрытия. Доверяй своей локальной валидации.
    - **Слепое копирование чужих решений.** Форк золотого ноутбука без понимания не даёт навыка и проваливается на собеседовании.
    - **Утечка токена Hugging Face.** Токен в коде или в публичном ноутбуке — это доступ к твоему аккаунту и платным ресурсам. Только переменные окружения и `hf auth login`.
    - **Скачивание гигантских моделей без нужды.** Модель на десятки миллиардов параметров не нужна для классификации тональности и не влезет в ноутбук. Бери минимальную достаточную.
    - **Доверие непроверенным датасетам и моделям.** Любой может выложить на Hub что угодно. Проверяй источник, лицензию, карточку и происхождение данных перед использованием в работе.
