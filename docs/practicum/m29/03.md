# M29.3 · Kaggle

### Что это и зачем

Простыми словами: Kaggle — это сайт, где компании и энтузиасты выкладывают наборы данных и задачи, а тысячи людей соревнуются, кто построит самую точную модель. Победитель определяется не мнением жюри, а числом — метрикой на данных, которых участники не видели. Это честно и объективно.

Конкретно: Kaggle принадлежит Google. Внутри — соревнования с денежными призами и без, гигантская библиотека датасетов, бесплатная среда для кода в браузере с доступом к GPU и TPU, бесплатные интерактивные микрокурсы и форумы. Тебе не нужно ничего устанавливать, чтобы начать: всё работает в браузере.

### Устройство платформы

Kaggle состоит из шести связанных частей. Разберём каждую.

**Competitions (соревнования)** — задачи с данными и метрикой. Бывают учебные (Getting Started, например Titanic — живут вечно, для тренировки), Featured (с призами от компаний), Research и Community. Ты загружаешь файл с предсказаниями, система считает метрику и ставит тебя в таблицу.

**Datasets (датасеты)** — десятки тысяч открытых наборов данных на любую тему: от транзакций до медицины. Можно искать, скачивать, форкать и публиковать свои. Для аналитика это кладезь: ищешь по словам «casino», «churn», «iGaming» и находишь реальные размеченные данные для тренировки.

**Notebooks (Code)** — Jupyter-ноутбуки прямо в облаке Kaggle. Простыми словами: это бесплатный компьютер в браузере с предустановленным Python, pandas, scikit-learn, PyTorch. Конкретно: дают примерно 30 часов GPU в неделю и доступ к TPU, до 12 часов на запуск. Ты пишешь код, жмёшь Run, считаешь модель на видеокарте, которой у тебя нет дома, и всё это бесплатно.

**Kaggle Learn** — короткие практические микрокурсы: Python, Pandas, Intro to ML, Feature Engineering, Intro to Deep Learning и другие. Каждый — пара часов, теория сразу закрепляется упражнением в ноутбуке. Это лучшая бесплатная точка входа в практику.

**Models** — хаб предобученных моделей (в том числе зеркала с Hugging Face), которые можно подключить к ноутбуку в один клик.

**Discussions (форумы)** — обсуждения по каждому соревнованию. Здесь топовые участники делятся идеями, разбирают подходы, спорят о валидации. Чтение форумов сильного соревнования заменяет полкурса по прикладному ML.

### Как завести аккаунт и сделать первый submission

Пройдём весь путь от регистрации до первой строки в leaderboard на примере соревнования Titanic — это классический Getting Started, где надо предсказать, кто выжил при крушении.

Шаг 1. Зайди на kaggle.com, зарегистрируйся через Google или email. Зайди в Settings и подтверди номер телефона (Phone Verification) — без этого не дадут GPU и возможность сабмита.

Шаг 2. Открой страницу соревнования (ищи «Titanic» в Competitions), нажми Join Competition и прими правила.

Шаг 3. Нажми New Notebook на странице соревнования — откроется ноутбук, к которому данные соревнования уже подключены в папке `/kaggle/input/`.

Шаг 4. Вставь и запусти baseline-код. Это не серебряная модель, а минимальный честный старт.

```python
import pandas as pd
from sklearn.ensemble import RandomForestClassifier

train = pd.read_csv("/kaggle/input/titanic/train.csv")
test = pd.read_csv("/kaggle/input/titanic/test.csv")

features = ["Pclass", "Sex", "SibSp", "Parch"]
X = pd.get_dummies(train[features])
X_test = pd.get_dummies(test[features])
y = train["Survived"]

model = RandomForestClassifier(n_estimators=200, max_depth=5, random_state=42)
model.fit(X, y)
preds = model.predict(X_test)

submission = pd.DataFrame({"PassengerId": test.PassengerId, "Survived": preds})
submission.to_csv("submission.csv", index=False)
```

Шаг 5. Файл `submission.csv` появится в выводе ноутбука. Нажми Save Version (Save & Run All), дождись выполнения, затем на вкладке Output нажми Submit. Либо на странице соревнования — Submit Predictions и загрузи файл.

Шаг 6. Через несколько секунд увидишь свой score и позицию в public leaderboard. Поздравляю — это твой первый submission.

!!! tip "Делай маленький честный baseline первым"

    Не гонись сразу за сложной моделью. Первый сабмит нужен, чтобы проверить, что весь конвейер работает: данные читаются, формат файла верный, метрика считается. Сильную модель строишь уже поверх рабочего конвейера.

!!! note "Code-соревнования"

    Многие современные Featured-соревнования — это code-соревнования: ты сдаёшь не CSV-файл, а ноутбук, который Kaggle сам запускает на скрытом тесте в офлайне (без доступа в интернет, с лимитом по времени). Titanic и House Prices принимают CSV, но в боевых соревнованиях чаще встретишь именно сдачу ноутбука — учитывай это при выборе.

### Public vs private leaderboard

Простыми словами: пока соревнование идёт, ты видишь свой результат только на части скрытого тестового набора — это public leaderboard. Когда соревнование закрывается, считается результат на другой, ранее невидимой части — private leaderboard. Именно private определяет победителей.

Зачем так сделано. Если бы итог считался по тем же данным, что ты видишь во время соревнования, можно было бы тысячами сабмитов подогнать модель под эти конкретные данные. Это и есть утечка валидации: ты «учишься» на тесте через обратную связь. Модель выглядит отличной на public, но проваливается на private, потому что выучила шум, а не закономерность.

Конкретно это бьёт по тем, кто выбирает финальную модель по public score. Классический сценарий: человек был в топ-10 на public, а после закрытия упал на 800-е место — его модель переобучилась на public LB. Это называется shake-up.

Защита — честная локальная валидация. Ты строишь свою кросс-валидацию на тренировочных данных и доверяешь ей больше, чем public score. Если локальная метрика и public LB расходятся — доверяй локальной, она честнее. Это прямая связь с темой честной валидации и утечек (см. модуль M13 про валидацию и кросс-валидацию и модуль M17 про data leakage). Тот же принцип в твоей работе: если модель оттока показала AUC 0.9 на той же выборке, на которой её настраивали по сетке гиперпараметров, — это не качество модели, это утечка через подбор.

!!! question "Проверь себя"

    1. На какой части скрытого теста считается public leaderboard, а на какой — private, и какой из них определяет победителей?
    2. Почему выбор финальной модели по public score — это форма утечки валидации?
    3. Что такое shake-up и из-за чего человек из топ-10 на public падает на 800-е место?
    4. Если локальная кросс-валидация и public LB расходятся — чему доверять и почему?

??? success "Ответы"

    1. Public — на видимой во время соревнования части; private — на другой, ранее невидимой части, и именно private определяет победителей.
    2. Тысячами сабмитов модель подгоняется под видимые данные через обратную связь — она «учится» на тесте и выучивает шум, а не закономерность.
    3. Shake-up — резкая перестановка после закрытия: модель переобучилась на public LB и провалилась на private.
    4. Локальной валидации — она честнее, потому что не настроена через обратную связь от leaderboard.

### Как учиться по топовым публичным notebooks

В каждом соревновании на вкладке Code люди публикуют свои ноутбуки, отсортированные по голосам. Топовые — это бесплатные мастер-классы по feature engineering и валидации.

Простыми словами: ты смотришь, как сильные участники думают над задачей. Конкретно — правильный способ читать чужой ноутбук: сначала пойми, как устроена валидация (это важнее самой модели), потом — какие признаки они создали и почему, и только потом — выбор модели. Форкни ноутбук (Copy & Edit), запусти у себя, поменяй один кусок, посмотри как изменится результат. Понимание приходит от изменения, а не от чтения.

### Прогрессия рангов и медали

Kaggle геймифицирует прогресс по пяти ступеням в каждой из категорий (Competitions, Notebooks, Datasets, Discussions): **Novice → Contributor → Expert → Master → Grandmaster**.

Простыми словами: чем выше ранг, тем серьёзнее воспринимают твой профиль. Конкретно: ранги растут за медали. Медали (бронза, серебро, золото) дают за попадание в верхние проценты leaderboard, за высоко оценённые ноутбуки и обсуждения. Например, чтобы стать Competitions Expert, нужны две бронзовые медали, Master — золото и серебро, Grandmaster — несколько золотых, включая личное (solo). Это не самоцель, но честный публичный индикатор уровня.

### Kaggle как портфолио

Профиль Kaggle — это резюме, которое нельзя сфальсифицировать. Работодатель видит, в каких соревнованиях ты участвовал, какие ноутбуки публиковал, как объясняешь свои решения. Хорошо оформленный публичный ноутбук с чистым EDA и грамотной валидацией на близкой к работе теме (например, предсказание оттока на открытых данных) стоит больше, чем строчка «знаю ML» в CV. Линкуй профиль в резюме и LinkedIn.

### Kaggle API и CLI

Когда надоест веб-интерфейс, всё то же самое делается из терминала. Это нужно, чтобы качать данные в свой проект, автоматизировать и работать в своей среде.

Установка и настройка. Создай API-токен в Settings → Create New Token, скачается `kaggle.json`. Положи его правильно и закрой права.

```bash
uv pip install kaggle
mkdir -p ~/.kaggle
mv ~/Downloads/kaggle.json ~/.kaggle/kaggle.json
chmod 600 ~/.kaggle/kaggle.json
```

Основные команды.

```bash
kaggle competitions list
kaggle competitions download -c titanic
kaggle competitions submit -c titanic -f submission.csv -m "rf baseline"

kaggle datasets list -s churn
kaggle datasets download -d blastchar/telco-customer-churn
```

`kaggle competitions download` тянет данные соревнования, `kaggle datasets download` — любой датасет по его slug (видно в URL). `kaggle competitions submit` отправляет файл предсказаний прямо из терминала.

Для загрузки прямо в Python-код, помимо CLI, есть библиотека `kagglehub`: вызов `kagglehub.dataset_download("blastchar/telco-customer-churn")` тянет и кэширует данные, отдавая путь к ним, — удобно внутри ноутбука без ручного CLI.

### Этика

Главное правило: не копировать бездумно. Форкнуть чужой золотой ноутбук, сменить random seed и отправить — это самообман, ты ничему не научился и медаль ничего не значит. Бери чужие идеи как материал для понимания, переписывай своими руками, проверяй, почему это работает. На собеседовании тебя спросят не «какой ноутбук ты форкнул», а «почему ты выбрал такую валидацию» — и тут видно, копировал ты или думал.
