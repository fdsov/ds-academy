# M29.6 · Практика

### Быстрый квиз

Выбери вариант — ответ подсветится сразу.

```quiz
Что определяет победителей соревнования на Kaggle?
[ ] public leaderboard — результат на видимой во время соревнования части теста
[x] private leaderboard — результат на ранее невидимой части теста
[ ] суммарный score по всем сабмитам за время соревнования
[ ] голосование топовых участников на форуме
> Public виден во время соревнования, но итог считают на скрытой private-части — именно она определяет победителей.
---
Почему выбор финальной модели по public score — это форма утечки валидации?
[ ] потому что public-набор всегда меньше private
[x] потому что тысячами сабмитов модель подгоняют под видимые данные через обратную связь и она выучивает шум
[ ] потому что Kaggle штрафует за частые сабмиты
[ ] потому что private-набор содержит другие признаки
> Ориентируясь на public score, ты фактически «учишься» на тесте через обратную связь и ловишь шум, а не закономерность — это и есть утечка.
---
Что делает функция `pipeline` из библиотеки transformers?
[ ] дообучает модель на твоих данных за пять строк
[x] прячет токенизацию, загрузку весов и постобработку за одним вызовом и запускает готовую модель
[ ] публикует демо модели в интернете
[ ] конвертирует датасет в формат Hugging Face
> `pipeline` даёт готовый инференс «из коробки»: скачивает модель под задачу и применяет её к тексту без обучения.
---
Зачем читать карточку модели (model card) перед использованием?
[ ] чтобы узнать цену запуска модели на Inference Endpoints
[ ] чтобы скачать веса модели одним кликом
[x] чтобы понять, что модель умеет, на чём обучена, её ограничения и лицензию
[ ] чтобы автоматически выбрать минимальный размер модели
> Карточка — это «паспорт» модели: применимость, ограничения и лицензия, что особенно критично в гемблинг-продукте.
---
Какой моделью разумно решать задачу классификации тональности отзывов?
[ ] обязательно самой крупной моделью на 70B параметров
[x] минимальной моделью, которая решает задачу
[ ] любой моделью без лицензии, лишь бы было много загрузок
[ ] только моделью, обученной лично тобой с нуля
> Крупная модель не влезет в ноутбук и избыточна — берут минимальную достаточную модель под задачу.
---
Где нужно хранить токен доступа Hugging Face?
[ ] прямо строкой `hf_xxxx` в ноутбуке, чтобы не потерять
[ ] в публичном репозитории, чтобы команда имела доступ
[x] в переменной окружения или через `hf auth login`, а в коде читать `os.environ["HF_TOKEN"]`
[ ] в названии файла с предсказаниями
> Токен в коде или в публичном ноутбуке утекает и даёт доступ к аккаунту и платным ресурсам — храни его в окружении или через `hf auth login`.
---
Чем команда `kaggle competitions download` отличается от `kaggle datasets download`?
[x] первая тянет данные соревнования по его slug, вторая — любой датасет по его slug
[ ] первая работает только в браузере, вторая — только в терминале
[ ] первая скачивает модели, вторая — ноутбуки
[ ] между ними нет разницы, это синонимы
> `competitions download -c <slug>` качает данные соревнования, а `datasets download -d <slug>` — произвольный датасет по slug из URL.
```

Задания идут по нарастающей сложности. Не прыгай через ступени.

**Разминка.**

1. Заведи аккаунты на Kaggle и Hugging Face, подтверди телефон на Kaggle.
2. Пройди один микрокурс Kaggle Learn целиком (рекомендую Intro to Machine Learning) — с упражнениями, не пропуская.

**Основное.**

3. Сделай первый submission на Titanic по шагам из лекции и убедись, что видишь себя в public leaderboard.
4. Установи `transformers` через uv и запусти `pipeline("sentiment-analysis")` локально на десяти своих фразах. Сравни, где модель ошибается.

**Челлендж.**

5. Возьми учебное соревнование (Titanic или Spaceship Titanic), собери честный baseline с локальной кросс-валидацией, затем улучши его feature engineering. Зафиксируй: насколько выросла локальная метрика и насколько public LB — расходятся ли они.

**Мини-проект.**

6. Подними Gradio-демо на Hugging Face Spaces: модель тональности или классификатор по своему выбору. Получи публичную ссылку и положи её в заметку для портфолио.
