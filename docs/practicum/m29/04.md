# M29.4 · Hugging Face

### Что это и зачем

Простыми словами: представь огромный склад, где лежат уже обученные модели — для текста, картинок, звука, — а также датасеты и живые демо. Приходишь, берёшь нужное с полки, подключаешь к своему коду. Ничего обучать с нуля не надо.

Конкретно: Hugging Face — это хаб (huggingface.co), где хранятся модели (Model Hub), датасеты (Datasets), интерактивные приложения (Spaces) и инструменты для запуска моделей. Вокруг — набор Python-библиотек, которые делают загрузку и запуск любой модели тривиальными.

### Экосистема библиотек

Четыре основные библиотеки, которые покрывают почти всё.

**transformers** — главная библиотека. Загружает и запускает модели (BERT, GPT-подобные, T5, современные LLM) единым интерфейсом. Хочешь классификацию текста, перевод, генерацию, эмбеддинги — всё здесь.

**datasets** — загрузка и обработка готовых датасетов одной строкой, с потоковой подгрузкой, чтобы не держать всё в памяти.

**huggingface_hub** — клиент для самого хаба: скачивание файлов, аутентификация по токену, публикация своих моделей и датасетов.

**accelerate** — запуск обучения и инференса на любом железе (CPU, один GPU, несколько GPU) без переписывания кода. Нужна, когда дойдёшь до дообучения моделей.

### Предобученная модель за пять строк

Главная магия Hugging Face — функция `pipeline`. Она прячет всю сложность (токенизация, загрузка весов, постобработка) за одним вызовом.

```python
from transformers import pipeline

clf = pipeline("sentiment-analysis")
result = clf("This casino app keeps crashing after the last update")
print(result)
```

Результат будет вроде `[{'label': 'NEGATIVE', 'score': 0.998}]`. Первая строка импортирует функцию, вторая скачивает модель по умолчанию для задачи тональности, третья применяет её к тексту. Так за минуту можно разметить тональность тысяч отзывов или тикетов поддержки.

Второй частый кейс — эмбеддинги, числовые представления текста для поиска похожего и кластеризации игроков по их обращениям.

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("sentence-transformers/all-MiniLM-L6-v2")
vectors = model.encode(["игрок просит вернуть депозит", "запрос на возврат средств"])
print(vectors.shape)
```

Два близких по смыслу текста дадут близкие векторы — это основа семантического поиска и дедупликации обращений.

### Загрузка готового датасета

```python
from datasets import load_dataset

ds = load_dataset("imdb")
print(ds["train"][0])
print(len(ds["train"]))
```

Одна строка скачивает и кэширует датасет, дальше с ним работаешь как со словарём сплитов (`train`, `test`). Для больших наборов добавляй `streaming=True`, чтобы данные подгружались по мере чтения, не забивая диск и память.

### Model Hub и выбор модели

На huggingface.co/models десятки тысяч моделей. Как не утонуть.

Простыми словами: у каждой модели есть «паспорт» — карточка модели. Читай её перед использованием. Конкретно, на что смотреть:

- **Карточка модели (model card)** — что модель умеет, на чём обучена, известные ограничения и примеры кода.
- **Лицензия** — критично для работы. `apache-2.0` и `mit` обычно разрешают коммерческое использование, у некоторых моделей лицензия запрещает коммерцию или требует согласия. В гемблинге это особенно важно: проверяй, можно ли использовать модель в продукте.
- **Размер** — число параметров и вес файлов. Модель на 70B параметров не влезет в твой ноутбук и не нужна для классификации тональности. Бери минимальную модель, которая решает задачу.
- **Популярность и свежесть** — число загрузок, лайков, дата обновления. Тысячи загрузок и недавнее обновление — хороший сигнал, но не замена проверке.
- **Задача (task)** — фильтруй модели по задаче (text-classification, embeddings, translation), чтобы видеть только подходящие.

    3. Нужна ли модель на 70B параметров для классификации тональности — и какой принцип выбора размера?

??? success "Ответы"

    1. Это «паспорт» модели: что умеет, на чём обучена, ограничения, примеры кода — читают, чтобы понять применимость.
    2. Некоторые лицензии запрещают коммерческое использование или требуют согласия; в продукте это юридический риск, нужно проверять `apache-2.0`/`mit` против ограничивающих.
    3. Не нужна — бери минимальную модель, которая решает задачу; крупная не влезет в ноутбук и избыточна.
    4. Популярность и свежесть (загрузки, лайки, дата обновления) и фильтр по задаче — это сигналы, но не гарантия качества, поэтому модель всё равно проверяют.

### Spaces и Gradio

Spaces — это бесплатный хостинг для интерактивных демо твоих моделей. Gradio — библиотека, которая из функции делает веб-интерфейс в несколько строк.

Простыми словами: написал функцию, которая принимает текст и возвращает ответ — Gradio оборачивает её в страницу с полем ввода и кнопкой, а Spaces публикует это в интернете по ссылке. Конкретно:

```python
import gradio as gr
from transformers import pipeline

clf = pipeline("sentiment-analysis")

def predict(text):
    r = clf(text)[0]
    return f"{r['label']} ({r['score']:.2f})"

gr.Interface(fn=predict, inputs="text", outputs="text").launch()
```

Чтобы выложить: создай новый Space (тип Gradio) на сайте, положи этот код в `app.py`, добавь `requirements.txt` с `transformers`, `torch`, `gradio` — и Space сам соберётся и даст публичную ссылку. Это идеальный способ показать работу в портфолио: не «у меня есть модель», а живое демо, которое работодатель потыкает сам.

### Inference Providers и Endpoints

Иногда модель не нужно скачивать — её можно дёрнуть как сервис.

**Inference Providers** (serverless) — простыми словами: ты отправляешь текст по HTTP, получаешь предсказание, модель крутится на стороне провайдера. Раньше это называлось Inference API; теперь это роутер по нескольким провайдерам, собственный провайдер Hugging Face называется `hf-inference`. Удобно для прототипов и небольших нагрузок, есть бесплатный лимит. Конкретно — это пара строк через `huggingface_hub.InferenceClient` с твоим токеном.

**Inference Endpoints** — управляемый продакшн-хостинг: выделенная машина под твою модель с автоскейлом, уже за деньги. Это путь, когда демо переросло в реальный сервис с нагрузкой.

Связь с теорией: чтобы осознанно выбирать модель по карточке и понимать, что значит «энкодер», «декодер», «контекстное окно», нужна база по архитектурам трансформеров и устройству LLM (см. модуль M19 про архитектуры нейросетей и модуль M20 про большие языковые модели). Hugging Face — это руки, теория моделей — голова.

### Приватность и токены доступа

Чтобы качать приватные модели, публиковать своё и пользоваться Inference Providers, нужен токен доступа.

Создай токен в Settings → Access Tokens (с минимально нужными правами: read для скачивания, write только если публикуешь). Авторизуйся.

```bash
uv pip install huggingface_hub
hf auth login
```

Команда `hf auth login` спросит токен и сохранит его в системе, после чего код подхватит его автоматически.

!!! warning "Никогда не пиши токен прямо в коде"

    Не вставляй `hf_xxxx` строкой в ноутбук или скрипт и тем более не коммить в git. Утёкший токен — это доступ к твоему аккаунту и потенциально к платным ресурсам. Храни токен в переменной окружения или через `hf auth login`, а в коде читай `os.environ["HF_TOKEN"]`.

    2. Зачем вообще нужен токен доступа и какие права (scope) ставить для скачивания, а какие — для публикации?
    3. Почему нельзя писать `hf_xxxx` строкой в коде и где токен хранить вместо этого?

??? success "Ответы"

    1. Inference Providers — serverless: шлёшь HTTP, модель крутится у провайдера (свой провайдер HF — `hf-inference`), удобно для прототипов и малых нагрузок (есть бесплатный лимит); Endpoints — выделенный продакшн-хостинг с автоскейлом за деньги, когда демо переросло в сервис.
    2. Токен нужен для приватных моделей, публикации и Inference API; read — для скачивания, write — только если публикуешь.
    3. Утёкший токен даёт доступ к аккаунту и платным ресурсам; храни в переменной окружения или через `hf auth login`, в коде читай `os.environ["HF_TOKEN"]`, в git не коммить.
