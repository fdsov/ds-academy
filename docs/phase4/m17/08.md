# M17.8 · Интерпретируемость: глобальная против локальной

### Интуиция

Когда модель выдала число, есть два принципиально разных вопроса. Первый: "как эта модель устроена вообще, на что она в среднем смотрит по всей популяции игроков?" Второй: "почему именно для вот этого игрока она выдала score 0.92 и пометила его как высокий риск оттока?" Первый вопрос — глобальный, второй — локальный. Их легко спутать, но ответы на них дают разные инструменты и разные люди их заказывают.

Бытовая картинка. Глобальная интерпретируемость — это инструкция к автомобилю: "расход топлива в среднем растёт с массой груза и скоростью". Локальная — это объяснение конкретной поездки: "вчера ты сжёг бак за 300 км, потому что вёз прицеп и ехал в гору". Инструкция не объясняет конкретную поездку (вчерашний расход может быть аномальным), а объяснение одной поездки нельзя обобщать на все ("значит, машина всегда жрёт бак за 300 км" — неверно).

Кто что заказывает в продуктовой аналитике гемблинга:

- Продукт-менеджер хочет глобальную картину: "отток в основном объясняется падением частоты ставок и ростом дней без депозита" — это решает, какую фичу строить.
- Саппорт, объясняющий игроку ограничение по антифроду, хочет локальную: "этот конкретный аккаунт помечен из-за вот этих трёх транзакций с одного IP" — это нужно для конкретного разговора.
- Комплаенс/регулятор хочет локальную: по GDPR и многим гемблинг-лицензиям человек имеет право на объяснение автоматизированного решения, которое его касается. "Модель в среднем смотрит на X" тут не ответ — нужно объяснение именно его кейса.

### Формально

Пусть обучена модель $\hat{f}: \mathcal{X} \to \mathbb{R}$ (например, вероятность оттока).

- **Глобальная интерпретируемость** характеризует поведение $\hat{f}$ на всём распределении данных $p(x)$ — усреднённый вклад признаков. Инструменты: коэффициенты линейных моделей, feature importance деревьев (по уменьшению impurity), permutation importance, глобальный SHAP summary (среднее $|\phi_j|$ по выборке), partial dependence plots (PDP).
- **Локальная интерпретируемость** объясняет одно предсказание $\hat{f}(x_i)$ для конкретного объекта $x_i$. Инструменты: SHAP values для строки, LIME, локальные surrogate-модели, контрфактические объяснения.

SHAP даёт мост между уровнями. Для объекта $x$ значения Шепли $\phi_j(x)$ раскладывают предсказание в аддитивную сумму вкладов признаков:

$$
\hat{f}(x) = \phi_0 + \sum_{j=1}^{M} \phi_j(x),
$$

где $\phi_0 = \mathbb{E}[\hat{f}(X)]$ — базовое (среднее по популяции) предсказание, а $\phi_j(x)$ — вклад признака $j$ именно в это предсказание. Это локальное объяснение. Глобальную важность получают агрегированием по выборке:

$$
I_j = \frac{1}{n}\sum_{i=1}^{n} \bigl|\phi_j(x_i)\bigr|.
$$

Ключевой момент: знак и величина локального $\phi_j$ для отдельного игрока могут расходиться с глобальной важностью. Признак, неважный в среднем, способен быть решающим для конкретного редкого кейса — именно из-за усреднения по модулю в $I_j$ это незаметно на глобальном уровне.

#### Разбор на пальцах: churn-score одного игрока против общей картины

Churn-модель (вероятность оттока за 30 дней). Базовое среднее по популяции $\phi_0 = 0.20$. Берём игрока, которому модель выдала $\hat{f}(x) = 0.78$. SHAP-разложение строки:

| Признак | $\phi_j(x)$ |
|---|---|
| days_since_last_deposit = 21 | +0.34 |
| bet_frequency_7d = 2 (упала с 30) | +0.22 |
| support_tickets_7d = 3 | +0.07 |
| vip_tier = Gold | −0.05 |

Проверим аддитивность: $0.20 + 0.34 + 0.22 + 0.07 - 0.05 = 0.78$. Сходится с предсказанием. Локальный вывод: этого игрока тянет в отток прежде всего долгий перерыв в депозитах и обвал частоты ставок, а Gold-статус слегка удерживает.

Теперь глобальная картина по всей выборке (средние $|\phi_j|$):

| Признак | $I_j$ |
|---|---|
| bet_frequency_7d | 0.18 |
| days_since_last_deposit | 0.15 |
| avg_bet_size | 0.09 |
| support_tickets_7d | 0.02 |

Глобально частота ставок важнее, чем дни без депозита. Но у нашего игрока всё наоборот: главный драйвер — именно days_since_last_deposit (+0.34). Если саппорт будет действовать по глобальной табличке ("в среднем главное — частота ставок") и предложит фрибет на ставки, он промахнётся: игроку нужен депозитный триггер (кэшбэк/бонус на пополнение). Вот зачем нужна локальная интерпретация — глобальная важность здесь даёт неправильный персональный совет.

Эффект в деньгах. Допустим, ежемесячно модель помечает 5000 игроков с высоким риском, средний игрок приносит ARPU $40/мес, а точечная retention-кампания удерживает 8% помеченных вместо 3% при шаблонном подходе. Разница 5 процентных пунктов на 5000 игроков — это $0.05 \times 5000 \times \$40 = \$10\,000$ в месяц удержанной выручки за счёт того, что воздействие подобрано по локальному объяснению, а не по усреднённой картине.

!!! warning "Типичная ошибка"

    Новичок берёт глобальный feature-importance график и выдаёт его за объяснение конкретного предсказания: "модель пометила этого игрока, потому что частота ставок — самый важный признак в модели". Это подмена уровней. Глобальная важность говорит, на что модель смотрит в среднем по популяции, а не почему сработала именно эта строка. Для конкретного игрока драйвер может быть другим (и даже с противоположным знаком). Объяснять одно предсказание нужно локальным методом (SHAP/LIME для строки), а не глобальной важностью.

!!! info "Permutation importance: ещё одна ловушка уровней"

    Permutation importance — глобальный метод: он показывает, насколько падает качество модели на всей выборке, если перемешать значения признака. При сильно скоррелированных признаках (например, bet_frequency и avg_bet_size) он может занижать важность обоих, потому что модель компенсирует один через другой. Это искажает глобальную картину, но никак не относится к объяснению отдельного кейса — не путайте эти задачи.

!!! question "Проверь себя"

    1. Сформулируй разницу между глобальной и локальной интерпретируемостью одним предложением каждая.
    2. Что в SHAP-разложении означает базовое значение $\phi_0$ и почему сумма всех $\phi_j(x)$ плюс $\phi_0$ равна предсказанию?
    3. Почему глобальный feature importance нельзя использовать как объяснение конкретного предсказания?
    4. Саппорту нужно объяснить игроку, почему его аккаунт ограничен антифрод-моделью. Какой тип интерпретируемости ему нужен и какой инструмент подойдёт?

??? success "Ответы"

    1. Глобальная: как модель ведёт себя в среднем по всему распределению данных (какие признаки важны в целом). Локальная: почему модель выдала конкретное предсказание для конкретного объекта $x_i$.
    2. $\phi_0 = \mathbb{E}[\hat{f}(X)]$ — среднее предсказание по популяции (точка отсчёта). SHAP по построению аддитивен: вклады $\phi_j(x)$ распределяют отклонение конкретного предсказания от среднего, поэтому $\phi_0 + \sum_j \phi_j(x) = \hat{f}(x)$.
    3. Глобальная важность усреднена по всей выборке (через $|\phi_j|$) и теряет информацию о конкретной строке: для отдельного игрока решающим может быть признак, неважный в среднем, причём с любым знаком. Объяснение одного кейса требует локального метода.
    4. Локальная интерпретируемость; SHAP values (или LIME) для конкретной строки этого игрока, чтобы показать, какие именно признаки толкнули score вверх.

!!! example "Мини-задание"

    Дано SHAP-разложение churn-предсказания одного игрока: $\phi_0 = 0.20$; days_since_last_deposit +0.30; bet_frequency_7d +0.15; deposit_amount_30d −0.12; vip_tier −0.03. За 2-3 минуты: (1) посчитай предсказание $\hat{f}(x)$ и проверь аддитивность; (2) назови главный драйвер риска и главный удерживающий фактор; (3) предложи одно retention-воздействие, подобранное именно под этого игрока, а не под "среднюю" глобальную важность.
