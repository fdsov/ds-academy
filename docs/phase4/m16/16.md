# M16.16 · Код: стек 2026

Сквозной пример: churn-модель по логу ставок. Стек — pandas 2.x / polars для агрегатов, scikit-learn 1.5+, LightGBM, Optuna 4.x.

### Признаки из лога ставок

```python
import polars as pl

# bets: player_id, ts (datetime), bet_amount, win_amount, game_id, deposit_amount
# cutoff — момент, ДО которого считаем признаки; таргет — отток ПОСЛЕ него
cutoff = pl.datetime(2026, 6, 1)

hist = bets.filter(pl.col("ts") < cutoff)

def window_aggs(df, days, suffix):
    start = cutoff - pl.duration(days=days)
    w = df.filter(pl.col("ts") >= start)
    return w.group_by("player_id").agg(
        pl.col("bet_amount").sum().alias(f"bet_sum_{suffix}"),
        pl.col("bet_amount").mean().alias(f"bet_mean_{suffix}"),
        pl.col("bet_amount").std().alias(f"bet_std_{suffix}"),   # импульсивность
        pl.col("ts").n_unique().alias(f"active_events_{suffix}"),
        pl.col("game_id").n_unique().alias(f"games_{suffix}"),
        pl.col("deposit_amount").sum().alias(f"dep_sum_{suffix}"),
    )

f7 = window_aggs(hist, 7, "7d")
f30 = window_aggs(hist, 30, "30d")

feats = f7.join(f30, on="player_id", how="full", coalesce=True)

feats = feats.with_columns([
    # отношение окон — ранний сигнал затухания
    (pl.col("bet_sum_7d") / (pl.col("bet_sum_30d") + 1)).alias("bet_ratio_7_30"),
    # доля выигрыша по логу (за 30д)
    # recency: дней с последней ставки до cutoff
    ((cutoff - hist.group_by("player_id").agg(pl.col("ts").max())
      .get_column("ts")).dt.total_days()).alias("days_since_last_bet"),
])
```

Логику recency на практике считают отдельным join'ом; здесь показан принцип — все окна смотрят строго назад от `cutoff`.

### Пайплайн без утечки

```python
import numpy as np
from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import RobustScaler, TargetEncoder
from sklearn.impute import SimpleImputer
from lightgbm import LGBMClassifier

num_cols = ["bet_sum_7d", "bet_mean_7d", "bet_std_7d", "dep_sum_30d",
            "bet_ratio_7_30", "days_since_last_bet"]
cat_cols = ["country", "payment_method"]

numeric = Pipeline([
    ("impute", SimpleImputer(strategy="median", add_indicator=True)),  # флаг пропуска
    ("scale", RobustScaler()),  # тяжёлые хвосты сумм
])

categorical = Pipeline([
    ("impute", SimpleImputer(strategy="constant", fill_value="Unknown")),
    # TargetEncoder делает out-of-fold cross-fitting только в fit_transform;
    # в пайплайне под CV вызывается именно он на train-фолде — не протекает
    ("encode", TargetEncoder(smooth="auto")),
])

prep = ColumnTransformer([
    ("num", numeric, num_cols),
    ("cat", categorical, cat_cols),
])

pipe = Pipeline([
    ("prep", prep),
    ("model", LGBMClassifier(objective="binary", n_estimators=600,
                             learning_rate=0.03, random_state=42)),
])
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude в Cursor, Copilot) сильны в feature engineering как генератор гипотез и борец с бойлерплейтом. Хороший рабочий промпт:

    "Вот схема таблицы bets (player_id, ts, bet_amount, win_amount, game_id, deposit_amount) и задача — предсказать отток на горизонте 14 дней по данным до cutoff-даты. Предложи 20 признаков, сгруппируй их по типу (recency, frequency, monetary, волатильность, оконные отношения), и для каждого укажи, на каком окне считать и почему он может предсказывать отток. Отдельно отметь признаки с риском утечки из будущего."

    Где помогает надёжно: накидать список оконных агрегатов, написать boilerplate ColumnTransformer, объяснить разницу между методами кодирования, перевести идею признака в polars/pandas код.

    Где врёт и что перепроверять обязательно. Первое — утечка: LLM легко предложит target encoding без out-of-fold или признак, считающий статистику по всему логу включая будущее. Каждый сгенерированный признак проверяй вопросом «использует ли он данные после cutoff». Второе — устаревший API: модель часто пишет `cross_val_score` со scaler, обученным до split, или старый sklearn-синтаксис; сверяйся с актуальной докой. Третье — выдуманные функции и параметры (галлюцинации API Optuna/LightGBM). Четвёртое — статистическая некорректность: предложит подбирать порог классификации на тех же данных, что и метрику. LLM ускоряет генерацию, но валидатор корректности — ты.

!!! note "Стек 2026"

    Актуально: scikit-learn 1.5+ с встроенным `TargetEncoder` (out-of-fold cross-fitting из коробки — больше не нужны сторонние обёртки для базового target encoding); Optuna 4.x как дефолт для тюнинга; polars для тяжёлых оконных агрегатов по логам (на больших данных кратно быстрее pandas); LightGBM и CatBoost с нативной обработкой категорий и пропусков; SHAP для интерпретации важности.

    Устарело относительно старых курсов: `GridSearchCV` как основной инструмент тюнинга (остался для крошечных сеток); ручное one-hot всего подряд (для деревьев — нативные категории); слепое применение `StandardScaler` ко всему перед любой моделью; самописный target encoding без регуляризации; `category_encoders` как единственный путь (теперь многое в самом sklearn).

!!! warning "Ловушки"

    - Target encoding без out-of-fold или фит энкодера до train/test split — утечка номер один, даёт фейковый AUC.
    - Масштабирование признаков перед бустингом — бесполезная работа, маркер непонимания того, как деревья делают сплиты.
    - Оконные агрегаты и лаги, заглядывающие за cutoff-дату — утечка из будущего, классика на churn и кредитном скоринге.
    - Подбор гиперпараметров и оценка качества на одних и тех же фолдах — оптимистичная оценка; нужна вложенная CV или честный hold-out.
    - Доверие встроенной feature importance деревьев — она смещена к высококардинальным признакам; используй permutation importance или SHAP.
    - One-hot высококардинальной категории (тысячи игр) — взрыв размерности и переобучение; бери частотное/target encoding или нативные категории.
    - Заполнение пропусков средним без флага `is_missing` — стирает информативность самого факта пропуска (часто MNAR в поведенческих данных).
    - min-max scaling на данных с китами — один выброс сжимает всех к нулю; для тяжёлых хвостов robust scaler плюс лог.
