# M16.5 · Кодирование категорий

Модели работают с числами. Категории надо перевести в числа так, чтобы не наврать модели о структуре данных.

### One-hot encoding

Интуиция: для каждой категории заводим отдельную колонку-флаг. Страна = {RU, KZ, UZ} превращается в три колонки 0/1. Модель не делает ложного вывода, что «KZ больше RU».

Формально: категория с $k$ уникальными значениями отображается в $k$ (или $k-1$, чтобы убрать линейную зависимость) бинарных столбцов, ровно один из которых равен 1. Это ортогональное представление без навязанного порядка.

Когда применять: номинальные категории с небольшим числом значений (примерно до 10-15). Обязательно для линейных моделей и нейросетей. Проблема — высокая кардинальность: 5000 игр дадут 5000 разреженных колонок, что убивает память и переобучает деревья.

### Label encoding

Интуиция: просто пронумеровать категории — RU=0, KZ=1, UZ=2.

Формально: биекция из множества категорий в $\{0,1,\dots,k-1\}$.

Когда применять: для порядковых признаков, где номер отражает реальный порядок (VIP bronze=0, silver=1, gold=2). Для номинальных категорий label encoding опасен с линейными моделями — он навяжет ложную метрику «UZ в два раза больше KZ». А вот деревьям label encoding номинальных категорий часто сходит с рук: дерево делает пороговые разрезы и может изолировать любое значение серией сплитов. LightGBM и CatBoost вообще принимают категории нативно — это предпочтительный путь в 2026.

### Target / mean encoding и его главная опасность

Интуиция: заменим категорию на средний таргет внутри неё. Для churn-задачи: каждую страну заменяем на среднюю долю ушедших игроков из этой страны. Сильный, компактный признак — одна колонка вместо тысяч, и в ней уже закодирована связь с целью.

Формально: для категории $c$ кодируем значением

$$
\hat{e}(c) = \frac{\sum_{i: x_i = c} y_i + \alpha \cdot \bar{y}}{n_c + \alpha}
$$

где $n_c$ — число объектов категории $c$, $\bar{y}$ — глобальное среднее таргета, $\alpha$ — сила сглаживания (regularization). Сглаживание тянет редкие категории к глобальному среднему, чтобы категория с двумя наблюдениями не получила экстремальную оценку.

Главная опасность — утечка таргета. Если ты считаешь среднее по таргету, используя строку, которую потом будешь предсказывать, ты «подсмотрел ответ». Модель запомнит, что для этого конкретного игрока среднее уже сдвинуто его собственным таргетом. На обучении метрика взлетает, в проде проваливается.

!!! warning "Как target encoding протекает и как его лечить"

    Наивный `groupby(cat).y.mean()` по всему трейну — это утечка: значение признака для объекта зависит от его собственного таргета. Лечения два. Первое — out-of-fold: считать кодировку для каждого фолда только по остальным фолдам (как делает sklearn `TargetEncoder` с внутренним cross-fitting). Второе — упорядоченное кодирование как в CatBoost (ordered target statistics): для каждого объекта используются только «прошлые» строки. И в любом случае фит энкодера должен жить внутри пайплайна и кросс-валидации, а не до неё.

### Частотное (frequency / count) encoding

Интуиция: заменим категорию на то, как часто она встречается. Редкие платёжные методы получат маленькое число, популярные — большое.

Формально: $\hat{e}(c) = n_c / N$ (доля) или просто $n_c$ (счётчик).

Когда применять: для высококардинальных категорий, когда сама популярность несёт сигнал. В антифроде это работает: редкий BIN-код карты или редкий комбо «гео + устройство» сам по себе подозрителен. Плюс — не использует таргет, поэтому не протекает.

!!! question "Проверь себя"

    1. Почему label encoding номинальной категории опасен для линейной модели, но часто сходит с рук дереву?
    2. В чём именно состоит утечка при наивном target encoding и какие два механизма её лечат?
    3. Чем частотное кодирование принципиально безопаснее target encoding, и в какой задаче популярность категории сама по себе сигнал?

??? success "Ответы"

    1. Линейной модели номер навяжет ложную метрику («UZ вдвое больше KZ») — она встроит этот порядок в коэффициент. Дерево делает пороговые разрезы и может изолировать любое значение серией сплитов, порядок номеров ему не мешает.
    2. Значение признака для объекта зависит от его собственного таргета — «подсмотрен ответ», метрика на трейне завышена. Лечат out-of-fold кодированием (для каждого фолда считаем по остальным) и упорядоченным кодированием как в CatBoost (только «прошлые» строки). Плюс фит энкодера внутри пайплайна и CV.
    3. Частотное не использует таргет вообще, поэтому структурно не может протечь. Популярность как сигнал — в антифроде: редкий BIN-код или редкое комбо «гео + устройство» подозрительны сами по себе.

!!! example "Мини-задание"

    Возьми категорию `country` со значениями RU (4000 строк), KZ (300), UZ (12) и глобальной долей оттока $\bar{y}=0.20$. Внутри UZ ушли 9 из 12 игроков. Посчитай target-кодировку UZ по формуле сглаживания при $\alpha=0$ и при $\alpha=20$. Сравни два числа и объясни в одной фразе, зачем нужно сглаживание для редкой категории.
