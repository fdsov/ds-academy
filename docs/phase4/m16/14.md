# M16.14 · Пайплайны sklearn, чтобы препроцессинг не утекал

Это место, где горят даже опытные. Если ты обучил `StandardScaler` или `TargetEncoder` на всём датасете до train/test split, статистики (среднее, std, средние по таргету) уже «видели» тест. Это утечка препроцессинга, и она завышает offline-метрику.

Правило: любая обучаемая трансформация должна фититься только на тренировочной части каждого фолда. Инструмент для этого — `Pipeline` и `ColumnTransformer`. Пайплайн склеивает препроцессинг и модель в один объект, который при кросс-валидации фитится заново на каждом train-фолде и только применяется к val-фолду. Утечка становится структурно невозможной.
