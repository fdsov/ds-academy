# M15.13 · Практика

### Быстрый квиз

Выбери вариант — ответ подсветится сразу.

```quiz
Чем принципиально отличается постановка задачи в unsupervised от supervised?
[ ] В unsupervised всегда меньше признаков, чем в supervised
[x] В unsupervised нет меток y, есть только признаки x, и модель ищет структуру в распределении p(x)
[ ] В unsupervised обязательно используется нейросеть вместо линейной модели
> В обучении без учителя нет таргета y и метрики ошибки — модель опирается только на признаки и ищет структуру данных.
---
Почему в обучении без учителя не бывает однозначно «правильного» ответа?
[ ] Потому что алгоритмы всегда сходятся к локальному минимуму
[ ] Потому что данных обычно слишком мало для точного ответа
[x] Потому что никто не задал, по какому признаку считать объекты похожими — разные алгоритмы дают разные, но одинаково валидные разбиения
> Результат бывает полезным или бесполезным для конкретной задачи, а не правильным или неправильным: критерий «похожести» выбирает сам аналитик.
---
Зачем перед K-means почти всегда нужна стандартизация признаков?
[x] K-means считает евклидово расстояние, и признак с большими единицами (депозит в тысячах) будет доминировать, разбив кластеры фактически только по нему
[ ] Без стандартизации алгоритм Ллойда не сходится
[ ] Стандартизация увеличивает число кластеров, которые можно найти
> Это ошибка номер один новичков: без нормировки депозит в тысячах полностью перебивает число сессий в десятках.
---
Почему DBSCAN правильно разделит два кластера-полумесяца, а K-means — нет?
[ ] DBSCAN заранее знает истинное число кластеров
[x] DBSCAN идёт по связным областям высокой плотности, а не по расстоянию до центра, поэтому прослеживает изогнутую форму
[ ] K-means не работает с двумерными данными
> K-means режет данные по сферам вокруг центроидов и не справляется с невыпуклыми формами; DBSCAN опирается на плотность.
---
Почему по картинке UMAP/t-SNE нельзя судить о расстояниях между «островами» кластеров?
[ ] Потому что UMAP всегда рисует кластеры одинакового размера
[ ] Потому что UMAP работает только с числом точек меньше тысячи
[x] Потому что расстояния, размеры и плотность кластеров на проекции искажены, а разные random seed дают разные картинки
> Это инструмент разведки и презентации гипотез, а не карта реальности; кластеризовать надо в исходном пространстве, а UMAP — только смотреть глазами.
---
Почему выход Isolation Forest нельзя автоматически превращать в бан игрока?
[ ] Потому что Isolation Forest часто пропускает реальный фрод
[x] Потому что аномалия не равно фрод: нетипичное бывает легитимным (новый VIP с огромным депозитом), это сигнал для расследования, а не приговор
[ ] Потому что detector работает только на размеченных данных
> Unsupervised-детектор аномалий — первая линия и фильтр; финальное решение принимают правила, ручная проверка или supervised-модель на подтверждённых случаях.
---
Что должно пересечься, чтобы кластеризация без таргета засчитывалась как хорошая?
[ ] Только высокий силуэт и низкий индекс Дэвиса — Болдина
[ ] Только бизнес-интерпретируемость сегментов
[x] Метрика (геометрически разумно) + бизнес (полезно) + стабильность (не случайно)
> Высокий силуэт сам по себе ничего не значит, если кластеры неинтерпретируемы или разваливаются при перезапуске — нужно пересечение всех трёх условий.
```

Разминка. Возьми датасет игроков (или сгенерируй RFM-признаки). Построй для K-means график силуэта по $k$ от 2 до 8. Выбери $k$, обоснуй выбор словами. Сравни результат с масштабированием и без — посмотри, как меняются кластеры.

Основное. На тех же данных запусти три алгоритма: K-means, AgglomerativeClustering (Ward) и HDBSCAN. Сравни число и состав кластеров. Профилируй кластеры K-means: посчитай средние признаки, дай сегментам имена, опиши поведение каждого. Построй UMAP-проекцию, раскрась точками по сегментам и проверь визуально, насколько кластеры разделимы.

Челлендж. Собери антифрод-пайплайн: на платёжных признаках обучи Isolation Forest и LOF, сравни топ-50 аномалий каждого метода (насколько пересекаются). Для топ-аномалий вручную (по сырым данным) определи, какие выглядят легитимными (VIP, новый крупный игрок), а какие реально подозрительны. Сделай вывод о precision детектора на глаз и о том, какие признаки стоит добавить.

Мини-проект. End-to-end сегментация игроков. Шаги: подготовка RFM-признаков с логарифмированием денег, масштабирование, выбор $k$ по силуэту, финальная кластеризация, профилирование и нейминг сегментов, проверка стабильности (перезапуск с разными seed / бутстрэп), UMAP-визуализация для отчёта, и одна страница с бизнес-выводами: какие сегменты есть, чем отличаются, какой оффер каждому. Защити выбор $k$ и каждое имя сегмента числами.
