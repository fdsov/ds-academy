```python
import numpy as np
import pandas as pd
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score
from sklearn.decomposition import PCA
from sklearn.ensemble import IsolationForest
import umap

# RFM-признаки игроков: Recency (дней с последней ставки),
# Frequency (число игровых дней), Monetary (суммарный депозит)
df = pd.DataFrame({
    "recency": rec_days,
    "frequency": active_days,
    "monetary": total_deposit,
})

# Масштабирование обязательно: депозиты и дни в разных единицах
X = StandardScaler().fit_transform(df)

# Подбор k через силуэт
for k in range(2, 8):
    labels = KMeans(n_clusters=k, n_init=10, random_state=42).fit_predict(X)
    print(k, round(silhouette_score(X, labels), 3))

# Финальная сегментация
km = KMeans(n_clusters=4, n_init=10, random_state=42)
df["segment"] = km.fit_predict(X)

# Профилирование кластеров — без этого числа кластеров бессмысленны
print(df.groupby("segment").agg(["mean", "size"]))
```

```python
# Визуализация структуры через UMAP (только посмотреть, не выводы)
emb = umap.UMAP(n_neighbors=15, min_dist=0.1, random_state=42).fit_transform(X)

import matplotlib.pyplot as plt
plt.scatter(emb[:, 0], emb[:, 1], c=df["segment"], s=5, cmap="tab10")
plt.title("UMAP projection of player segments")
plt.show()
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude, Cursor, Copilot) сильны в этом модуле на трёх задачах: быстро накидать boilerplate пайплайна (scaler → модель → метрики), объяснить параметры алгоритма, и помочь с интерпретацией кластеров («вот средние RFM по 4 сегментам, предложи бизнес-названия и гипотезы по офферам»).

    Пример рабочего промпта: «У меня 4 кластера игроков казино с такими средними по признакам recency/frequency/monetary: [таблица]. Дай каждому кластеру короткое имя, опиши поведение и предложи по одной маркетинговой гипотезе на сегмент. Отметь, какие выводы требуют проверки A/B-тестом.»

    Где LLM врёт и что перепроверять обязательно: модель уверенно посоветует «оптимальное k» по описанию данных, не видя их, — это галлюцинация, k выбирается только по силуэту и бизнес-смыслу на твоих данных. Модель может предложить запустить K-means без масштабирования — всегда проверяй наличие scaler. Модель путает «аномалия» и «фрод» и легко напишет код, который банит по Isolation Forest, — никогда не автоматизируй санкции на сыром выходе детектора аномалий. И LLM с удовольствием придумает интерпретацию любых кластеров, даже если они статистический мусор, — сначала проверь, что кластеры стабильны, потом проси интерпретацию.

!!! note "Стек 2026"

    Актуально: UMAP — стандарт нелинейной визуализации, вытеснил t-SNE почти везде (быстрее, лучше глобальная структура, масштабируется). HDBSCAN — дефолт разведочной кластеризации вместо ручного подбора eps в DBSCAN. Isolation Forest — рабочая лошадь антифрода. PCA жив и незаменим как быстрый линейный препроцессинг и диагностика. В scikit-learn 1.x HDBSCAN встроен (`sklearn.cluster.HDBSCAN`), отдельный пакет уже не обязателен.

    Устарело относительно старых курсов: t-SNE как основной инструмент визуализации (медленный, ломает глобальную структуру) — теперь это запасной вариант. «Просто запустить K-means с k=наугад» без силуэта и профилирования. Использование PCA для визуализации сложной нелинейной структуры — для картинки берут UMAP, PCA оставляют для дисперсии и препроцессинга. Для эмбеддингов из нейросетей (текст, поведение) связка «эмбеддинг → UMAP/HDBSCAN» в 2026 стала стандартным разведочным пайплайном.

!!! warning "Ловушки"

    - Забыть масштабирование перед K-means/PCA/детектором аномалий — признак с большими единицами захватит всё. Ошибка номер один.
    - Выбирать k по «красоте локтя» вместо силуэта и бизнес-смысла. Локоть часто не читается.
    - Делать выводы из расстояний и размеров кластеров на UMAP/t-SNE — они искажены, это не карта реальности.
    - Считать выход Isolation Forest приговором. Аномалия — это кандидат на расследование, не фродер.
    - Запускать K-means на нешаровых кластерах (полумесяцы, кольца) — он их разрежет неправильно; нужен DBSCAN/HDBSCAN.
    - PCA перед антифродом может убить редкий сигнал в направлении малой дисперсии. На каком сеньоры горят: уверенно ставят PCA(0.95) в начало пайплайна «для скорости» и теряют именно тот тонкий признак, который ловил мошенников.
    - Не фиксировать random_state и удивляться, что кластеры «прыгают» между запусками — а потом строить на этом продуктовые решения.
    - Кластеризовать на сырых RFM без логарифмирования денежных величин — распределение депозитов тяжелохвостое, киты растягивают всё.
