# M15.1 · Зачем этот модуль

Представь, что тебе дали выгрузку на миллион игроков казино: депозиты, ставки, частота входов, время сессий, любимые игры. Никто не написал рядом метку «этот хороший, этот фродер, этот вот-вот уйдёт». Просто сырые числа. И продакт спрашивает: «На какие группы делятся наши игроки? Кто из них ведёт себя странно? Можно ли описать поведение десятком чисел вместо двух сотен признаков?»

Это и есть мир обучения без учителя (unsupervised learning). Здесь нет правильного ответа, который модель должна угадать. Есть только структура данных, которую нужно вытащить наружу. Без этих методов ты застрянешь в трёх местах: не сможешь сегментировать аудиторию иначе как руками по порогам («депозит > 100» — это не сегментация, это произвол), не сможешь визуализировать данные размерности больше трёх, и не поймаешь аномалию, для которой у тебя нет размеченных примеров фрода (а размеченных примеров фрода почти никогда нет вовремя).

Если в модуле про обучение с учителем (см. модуль M13-M14) у тебя всегда был таргет и метрика, считающая ошибку, то здесь главная боль — как понять, что результат вообще хороший, когда сравнивать не с чем. Этому и научимся: строить структуру и честно её валидировать.
