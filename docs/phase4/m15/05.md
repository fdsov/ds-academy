# M15.5 · Иерархическая кластеризация

### Интуиция

Вместо того чтобы сразу резать на $k$ групп, строим дерево родства. Снизу вверх (агломеративный подход): сначала каждый объект — отдельный кластер. На каждом шаге сливаем два ближайших кластера. В конце всё слилось в один. Получается дерево слияний — дендрограмма. Хочешь 3 кластера — режешь дерево горизонтальной линией на нужной высоте.

Главное преимущество: не надо задавать $k$ заранее, можно посмотреть на дерево и выбрать уровень разреза. И само дерево информативно — показывает вложенную структуру (внутри VIP-игроков есть подгруппы).

### Формально

Агломеративная кластеризация требует определить расстояние между кластерами (linkage):

- single linkage: расстояние между ближайшими точками двух кластеров (склонен к «цепочкам»);
- complete linkage: расстояние между самыми дальними точками (компактные кластеры);
- average linkage: среднее попарное расстояние;
- Ward: слияние, минимально увеличивающее суммарную внутрикластерную дисперсию (чаще всего лучший дефолт, концептуально близок к K-means).

Дендрограмма — это диаграмма, где высота слияния равна расстоянию между объединяемыми кластерами. Большой вертикальный «прыжок» на дендрограмме означает, что мы слили далёкие группы — естественное место для разреза.

Сложность наивной реализации $O(n^3)$ по времени и $O(n^2)$ по памяти, поэтому на миллионах строк иерархическая кластеризация в лоб не работает — её применяют на десятках тысяч объектов или на предварительно агрегированных данных.

!!! question "Проверь себя"

    1. В чём главное преимущество агломеративной кластеризации перед K-means по части выбора числа кластеров?
    2. Что такое дендрограмма и что означает большой вертикальный «прыжок» на ней?
    3. Чем отличаются single, complete и Ward linkage, и какой обычно лучший дефолт?
    4. Почему иерархическую кластеризацию не запускают в лоб на миллионах строк?

??? success "Ответы"

    1. Не надо задавать $k$ заранее: строится дерево слияний, а уровень разреза выбирается потом, глядя на структуру.
    2. Дерево слияний, где высота слияния равна расстоянию между объединяемыми кластерами. Большой прыжок означает, что слили далёкие группы — естественное место для разреза.
    3. Single — по ближайшим точкам (склонен к цепочкам), complete — по самым дальним (компактные кластеры), Ward — минимизирует прирост внутрикластерной дисперсии (обычно лучший дефолт, близок к K-means).
    4. Сложность наивной реализации $O(n^3)$ по времени и $O(n^2)$ по памяти; применяют на десятках тысяч объектов или на агрегированных данных.
