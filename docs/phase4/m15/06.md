# M15.6 · DBSCAN: кластеризация по плотности

### Интуиция

K-means спрашивает «к какому из $k$ центров ты ближе». DBSCAN спрашивает иначе: «есть ли вокруг тебя толпа?» Кластер — это связная область высокой плотности точек. Если вокруг точки в радиусе $\varepsilon$ много соседей — она в гуще кластера. Если соседей мало, но она примыкает к гуще — она на краю. Если вокруг пусто — это шум, выброс. DBSCAN не пытается засунуть каждую точку в кластер; он честно помечает аномалии как «ничьи».

Это ровно то, что нужно для данных с выбросами и для кластеров сложной формы: два полумесяца DBSCAN разделит правильно, потому что он идёт по плотности, а не по расстоянию до центра.

### Формально

DBSCAN имеет два параметра: $\varepsilon$ (радиус окрестности) и `min_samples` (минимум точек в окрестности, чтобы считать область плотной). Определения:

- Core point (ядровая): в её $\varepsilon$-окрестности не меньше `min_samples` точек.
- Border point (граничная): не ядровая, но попадает в окрестность какой-то ядровой.
- Noise (шум): ни то, ни другое — выброс.

Кластеры формируются как связные компоненты ядровых точек (плюс примкнувшие граничные). Число кластеров не задаётся — оно выводится из данных.

Как выбрать $\varepsilon$: строят k-distance график — для каждой точки расстояние до её $k$-го соседа ($k$ = `min_samples`), сортируют по возрастанию. Резкий изгиб («колено») кривой указывает на разумный $\varepsilon$.

!!! info "Когда DBSCAN лучше K-means, а когда хуже"

    DBSCAN выигрывает: кластеры произвольной формы, есть выбросы/шум, число кластеров неизвестно. DBSCAN проигрывает: кластеры сильно разной плотности (один $\varepsilon$ на всех не подходит), очень высокая размерность (расстояния «схлопываются» — проклятие размерности), нужна предсказуемость на новых точках. Эволюция DBSCAN — HDBSCAN — снимает проблему единой плотности и в 2026 является сильным дефолтом для разведочной кластеризации.

!!! question "Проверь себя"

    1. Какой вопрос задаёт DBSCAN про каждую точку, в отличие от вопроса K-means?
    2. Дай определения core point, border point и noise.
    3. Почему DBSCAN правильно разделит два полумесяца, а K-means — нет?
    4. В каких ситуациях DBSCAN проигрывает, и что предлагает HDBSCAN?

??? success "Ответы"

    1. DBSCAN спрашивает «есть ли вокруг тебя толпа» (плотность в радиусе $\varepsilon$), а K-means — «к какому из $k$ центров ты ближе».
    2. Core — в её $\varepsilon$-окрестности не меньше `min_samples` точек; border — не ядровая, но попадает в окрестность ядровой; noise — ни то ни другое, выброс.
    3. DBSCAN идёт по связным областям высокой плотности, а не по расстоянию до центра, поэтому изогнутую форму он прослеживает; K-means режет по сферам вокруг центроидов.
    4. Проигрывает при сильно разной плотности кластеров (один $\varepsilon$ не подходит), очень высокой размерности и когда нужна предсказуемость на новых точках. HDBSCAN снимает проблему единой плотности.
