Базовый рабочий поток для churn-модели на градиентном бустинге. Стек: Python 3.12, scikit-learn 1.7+ (актуальная 1.9), CatBoost 1.2+, SHAP 0.46+.

```python
import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.metrics import roc_auc_score, average_precision_score
from catboost import CatBoostClassifier, Pool

# df: одна строка на игрока, таргет churn_14d (ушёл в ближайшие 14 дней)
features = [
    "first_deposit", "bets_day1", "sessions_7d", "days_since_last_deposit",
    "avg_bet", "ggr_30d", "country", "payment_method", "device", "traffic_source",
]
cat_features = ["country", "payment_method", "device", "traffic_source"]
target = "churn_14d"

X = df[features]
y = df[target]

# Стратификация обязательна при дисбалансе классов
X_train, X_valid, y_train, y_valid = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)

train_pool = Pool(X_train, y_train, cat_features=cat_features)
valid_pool = Pool(X_valid, y_valid, cat_features=cat_features)

model = CatBoostClassifier(
    iterations=2000,
    learning_rate=0.03,
    depth=6,
    l2_leaf_reg=3.0,
    eval_metric="AUC",
    # вес положительного класса при дисбалансе ~5% оттока
    auto_class_weights="Balanced",
    early_stopping_rounds=100,
    random_seed=42,
    verbose=200,
)
model.fit(train_pool, eval_set=valid_pool)

proba = model.predict_proba(X_valid)[:, 1]
print(f"ROC-AUC:  {roc_auc_score(y_valid, proba):.4f}")
print(f"PR-AUC:   {average_precision_score(y_valid, proba):.4f}")
```

Выбор порога под бизнес-задачу — мы хотим, чтобы из помеченных как «риск оттока» хотя бы 60% действительно уходили (precision), иначе ретеншн-бюджет жжётся впустую.

```python
from sklearn.metrics import precision_recall_curve

prec, rec, thr = precision_recall_curve(y_valid, proba)
# ищем минимальный порог, дающий precision >= 0.60
mask = prec[:-1] >= 0.60
best_thr = thr[mask][np.argmax(rec[:-1][mask])]
print(f"Порог: {best_thr:.3f}, recall при нём: {rec[:-1][mask].max():.3f}")

flagged = proba >= best_thr
```

Честная интерпретация через SHAP вместо встроенной важности.

```python
import shap

explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(valid_pool)

# Глобальная важность: средний абсолютный вклад каждого признака
shap.summary_plot(shap_values, X_valid, plot_type="bar")
# Разбор конкретного игрока: почему модель дала ему высокий риск
shap.plots.waterfall(shap.Explanation(
    values=shap_values[0], base_values=explainer.expected_value,
    data=X_valid.iloc[0], feature_names=features,
))
```

Для сравнения — линейный baseline с регуляризацией. На нём всегда стоит начинать: если бустинг не обыгрывает регуляризованную логистическую регрессию заметно, возможно, проблема в данных, а не в модели.

```python
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.compose import ColumnTransformer
from sklearn.linear_model import LogisticRegression

num_features = [f for f in features if f not in cat_features]
pre = ColumnTransformer([
    ("num", StandardScaler(), num_features),          # масштаб обязателен для L1/L2
    ("cat", OneHotEncoder(handle_unknown="ignore"), cat_features),
])
logreg = make_pipeline(
    pre,
    LogisticRegression(penalty="l1", solver="liblinear", C=0.5, class_weight="balanced"),
)
logreg.fit(X_train, y_train)
base_auc = roc_auc_score(y_valid, logreg.predict_proba(X_valid)[:, 1])
print(f"Baseline ROC-AUC: {base_auc:.4f}")
```

!!! note "Стек 2026"

    Актуально: градиентный бустинг (XGBoost / LightGBM / CatBoost) — по-прежнему король табличных данных, нейросети их не вытеснили; SHAP — стандарт интерпретации; scikit-learn 1.7+ (актуальная 1.9) как клей для пайплайнов; калибровка вероятностей (CalibratedClassifierCV, изотоническая регрессия) — обязательна, когда вероятности идут в денежные решения. Появились табличные foundation-модели вроде TabPFN-2.5 — сильны на маленьких выборках, но на больших гемблинг-данных бустинг остаётся практичнее.

    Устарело относительно старых курсов: ручной one-hot десятков категорий (CatBoost делает это нативно и лучше); опора на встроенную feature_importances_ как на истину (заменена SHAP и permutation importance); accuracy как метрика по умолчанию; представление, что нейросеть всегда сильнее на табличке — на практике до десятков тысяч строк бустинг почти всегда выигрывает по качеству, скорости и стоимости.

!!! warning "Ловушки"

    Утечка таргета (data leakage) — главный убийца. Признак days_since_last_deposit, посчитанный на момент после оттока, или агрегат, заглядывающий в будущее окно, дадут фантастический AUC на валидации и провал в продакшене. Всегда фиксируй точку отсчёта признаков строго до окна прогноза.

    Утечка через препроцессинг — масштабирование, отбор признаков или target encoding, выполненные на всей выборке до train/test split. Это сливает информацию из теста в обучение. Лечится оборачиванием всего препроцессинга в Pipeline и применением только внутри фолдов кросс-валидации.

    Оптимизация не той метрики — гонишься за ROC-AUC, когда бизнесу важна precision в топ-1% самых рисковых. На дисбалансе сеньоры горят, доверяя ROC-AUC вместо PR-AUC.

    Некалиброванные вероятности — бустинг и SVM выдают скоры, которые ранжируют хорошо, но как вероятности смещены. Если на их основе считается ожидаемый LTV или решается, сколько потратить на удержание, числа врут. Калибруй.

    Случайный сплит временных данных — в продуктовых задачах будущее предсказывают по прошлому, значит валидация должна быть по времени (train на раннем периоде, test на позднем), а не случайной. Случайный сплит завышает качество, потому что модель «подсматривает» в ту же временную эпоху.

    Дрейф признаков — модель, обученная на данных до маркетинговой кампании, деградирует после неё, когда состав трафика меняется. Мониторь распределения признаков в продакшене.
