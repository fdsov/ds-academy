# M13.14 · Общий ML-воркфлоу

1. Сформулировать задачу в терминах ML: что таргет, что признаки, какая метрика отражает бизнес-цель.
2. Собрать и почистить данные, зафиксировать момент предсказания (point-in-time), чтобы избежать утечек из будущего.
3. Разбить на train/validation/test (стратифицированно или по времени).
4. Построить базовую модель (baseline) — простую, чтобы было с чем сравнивать.
5. Препроцессинг и признаки — внутри Pipeline, обучаемый только на train.
6. Обучить, подобрать гиперпараметры по CV на train+validation.
7. Оценить на test ровно один раз правильной метрикой.
8. Проверить на утечки, стабильность, бизнес-смысл; задеплоить; мониторить дрейф.
