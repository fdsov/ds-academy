# M13.6 · Разбиение данных: train / validation / test

### Интуиция

Если ты учишь студента по задачнику, а потом экзаменуешь его по тем же задачам — он получит пятёрку, даже если просто зазубрил ответы. Чтобы проверить настоящее понимание, экзамен должен состоять из задач, которые он не видел. С моделью то же самое: оценивать качество на тех же данных, на которых училась, бессмысленно — она их "запомнила".

Поэтому данные делят на три части:

- **Train** — на чём модель учится (подбирает параметры).
- **Validation** — на чём ты подбираешь гиперпараметры и выбираешь между моделями.
- **Test** — финальный, неприкосновенный экзамен, который трогаешь ровно один раз в самом конце.

### Формально

Мы случайно (или стратифицированно/по времени) разбиваем выборку на три непересекающихся подмножества, обычно в пропорции вроде 60/20/20 или 70/15/15. Параметры модели $\theta$ оптимизируются на train. Гиперпараметры $\lambda$ (глубина дерева, сила регуляризации, learning rate) — те, что не подбираются градиентом, — выбираются по качеству на validation. Test даёт несмещённую оценку обобщающей способности.

!!! warning "Почему test трогают один раз"

    Каждый раз, когда ты смотришь на test и принимаешь решение на его основе, ты "протекаешь" информацию из него в модель. После десятков итераций "посмотрел тест — поправил модель" твоя оценка на тесте становится такой же оптимистично-смещённой, как на train. Test — это запечатанный конверт, который вскрывают в конце. Для подбора используй validation или кросс-валидацию.

!!! question "Проверь себя"

    1. За что отвечает каждая из трёх частей: train, validation, test?
    2. Почему нельзя оценивать качество модели на тех же данных, на которых она училась?
    3. Что плохого в том, чтобы много раз "посмотреть на test и поправить модель"?

??? success "Ответы"

    1. Train — на нём модель подбирает параметры. Validation — на нём подбирают гиперпараметры и выбирают между моделями. Test — финальная несмещённая оценка, трогается один раз.
    2. Модель "запомнила" обучающие данные, поэтому покажет завышенное качество, как студент, зазубривший задачник перед экзаменом по тем же задачам.
    3. Каждый такой просмотр протекает информацию из test в решения о модели. Через десятки итераций оценка на test становится так же оптимистично смещена, как на train.

!!! example "Мини-задание"

    Возьми датасет из 10 000 строк. Распиши конкретные размеры train/validation/test для пропорции 70/15/15. Затем ответь одним предложением: на какой из трёх частей ты будешь сравнивать LogisticRegression и LightGBM между собой, и почему именно на ней.
