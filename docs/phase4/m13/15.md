### Три глагола sklearn

- `fit(X, y)` — обучить (подобрать параметры на данных).
- `predict(X)` — получить предсказания. Для вероятностей — `predict_proba(X)`.
- `transform(X)` — преобразовать данные (для трансформеров: скейлеры, энкодеры). `fit_transform` — обучить и сразу применить.

### Пример: churn-классификация с правильным Pipeline

```python
import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split, StratifiedKFold, cross_val_score
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.impute import SimpleImputer
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import (
    classification_report, confusion_matrix,
    roc_auc_score, average_precision_score,
)

# синтетический churn-датасет: редкий положительный класс (ушли ~8%)
rng = np.random.default_rng(7)
n = 20_000
df = pd.DataFrame({
    "deposits_7d": rng.poisson(2, n),
    "avg_bet": rng.gamma(2, 5, n),
    "days_since_reg": rng.integers(1, 400, n),
    "geo": rng.choice(["UZ", "KZ", "RU", "TR"], n),
})
# таргет завязан на поведение + шум
logit = -2.4 - 0.5 * df["deposits_7d"] + 0.003 * df["days_since_reg"]
p = 1 / (1 + np.exp(-logit))
df["churn"] = (rng.random(n) < p).astype(int)
print("Доля положительного класса:", df["churn"].mean().round(3))

X, y = df.drop(columns="churn"), df["churn"]
X_tr, X_te, y_tr, y_te = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)

num = ["deposits_7d", "avg_bet", "days_since_reg"]
cat = ["geo"]

pre = ColumnTransformer([
    ("num", Pipeline([("imp", SimpleImputer(strategy="median")),
                      ("sc", StandardScaler())]), num),
    ("cat", OneHotEncoder(handle_unknown="ignore"), cat),
])

# весь препроцессинг внутри Pipeline => нет утечки из test в train
model = Pipeline([
    ("pre", pre),
    ("clf", LogisticRegression(class_weight="balanced", max_iter=1000)),
])

# честная оценка через стратифицированную CV на обучающей части
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
pr_auc_cv = cross_val_score(model, X_tr, y_tr, cv=cv,
                            scoring="average_precision")
print("PR-AUC по CV:", pr_auc_cv.mean().round(3))

# финальное обучение и единственная оценка на test
model.fit(X_tr, y_tr)
proba = model.predict_proba(X_te)[:, 1]
print("ROC-AUC:", round(roc_auc_score(y_te, proba), 3))
print("PR-AUC :", round(average_precision_score(y_te, proba), 3))

# порог под бизнес: например, ловим recall не ниже 0.6
threshold = 0.3
pred = (proba >= threshold).astype(int)
print(confusion_matrix(y_te, pred))
print(classification_report(y_te, pred, digits=3))
```

Обрати внимание: `average_precision_score` — это и есть PR-AUC. На churn с долей положительного класса около 8% baseline PR-AUC ≈ 0.08, и любое значение заметно выше — реальный сигнал, тогда как ROC-AUC будет выглядеть высоким даже у посредственной модели.

### Почему именно PR-AUC для редкого churn

Положительный класс (ушедшие) редкий, и нас волнует именно он — мы хотим точечно ловить уходящих, чтобы слать им удержание. ROC-AUC раздувается за счёт огромного числа TN (оставшихся), и модель кажется лучше, чем она полезна на практике. PR-AUC напрямую отражает компромисс "сколько ушедших поймали (recall) и насколько чисты наши предупреждения (precision)", что и есть бизнес-вопрос ретеншн-команды.

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude в Cursor, Copilot) отлично ускоряют рутину этого модуля: сгенерировать каркас Pipeline, вспомнить сигнатуру метрики, объяснить незнакомый параметр, накидать варианты признаков.

    Пример толкового промпта: "У меня бинарная классификация churn, доля положительного класса 8%, данные временные (регистрации за полгода). Предложи схему валидации, метрику и объясни, почему не accuracy. Покажи sklearn Pipeline с ColumnTransformer и оценкой через cross_val_score. Не используй random k-fold, объясни почему."

    Где LLM врут и что обязательно перепроверять: они склонны (1) предлагать обычный `train_test_split` или k-fold для временных данных — это утечка из будущего, всегда требуй time-series split явно; (2) считать accuracy на дисбалансе и радоваться — игнорируй, требуй PR-AUC/recall; (3) ставить препроцессинг до сплита или скейлер вне Pipeline — проверяй, что всё обучается только на train; (4) путать, что `average_precision_score` это PR-AUC, и иногда подсовывать неверную метрику под `scoring`. Любую метрику и схему валидации, которую предложил LLM, прогоняй через вопрос "а в проде у меня будет эта информация в этот момент времени?".

!!! warning "Ловушки"

    - **Accuracy на дисбалансе** — 0.99 у модели, которая всегда говорит "не фрод". Самая частая ошибка новичка.
    - **Random k-fold на временных данных** — обучение на будущем. Горят даже сеньоры под дедлайном. Только time-series split.
    - **Препроцессинг до сплита** — скейлер/импьютер/энкодер, обученный на всём датасете, протёк из test. Всё внутри Pipeline.
    - **Подгонка под test** — многократные итерации с оглядкой на test превращают его в train. Test вскрывают один раз.
    - **Утечка таргета через признак-следствие** — признак, существующий только из-за случившегося таргета. Спрашивай про каждый признак "был ли он известен ДО момента предсказания".
    - **Ресэмплинг (SMOTE) до сплита** — синтетика редкого класса попадает в валидацию. Только внутри фолда, через imblearn Pipeline.
    - **Дефолтный порог 0.5 на дисбалансе** — почти всегда не оптимален. Подбирай порог под цену ошибок на validation.
    - **Сравнение моделей по разным метрикам или на разных сплитах** — невалидно. Фиксируй random_state и метрику.
