```python
import numpy as np
from scipy import stats
from statsmodels.stats.proportion import proportions_ztest, proportion_effectsize
from statsmodels.stats.power import NormalIndPower

# 1. Power-анализ ДО эксперимента: сколько игроков нужно на группу
baseline = 0.080          # текущая конверсия в депозит
mde_abs = 0.005           # минимально интересный прирост: +0.5 п.п.
effect = proportion_effectsize(baseline + mde_abs, baseline)  # Cohen's h
n_per_group = NormalIndPower().solve_power(
    effect_size=effect, alpha=0.05, power=0.8, alternative="two-sided"
)
print(f"Нужно на группу: {np.ceil(n_per_group):.0f}")

# 2. Анализ результата: z-тест пропорций
# control: 41 280 игроков, 3 302 депозита; test: 41 190, 3 560
conv = np.array([3302, 3560])
nobs = np.array([41280, 41190])
zstat, pval = proportions_ztest(conv, nobs)
p_c, p_t = conv / nobs
lift = p_t - p_c
print(f"Контроль {p_c:.4f} -> тест {p_t:.4f}, lift {lift:+.4f}, p={pval:.4f}")
```

CUPED-коррекция: вычитаем доэкспериментальную выручку, чтобы срезать дисперсию.

```python
import numpy as np

def cuped(y, x):
    # x — метрика игрока за preperiod, измеренная ДО рандомизации
    x = x - x.mean()
    theta = np.cov(y, x, ddof=1)[0, 1] / np.var(x, ddof=1)
    y_adj = y - theta * x
    return y_adj, theta

# y_test, y_ctrl — выручка за эксперимент; x_* — за прошлый месяц
y_test_adj, theta = cuped(y_test, x_test)
y_ctrl_adj, _ = cuped(y_ctrl, x_ctrl)
t, p = stats.ttest_ind(y_test_adj, y_ctrl_adj, equal_var=False)
var_reduction = 1 - np.var(y_test_adj) / np.var(y_test)
print(f"Снижение дисперсии: {var_reduction:.1%}, p_cuped={p:.4f}")
```

Робастная проверка для скошенной выручки и поправка на множественность.

```python
from scipy.stats import mannwhitneyu
from statsmodels.stats.multitest import multipletests

# Манна-Уитни: устойчив к китам и тяжёлым хвостам
u, p_mw = mannwhitneyu(y_test, y_ctrl, alternative="two-sided")

# FDR на пачке метрик: deposit_conv, arpu, retention_d7, support_load, chargeback
pvals = [0.004, 0.03, 0.21, 0.40, 0.55]
reject, p_adj, _, _ = multipletests(pvals, alpha=0.05, method="fdr_bh")
print(list(zip(reject, np.round(p_adj, 3))))
```

Байесовский A/B на бета-биномиальной модели.

```python
import numpy as np
rng = np.random.default_rng(42)

# Beta(1,1) — равномерный априор; добавляем наблюдённые успехи/провалы
a_post = rng.beta(1 + 3302, 1 + (41280 - 3302), size=200_000)
b_post = rng.beta(1 + 3560, 1 + (41190 - 3560), size=200_000)
prob_b_better = (b_post > a_post).mean()
expected_uplift = (b_post - a_post).mean()
print(f"P(test лучше) = {prob_b_better:.3f}, ожид. uplift = {expected_uplift:+.4f}")
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM сильны как ускоритель рутины и проверка логики дизайна. Хорошие задачи: сгенерировать каркас power-анализа, объяснить разницу между тестами на твоих данных, найти концептуальную дыру в дизайне эксперимента, переписать вывод для нетехнического стейкхолдера.

    Пример промпта: "Я тестирую новый онбординг на конверсию в первый депозит. Baseline 8%, хочу детектировать +0.5 п.п. при alpha=0.05, power=0.8, единица рандомизации — user_id. Дай: (1) формулировку H0/H1, (2) расчёт размера выборки на группу с формулой, (3) список из 4 guardrail-метрик для гемблинга, (4) три проверки санитарии перед анализом. В конце перечисли, что я мог упустить."

    Где LLM врут и что обязательно перепроверять: модели регулярно путают p-value с вероятностью гипотезы (воспроизводят народное заблуждение), могут предложить остановить тест по достижению значимости (peeking), берут z-тест там, где данные скошены и нужна непараметрика, и галлюцинируют формулы размера выборки. Любую числовую формулу и любой вывод о значимости перепроверяй руками или в `statsmodels`. LLM — генератор гипотез о дизайне, а не источник истины о статистике.

!!! note "Стек 2026"

    Актуально сейчас: `statsmodels` и `scipy` остаются базой; `pingouin` для эргономичных тестов с эффект-сайзами из коробки; CUPED и его расширение CUPAC (регрессионная коррекция через ML-предсказание ковариаты) — стандарт дисперсионной редукции в зрелых командах; always-valid / sequential testing (доверительные последовательности, mSPRT) вытесняют наивный фиксированный дизайн в платформах непрерывного экспериментирования; `polars` заменяет `pandas` на тяжёлых выгрузках.

    Что устарело относительно старых курсов: слепое доверие фиксированному z-тесту с ежедневным подглядыванием; игнорирование SRM-проверок; "p<0.05 и катим" без размера эффекта и CI; ручные таблицы критических значений (всё считает код). Чистый байес без guardrail и чистый частотный подход без дисперсионной редукции оба считаются неполными — зрелая практика комбинирует.

!!! warning "Ловушки"

    - Интерпретация p-value как вероятности правоты гипотезы. Главная и самая дорогая ошибка индустрии.
    - Peeking: остановка теста на первом пересечении порога значимости. Раздувает реальный alpha в разы.
    - Путаница статистической и практической значимости: значимый, но микроскопический эффект на огромной выборке.
    - Игнор множественных сравнений: 20 метрик без поправки гарантируют ложного "победителя".
    - Неверная единица рандомизации: рандомизация по сессии вместо пользователя — загрязнение групп.
    - t-тест по среднему на скошенной выручке с китами без робастной проверки и без бутстрэпа.
    - Отсутствие guardrail: целевая метрика выросла, retention обвалился, узнали через месяц.
    - Пропущенный SRM: группы 50/47, анализ невалиден, но результат всё равно докладывают.
    - "Незначимо" трактуется как "доказано, что эффекта нет" — это разные утверждения.
    - Недомощный тест: запустили на неделю без power-анализа, получили шум, закрыли хорошую фичу.
