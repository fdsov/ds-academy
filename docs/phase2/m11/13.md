Рабочий пример: оценка эффекта бонуса на 30-дневный LTV на синтетических наблюдательных данных, где вовлечённость — конфаундер. Сравним наивную разницу средних, регрессию с правильным контролем и DoubleML (double machine learning) — текущий стандарт для гибкой корректировки.

```python
import numpy as np
import pandas as pd
import statsmodels.formula.api as smf

rng = np.random.default_rng(42)
n = 20_000

# Вовлечённость — конфаундер: гонит и бонус, и LTV
engagement = rng.normal(0, 1, n)
p_bonus = 1 / (1 + np.exp(-(0.9 * engagement - 0.2)))
bonus = rng.binomial(1, p_bonus)

true_effect = 12.0  # истинный причинный эффект бонуса на LTV, у.е.
ltv = (40 + true_effect * bonus + 25 * engagement
       + rng.normal(0, 10, n))

df = pd.DataFrame({"ltv": ltv, "bonus": bonus, "engagement": engagement})

naive = df.loc[df.bonus == 1, "ltv"].mean() - df.loc[df.bonus == 0, "ltv"].mean()
print(f"Наивная разница средних: {naive:.2f}  (смещена вверх конфаундером)")

ols = smf.ols("ltv ~ bonus + engagement", data=df).fit()
print(f"Регрессия с контролем engagement: {ols.params['bonus']:.2f}")
```

```python
# Double machine learning: гибкие модели для propensity и исхода + ортогонализация
from doubleml import DoubleMLData, DoubleMLPLR
from sklearn.ensemble import HistGradientBoostingRegressor, HistGradientBoostingClassifier

dml_data = DoubleMLData(df, y_col="ltv", d_cols="bonus", x_cols=["engagement"])

dml = DoubleMLPLR(
    dml_data,
    ml_l=HistGradientBoostingRegressor(),   # модель исхода E[Y|X]
    ml_m=HistGradientBoostingClassifier(),  # модель воздействия E[T|X] (propensity)
    n_folds=5,                              # cross-fitting против переобучения
)
dml.fit()
print(dml.summary)  # coef рядом с true_effect=12, с честным доверительным интервалом
```

Для графовой части — пакет, проверяющий backdoor-критерий и набор корректировки автоматически:

```python
# pip install dowhy
from dowhy import CausalModel

model = CausalModel(
    data=df,
    treatment="bonus",
    outcome="ltv",
    graph="""digraph {
        engagement -> bonus; engagement -> ltv; bonus -> ltv;
    }""",
)
estimand = model.identify_effect()          # применяет backdoor-критерий
print(estimand)                              # покажет {engagement} как adjustment set
estimate = model.estimate_effect(estimand,
            method_name="backdoor.propensity_score_weighting")
print(estimate.value)

# Обязательный шаг: refutation — пробуем "сломать" оценку
refute = model.refute_estimate(estimand, estimate,
            method_name="random_common_cause")
print(refute)
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude, Cursor, Copilot) сильны в причинном выводе на трёх задачах: (1) набросать DAG из текстового описания домена; (2) сгенерировать скелет кода под DoWhy/DoubleML/EconML; (3) объяснить, почему конкретная переменная — конфаундер, медиатор или коллайдер.

    Пример промпта, который работает:

    "Я оцениваю эффект выдачи бонуса (binary) на 30-day LTV в гемблинг-продукте. Доступные переменные: вовлечённость до бонуса, число депозитов после бонуса, регион, возраст аккаунта, статус VIP. Для каждой переменной классифицируй её как конфаундер / медиатор / коллайдер / нейтральную относительно эффекта бонус -> LTV, обоснуй причинно и скажи, включать ли её в adjustment set по backdoor-критерию. Затем нарисуй DAG в формате DOT."

    Где LLM врёт и что перепроверять обязательно: модель уверенно объявит переменную конфаундером, не зная вашего домена, — направление стрелок задаёте ВЫ, не она; "после бонуса" почти всегда означает медиатор/коллайдер, а LLM любит совать пост-воздейственные фичи в контроли. Проверяйте каждое допущение exclusion restriction для IV (LLM выдаёт инструменты, которые нарушают исключение). И никогда не верьте сгенерированному выводу "эффект значим" без refutation-тестов и проверки overlap — LLM их пропускает.

!!! note "Стек 2026"

    Актуально: DoWhy (граф + идентификация + refutation), EconML (CATE, причинные леса, DR-learner, X-learner), DoubleML (ортогональные/cross-fitted оценки — текущий золотой стандарт для наблюдательных данных), CausalPy и pymc для байесовских DiD/RDD/ITS. Для DiD на staggered rollout — пакеты, реализующие оценщики Каллавэя–Сант'Анны (differences или pyfixest в Python), потому что классический two-way fixed effects на разновременном воздействии смещён (Гудман-Бэкон, 2021).

    Устарело относительно старых курсов: "просто добавь контроли в OLS и читай коэффициент" — без DAG это рулетка; наивный propensity matching 1:1 без проверки баланса и overlap; классический TWFE-DiD как универсальный ответ. Сдвиг последних лет — от ручного подбора моделей к doubly-robust + machine learning для nuisance-функций с честными доверительными интервалами.

!!! warning "Ловушки"

    - Контроль за пост-воздейственной переменной (медиатор/коллайдер) — главная причина смещения у опытных аналитиков. Никогда не контролируйте то, что измерено после воздействия, без обоснования через DAG.
    - "Контролирую всё, что есть" — антипаттерн. Лишний контроль не безопасен; он может внести collider bias.
    - Игнорирование overlap/positivity. Если у обработанных нет похожих контрольных (propensity около 0 или 1), оценка экстраполирует в пустоту. Всегда стройте гистограмму propensity по группам.
    - Слабый инструмент в IV: при малой корреляции инструмента с воздействием оценка взрывается и сильно смещается. Проверяйте F-статистику первой стадии (эвристика — больше 10).
    - Парадокс Симпсона без причинной модели: спорить "агрегат или подгруппы" бессмысленно — ответ диктует DAG, а не данные.
    - Параллельные тренды в DiD принимаются на веру. Всегда показывайте предпериодные тренды и делайте placebo-тест на фиктивной дате.
    - Подмена ATE на ATT/LATE: matching даёт ATT, IV — LATE. Это РАЗНЫЕ величины. Не выдавайте "эффект на compliers" за "эффект на всех".
    - Утечка SUTVA через спилловеры (реферальные программы, VIP-чаты): эффект протекает в контроль, и наивная разность занижает истинный.
