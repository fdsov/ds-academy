# M9.13 · Код: стек 2026

Базовый стек прикладной статистики на июнь 2026: `numpy` 2.x, `scipy.stats`, `statsmodels`, и для bootstrap — встроенный `scipy.stats.bootstrap`. Pandas остаётся для подготовки данных; для тяжёлых объёмов всё чаще `polars`.

```python
import numpy as np
from scipy import stats
from statsmodels.stats.proportion import proportion_confint

rng = np.random.default_rng(42)

# Депозиты с тяжёлым правым хвостом (логнормальное) — реалистично для гемблинга
deposits = rng.lognormal(mean=3.5, sigma=1.0, size=4000)

n = deposits.size
mean = deposits.mean()
s = deposits.std(ddof=1)          # ddof=1 -> несмещённая дисперсия (поправка Бесселя)
se = s / np.sqrt(n)

# Доверительный интервал для среднего через t-распределение
t_crit = stats.t.ppf(0.975, df=n - 1)
ci_mean = (mean - t_crit * se, mean + t_crit * se)
print(f"Средний депозит: {mean:.2f}, SE={se:.3f}, 95% CI = "
      f"[{ci_mean[0]:.2f}, {ci_mean[1]:.2f}]")
```

```python
# Bootstrap для МЕДИАНЫ депозита — аналитической формулы SE нет
res = stats.bootstrap(
    (deposits,),
    np.median,
    n_resamples=10_000,
    confidence_level=0.95,
    method="percentile",
    random_state=rng,
)
print(f"Медиана: {np.median(deposits):.2f}, "
      f"bootstrap 95% CI = [{res.confidence_interval.low:.2f}, "
      f"{res.confidence_interval.high:.2f}], "
      f"bootstrap SE = {res.standard_error:.3f}")
```

```python
# Bootstrap для RETENTION (доля вернувшихся на день 7)
returned_d7 = rng.binomial(1, 0.22, size=5000)   # 1 = вернулся

k, m = returned_d7.sum(), returned_d7.size
p_hat = k / m

# Корректный интервал для доли — Wilson, не Wald
lo, hi = proportion_confint(k, m, alpha=0.05, method="wilson")
print(f"Retention D7 = {p_hat:.4f}, Wilson 95% CI = [{lo:.4f}, {hi:.4f}]")

# Тот же retention через bootstrap — совпадёт по смыслу, полезно для сложных знаменателей
boot = stats.bootstrap(
    (returned_d7,),
    np.mean,
    n_resamples=10_000,
    confidence_level=0.95,
    random_state=rng,
)
print(f"Bootstrap CI = [{boot.confidence_interval.low:.4f}, "
      f"{boot.confidence_interval.high:.4f}]")
```

```python
# MLE вручную: параметр экспоненциального времени до churn
# Для экспоненты MLE(lambda) = 1 / mean(x)
lifetimes = rng.exponential(scale=30.0, size=2000)   # истинное среднее ~30 дней
lambda_mle = 1.0 / lifetimes.mean()
print(f"MLE lambda = {lambda_mle:.4f}, "
      f"оценка среднего lifetime = {1/lambda_mle:.2f} дней")

# Проверка через scipy: fit подбирает параметры тем же MLE
loc, scale = stats.expon.fit(lifetimes, floc=0)
print(f"scipy MLE scale (=среднее) = {scale:.2f}")
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude, Cursor, Copilot) сильны в статистике как генератор рабочего кода и как объяснятор формул. Хороший рабочий промпт:

    > "У меня pandas DataFrame `df` с колонками `user_id`, `deposit_amount` (тяжёлый правый хвост), `geo`. Напиши код на scipy 2026: посчитай медианный депозит по каждому geo и bootstrap 95% доверительный интервал для медианы (10000 ресэмплов, percentile-метод). Объясни, почему для медианы нельзя использовать формулу sigma/sqrt(n)."

    Где LLM врут и что перепроверять обязательно:

    - Путают стандартное отклонение и стандартную ошибку в объяснениях — сверяй, какая величина в знаменателе $\sqrt{n}$.
    - Дают Wald-интервал для пропорций по умолчанию; для редких событий требуй явно Wilson/Agresti-Coull.
    - Уверенно выдают частотную трактовку доверительного интервала ("вероятность параметра внутри 95%") как корректную — это ошибка, лови её.
    - Подсказывают `ddof=0` для дисперсии, давая смещённую оценку, когда нужна несмещённая (`ddof=1`).
    - Могут предложить bootstrap там, где данные зависимы (временной ряд), без block-варианта — это даст заниженный интервал.

    Правило: LLM ускоряет код и проверяет твою интуицию, но интерпретацию результата подписываешь ты. Любую формулировку про "вероятность" и "доверие" перечитывай сам.

!!! note "Стек 2026"

    Актуально сейчас:

    - `scipy.stats.bootstrap` — встроенный, не нужны самописные циклы и сторонние пакеты для большинства задач.
    - `numpy` 2.x с `default_rng` вместо устаревшего `np.random.seed` и глобального состояния.
    - `statsmodels` для корректных интервалов пропорций (Wilson) и power-анализа.
    - `polars` всё чаще вытесняет pandas на этапе подготовки больших выборок.
    - bootstrap стал дефолтным инструментом неопределённости в продуктовой аналитике — дешёвый компьют сделал формулы менее нужными.

    Устарело относительно старых курсов:

    - Ручные таблицы критических значений t и хи-квадрат из учебников — заменены `stats.t.ppf` и аналогами.
    - `np.random.seed(0)` как стандарт воспроизводимости — теперь генератор `rng = np.random.default_rng()`.
    - Слепое нормальное приближение для пропорций и тяжёлых хвостов там, где bootstrap честнее и почти бесплатен.
    - Догма "несмещённость превыше всего" — современный взгляд через MSE и bias-variance (общий с ML).

!!! warning "Ловушки"

    - **Стандартное отклонение вместо стандартной ошибки.** Самая частая путаница. SD растёт со сложностью данных, SE падает с $n$. Доверительный интервал строится на SE.
    - **Частотная трактовка доверительного интервала.** "Вероятность, что $\mu$ внутри, 95%" — неверно для классического CI. Это байесовский язык.
    - **`ddof=0` по умолчанию.** `numpy.var` без аргумента делит на $n$ — смещённая дисперсия. Для оценки популяции нужен `ddof=1`.
    - **Wald-интервал на редких событиях.** Конверсия 1% на малой выборке через Wald даёт отрицательную нижнюю границу. Бери Wilson.
    - **Большая выборка лечит всё.** Объём бьёт дисперсию, не смещение. Смещённые 10 млн строк хуже честных 10 тысяч для вывода о популяции.
    - **Bootstrap из смещённой выборки.** Воспроизводит смещение, создавая ложную точность. Сначала чини отбор, потом считай интервалы.
    - **Среднее тяжёлого хвоста как репрезентативная метрика.** Средний депозит сильно тянется китами; медиана и перцентили честнее описывают типичного игрока.
    - **Игнор цензурирования в lifetime.** Считать средний срок жизни только по ушедшим — занизить, потому что ценные ещё живы. Нужен survival-анализ.
