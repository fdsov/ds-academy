# M9.9 · Bootstrap — рабочий инструмент 2026

### Интуиция

Формула стандартной ошибки $\sigma/\sqrt{n}$ существует для среднего. А для медианы депозита? Для 90-го перцентиля времени сессии? Для retention как доли с хитрым знаменателем? Для отношения двух метрик? Аналитических формул либо нет, либо они уродливы.

Bootstrap решает это грубой силой. Идея почти наглая: твоя выборка — лучшая доступная модель популяции. Так давай будем многократно "пересобирать" выборки из самой выборки (с возвращением), каждый раз пересчитывать метрику и смотреть, как она гуляет. Этот разброс и есть оценка неопределённости — без единой формулы про распределение.

Аналогия: у тебя один срез игроков. Ты как будто множество раз "переигрываешь" эту неделю, каждый раз случайно вытягивая игроков из имеющихся (некоторые попадут дважды, некоторые ни разу), и смотришь, насколько прыгает retention. Реальную популяцию ты заменил эмпирической.

### Формально

Пусть выборка $x_1, \dots, x_n$. Bootstrap-процедура для статистики $\hat\theta$:

1. Сформируй bootstrap-выборку: $n$ элементов, выбранных из исходной с возвращением.
2. Посчитай на ней статистику $\hat\theta^{*(b)}$.
3. Повтори шаги 1-2 $B$ раз (обычно $B = 1000 \dots 10000$).
4. Разброс набора $\{\hat\theta^{*(1)}, \dots, \hat\theta^{*(B)}\}$ оценивает неопределённость $\hat\theta$.

Стандартная ошибка bootstrap — это стандартное отклонение bootstrap-распределения. Доверительный интервал — percentile-метод: берёшь 2.5-й и 97.5-й перцентили набора $\hat\theta^{*}$.

### Когда bootstrap лучше формул

- Метрика без простой формулы SE: медиана, перцентили, GGR на игрока, отношение метрик, коэффициент Джини, AUC.
- Сильно несимметричное распределение (депозиты — классический тяжёлый правый хвост), где нормальное приближение из CLT хромает на разумных $n$.
- Когда хочется честный интервал, не предполагая нормальности.

### Ограничения

Bootstrap не творит данные из ничего. Если исходная выборка смещена — bootstrap воспроизведёт смещение, а не вылечит его. Плохо работает для оценки экстремальных квантилей (максимум) и при сильной зависимости наблюдений (временные ряды требуют block bootstrap). И он стоит вычислений, хотя на данных 2026 это секунды.

!!! question "Проверь себя"

    1. Сформулируй центральную идею bootstrap одной фразой: что чем заменяется?
    2. Перечисли четыре шага bootstrap-процедуры и скажи, как из набора $\{\hat\theta^{*(b)}\}$ получить SE и percentile-CI.
    3. В каких случаях bootstrap бессилен? Вылечит ли он смещённый отбор?

??? success "Ответы"

    1. Реальная популяция заменяется эмпирической: твоя выборка считается лучшей доступной моделью популяции, и из неё многократно пересэмплируют с возвращением.
    2. (1) сформировать выборку из $n$ элементов с возвращением; (2) посчитать на ней статистику; (3) повторить $B$ раз ($1000\dots10000$); (4) разброс набора оценивает неопределённость. SE — стандартное отклонение bootstrap-распределения; CI — 2.5-й и 97.5-й перцентили набора.
    3. Bootstrap не творит данные из ничего: смещённую выборку он воспроизведёт со смещением, а не вылечит. Плохо работает для экстремальных квантилей (максимум) и при зависимых наблюдениях (нужен block bootstrap).
