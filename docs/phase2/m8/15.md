Актуальный научный стек на июнь 2026: Python 3.13, NumPy 2.x, SciPy 1.15+, pandas 2.x. Управление окружением через `uv`. Для распределений — `scipy.stats`, для быстрых симуляций — генератор `numpy.random.default_rng` (legacy `np.random.seed` устарел).

```python
import numpy as np
from scipy import stats

rng = np.random.default_rng(42)

# Антифрод через теорему Байеса
p_bot = 0.005
sens, fpr = 0.99, 0.02
posterior = (sens * p_bot) / (sens * p_bot + fpr * (1 - p_bot))
print(f"P(бот | флаг) = {posterior:.3f}")  # 0.199

# Биномиальное: A/B-тест на конверсию депозита
n, p = 1000, 0.05
print(stats.binom.mean(n, p), stats.binom.std(n, p))  # 50.0, 6.89
print(stats.binom.pmf(53, n, p))  # вероятность ровно 53 депозитов

# Логнормальные депозиты: сырьё уродливо, лог — нормален
deposits = rng.lognormal(mean=6.0, sigma=1.2, size=100_000)
print(np.mean(deposits), np.median(deposits))  # mean >> median из-за хвоста
print(stats.shapiro(rng.choice(np.log(deposits), 4000)).pvalue > 0.05)
```

```python
# ЦПТ в действии: среднее логнормальных стремится к нормали
import numpy as np
from scipy import stats

rng = np.random.default_rng(7)
pop = rng.lognormal(6.0, 1.2, size=1_000_000)  # генеральная совокупность депозитов

means = [np.mean(rng.choice(pop, size=500)) for _ in range(5000)]
# Распределение средних близко к нормальному, хотя исходное — нет
stat, pval = stats.normaltest(means)
print(f"normaltest p-value средних: {pval:.3f}")  # обычно > 0.05

se_theory = np.std(pop) / np.sqrt(500)   # теоретическая стандартная ошибка
se_empir = np.std(means)                  # эмпирическая
print(f"SE теория {se_theory:.2f} ~ эмпирика {se_empir:.2f}")
```

    LLM (Claude Opus 4.x, Cursor, Copilot в 2026) отлично ускоряют рутину вокруг вероятностей: подобрать распределение под описание данных, сгенерировать симуляцию для проверки интуиции, объяснить вывод формулы по шагам, превратить словесную задачу в байесовский расчёт.

    Пример промпта: "У меня счётчик депозитов в час: среднее 4,2, дисперсия 11,8. Какое распределение уместно, что означает соотношение среднего и дисперсии, и какой scipy-объект использовать? Покажи проверку overdispersion на Python."

    Где LLM врут и что перепроверять: модели регулярно путают PDF и вероятность (заявляют, что $f(x)$ — это вероятность точки); забывают условие конечной дисперсии для ЦПТ; неверно применяют независимость к зависимым событиям; путают условные вероятности местами ($P(A|B)$ вместо $P(B|A)$ — ровно ошибка прокурора). Численные ответы Байеса всегда пересчитывай руками или кодом — LLM часто ошибается в base rate именно потому, что и людям эта задача контринтуитивна. И не доверяй сгенерированному p-value без понимания, какой тест применён.

!!! warning "Ловушки"

    - Base rate fallacy: игнорировать априорную частоту и доверять "точности" детектора. На редких событиях даже отличный классификатор даёт горы ложных срабатываний.
    - Путать плотность с вероятностью: $f(x)$ непрерывной величины может быть больше 1, это не вероятность.
    - Применять корреляцию Пирсона к нелинейным связям и делать вывод "связи нет" при $\rho = 0$.
    - Считать корреляцию причинностью — классика, на которой горят и сеньоры в презентациях.
    - Применять нормальные методы к сырым деньгам: депозиты логнормальны, среднее чека на малой выборке нельзя гонять через z-тест без логарифма или бутстрапа.
    - Складывать дисперсии зависимых величин без ковариационного члена.
    - Путать независимость и несовместность событий.
    - Применять Пуассон при overdispersion (дисперсия много больше среднего) — нужно отрицательное биномиальное.
    - Забывать, что ЦПТ требует конечной дисперсии и достаточного $n$; на тяжёлых хвостах и малых выборках приближение ломается.
