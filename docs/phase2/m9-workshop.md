## Что отрабатываем

Руководитель спрашивает: "Средний депозит — сколько?". Из модуля M9 ты знаешь, что одно число врёт своей определённостью. Этот воркшоп заставляет тебя пройти весь путь оценки параметра руками: от точечной оценки до интервала, который ты можешь правильно интерпретировать.

Конкретно отрабатываем понятия модуля:

- точечная оценка $\hat\theta$ и её свойства (несмещённость, состоятельность) — M9.5;
- метод максимального правдоподобия для экспоненциального распределения — M9.6;
- стандартная ошибка $\text{SE}=s/\sqrt{n}$ против стандартного отклонения $s$ — M9.7;
- доверительный интервал через t-распределение и его корректная трактовка — M9.8;
- bootstrap для метрики без аналитической формулы (медиана, тяжёлый хвост) — M9.9;
- смещение наивной оценки на усечённой выборке — M9.12.

Артефакт на выходе: таблица оценок (среднее, медиана) с двумя типами интервалов (t-CI и bootstrap-CI), численная демонстрация смещения наивной оценки и MLE-оценка времени до churn — с твоей письменной интерпретацией каждой цифры.

```python
import numpy as np
from scipy import stats

rng = np.random.default_rng(42)

# ИСТИННАЯ популяция: 200000 игроков. Мы её знаем целиком — для проверки оценок.
mu_log, sigma_log = 3.5, 1.0
population = rng.lognormal(mean=mu_log, sigma=sigma_log, size=200_000)
TRUE_MEAN = population.mean()       # истинное mu, которое мы пытаемся восстановить

# ВЫБОРКА: то, что реально попало к нам в таблицу за неделю
sample = rng.choice(population, size=400, replace=False)

print(f"Размер выборки n = {sample.size}")
```

Истинное среднее держим в `TRUE_MEAN` только чтобы в конце проверить, попали ли мы. В реальной работе его нет — есть только `sample`.

## Ход работы

### Шаг 1: Точечная оценка и стандартная ошибка

Зачем: отрабатываем M9.5 и M9.7. Считаем точечную оценку $\bar{x}$, несмещённую дисперсию с поправкой Бесселя (`ddof=1`) и стандартную ошибку. Главное — не спутать разброс данных ($s$) с разбросом оценки ($\text{SE}$).

```python
n = sample.size
mean_hat = sample.mean()
s = sample.std(ddof=1)            # ddof=1 -> несмещённая, поправка Бесселя (M9.5)
se = s / np.sqrt(n)               # SE: разброс оценки, не данных (M9.7)

print(f"Точечная оценка среднего: {mean_hat:.2f}")
print(f"Стандартное отклонение s = {s:.2f}  (разброс игроков)")
print(f"Стандартная ошибка   SE = {se:.2f}  (разброс самой оценки)")
print(f"Отношение s/SE = {s/se:.1f}  (= sqrt(n) = {np.sqrt(n):.1f})")
```

Что получилось: $s$ большое (игроки разнятся от мелких до китов), а $\text{SE}$ в $\sqrt{n}\approx 20$ раз меньше. Это две принципиально разные величины: $s$ не падает с ростом $n$, $\text{SE}$ падает как $1/\sqrt{n}$.

!!! question "Проверь себя"

    1. Почему `ddof=1`, а не `ddof=0`? Что произойдёт с оценкой дисперсии при делении на $n$?
    2. У тебя $s\approx 50$, а $\text{SE}\approx 2.5$. Какую из величин ты сообщишь руководителю как "насколько мы уверены в среднем"?
    3. Сколько игроков нужно, чтобы $\text{SE}$ упала вдвое?

??? success "Ответы"

    1. `ddof=1` даёт несмещённую оценку (поправка Бесселя): мы уже потратили данные на $\bar{x}$, поэтому отклонения от $\bar{x}$ в среднем меньше отклонений от истинного $\mu$. `ddof=0` систематически занижает дисперсию.
    2. $\text{SE}$ — именно она характеризует разброс оценки по выборкам. $s$ — про разброс самих игроков, к уверенности в среднем отношения не имеет.
    3. Вчетверо больше: $\text{SE}\propto 1/\sqrt{n}$, уменьшение вдвое требует роста $n$ в 4 раза (закон убывающей отдачи, M9.7).

### Шаг 2: Доверительный интервал для среднего через t-распределение

Зачем: M9.8. Превращаем точечную оценку в интервал правдоподобных значений. При неизвестной $\sigma$ берём t-распределение с $n-1$ степенями свободы.

```python
t_crit = stats.t.ppf(0.975, df=n - 1)        # квантиль t для 95%
ci_low = mean_hat - t_crit * se
ci_high = mean_hat + t_crit * se

print(f"t-критическое (df={n-1}): {t_crit:.3f}  (-> ~1.96 при больших n)")
print(f"95% t-CI для среднего: [{ci_low:.2f}, {ci_high:.2f}]")
print(f"Истинное mu = {TRUE_MEAN:.2f}  ->  внутри интервала: "
      f"{ci_low <= TRUE_MEAN <= ci_high}")
```

Что получилось: интервал шириной в несколько единиц вокруг оценки. Истинное $\mu$ почти наверняка внутри — но это не гарантия, а свойство процедуры.

!!! question "Проверь себя"

    1. Какая трактовка этого `[ci_low, ci_high]` корректна, а какая — частотная ошибка?
    2. Почему t-распределение, а не нормальное? Когда разница исчезает?

??? success "Ответы"

    1. Корректно: "95% интервалов, построенных этой процедурой по разным выборкам, накроют $\mu$". Ошибка: "вероятность, что $\mu$ внутри ЭТОГО интервала, равна 0.95" — $\mu$ фиксировано, случаен интервал, а не параметр.
    2. Мы заменили неизвестную $\sigma$ оценкой $s$; расплата — более тяжёлые хвосты t. При больших $n$ ($n\gtrsim 100$) t почти неотличимо от нормального, $t_{crit}\to 1.96$.

### Шаг 3: Bootstrap там, где формулы нет

Зачем: M9.9. Для среднего формула $\text{SE}=s/\sqrt{n}$ есть. Для медианы — нет, а медиана честнее описывает типичного игрока на тяжёлом хвосте. Bootstrap решает это грубой силой: пересэмплируем выборку из самой себя.

Сначала руками, чтобы понять механику (как в основном задании M9.14):

```python
def bootstrap_ci(data, stat_fn, B=10_000, alpha=0.05, seed=0):
    boot_rng = np.random.default_rng(seed)
    n = data.size
    estimates = np.empty(B)
    for b in range(B):
        resample = boot_rng.choice(data, size=n, replace=True)   # с возвращением
        estimates[b] = stat_fn(resample)
    se_boot = estimates.std(ddof=1)
    lo, hi = np.percentile(estimates, [100*alpha/2, 100*(1-alpha/2)])
    return se_boot, (lo, hi)

se_med, ci_med = bootstrap_ci(sample, np.median)
print(f"Медиана: {np.median(sample):.2f}")
print(f"Bootstrap SE медианы: {se_med:.2f}")
print(f"Bootstrap 95% CI медианы: [{ci_med[0]:.2f}, {ci_med[1]:.2f}]")
```

Теперь сверяем со встроенным `scipy.stats.bootstrap` (стек 2026 — самописные циклы не нужны):

```python
res = stats.bootstrap(
    (sample,), np.median,
    n_resamples=10_000, confidence_level=0.95,
    method="percentile", random_state=rng,
)
print(f"scipy bootstrap CI медианы: "
      f"[{res.confidence_interval.low:.2f}, {res.confidence_interval.high:.2f}]")
print(f"scipy bootstrap SE: {res.standard_error:.2f}")
```

Что получилось: твоя ручная реализация и `scipy` дают практически одинаковый интервал. Медианный CI заметно уже среднего и не тянется китами — для тяжёлого хвоста это более честная "типичная" метрика.

!!! question "Проверь себя"

    1. Какую популяцию bootstrap использует вместо настоящей и почему это работает?
    2. Почему bootstrap-CI медианы выглядит "дёрганым" (границы прыгают на конкретные значения выборки)?

??? success "Ответы"

    1. Эмпирическую: сама выборка считается лучшей доступной моделью популяции, из неё пересэмплируют с возвращением. Работает, потому что разброс пересчитанной метрики приближает её истинную неопределённость без формул.
    2. Медиана малой выборки — это конкретное наблюдённое значение; при пересэмплинге она перескакивает между соседними точками данных, отсюда дискретность границ.

### Шаг 4: MLE для времени до churn

Зачем: M9.6. Реализуем метод максимального правдоподобия для экспоненциального распределения — времени жизни игрока. Для экспоненты MLE выводится аналитически: $\hat\lambda = 1/\bar{x}$. Проверяем тремя способами: формула, перебор по сетке, `scipy.fit`.

```python
lifetimes = rng.exponential(scale=30.0, size=2000)   # истинное среднее ~30 дней

# (1) Аналитический MLE
lambda_mle = 1.0 / lifetimes.mean()
print(f"MLE lambda = {lambda_mle:.4f} -> среднее время жизни {1/lambda_mle:.2f} дней")

# (2) Перебор по сетке: максимум лог-правдоподобия
def loglik(lam, x):
    return np.sum(np.log(lam) - lam * x)      # log f(x) = log(lam) - lam*x

grid = np.linspace(0.005, 0.10, 2000)
ll = [loglik(l, lifetimes) for l in grid]
lambda_grid = grid[np.argmax(ll)]
print(f"MLE по сетке = {lambda_grid:.4f}  (совпадает с формулой)")

# (3) scipy.fit — внутри тот же MLE
loc, scale = stats.expon.fit(lifetimes, floc=0)
print(f"scipy MLE scale (=1/lambda) = {scale:.2f} дней")
```

Что получилось: все три метода сходятся к одному $\hat\lambda$, а $1/\hat\lambda\approx 30$ дней — совпадает с истинным scale. Это материализует свойство MLE из M9.6: оценка делает наблюдённые данные максимально правдоподобными, и для экспоненты это ровно обратное среднее.

### Шаг 5: Смещение наивной оценки

Зачем: M9.12 и предупреждение "большая выборка не лечит смещение" из M9.4. Имитируем типичную ловушку: "мы видим средний депозит только по тем, кто задепозитил крупно" — нижние 20% популяции не попадают в данные. Показываем, что рост объёма НЕ приближает оценку к истине.

```python
threshold = np.quantile(population, 0.20)   # отсекаем нижние 20% — selection bias

def biased_mean(size):
    draw = rng.choice(population, size=size, replace=False)
    kept = draw[draw >= threshold]          # видим только "крупных"
    return kept.mean()

print(f"Истинное среднее популяции: {TRUE_MEAN:.2f}\n")
print(f"{'n':>8} | {'наивная оценка':>15} | {'ошибка':>8}")
for size in [500, 5_000, 50_000, 200_000]:
    est = biased_mean(size)
    print(f"{size:>8} | {est:>15.2f} | {est - TRUE_MEAN:>+8.2f}")
```

Что получилось: с ростом $n$ ошибка НЕ уходит — оценка стабильно завышена на одну и ту же величину. Объём бьёт дисперсию (числа от запуска к запуску стабилизируются), но систематическое смещение отбора не лечит. Смещённые 200 000 строк дают точную оценку неправильной величины.

!!! question "Проверь себя"

    1. Почему bootstrap из этой усечённой выборки НЕ спас бы ситуацию?
    2. Что нужно чинить вместо наращивания данных?

??? success "Ответы"

    1. Bootstrap воспроизводит смещение исходной выборки, а не лечит его: он пересэмплирует те же усечённые данные и даст узкий, уверенный, но смещённый интервал — ложную точность.
    2. Процесс отбора и разметки: понять, кто не попал в данные (M9.12, вопрос "кто не попал в выборку?"), и включить их либо явно моделировать усечение/цензурирование.

К таблице обязательно письменная интерпретация (это и есть половина артефакта). Например: "Средний депозит выборки 41.7 (95% CI [38.2, 45.2]) — но он сильно тянется китами; медиана 28.5 (95% CI [26.1, 31.0]) честнее описывает типичного игрока. CI означает, что при повторении замера 95% таких интервалов накроют истинное среднее, — а не что истинное среднее лежит здесь с вероятностью 95%."
