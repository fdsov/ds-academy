Стандарт обучения нейросетей в 2026 — PyTorch (2.x). TensorFlow в новых проектах почти не стартуют. Ниже — полная сеть на табличных данных: предсказываем вероятность депозита игрока.

```python
import torch
import torch.nn as nn
from torch.utils.data import TensorDataset, DataLoader

# Устройство: CUDA (NVIDIA), MPS (Apple Silicon) или CPU
device = (
    "cuda" if torch.cuda.is_available()
    else "mps" if torch.backends.mps.is_available()
    else "cpu"
)

# X: признаки игроков (n_bets, avg_stake, days_since_login, ...), y: депозит 0/1
X_train = torch.tensor(X_train_np, dtype=torch.float32)
y_train = torch.tensor(y_train_np, dtype=torch.float32).unsqueeze(1)

train_ds = TensorDataset(X_train, y_train)
train_dl = DataLoader(train_ds, batch_size=256, shuffle=True)


class DepositNet(nn.Module):
    def __init__(self, n_features: int):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(n_features, 128),
            nn.BatchNorm1d(128),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(128, 64),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(64, 1),  # logits, без сигмоиды
        )

    def forward(self, x):
        return self.net(x)


model = DepositNet(n_features=X_train.shape[1]).to(device)

# BCEWithLogitsLoss объединяет sigmoid + BCE численно стабильно
loss_fn = nn.BCEWithLogitsLoss()
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=1e-2)

for epoch in range(50):
    model.train()
    for xb, yb in train_dl:
        xb, yb = xb.to(device), yb.to(device)
        optimizer.zero_grad()          # обнулить градиенты с прошлого шага
        logits = model(xb)             # forward pass
        loss = loss_fn(logits, yb)
        loss.backward()                # backprop: autograd считает градиенты
        optimizer.step()               # обновление весов

# Инференс
model.eval()
with torch.no_grad():
    probs = torch.sigmoid(model(X_val.to(device)))
```

Три детали, на которых горят: `optimizer.zero_grad()` обязателен — иначе градиенты накапливаются между шагами; `model.train()` / `model.eval()` переключают dropout и batchnorm в нужный режим; `BCEWithLogitsLoss` вместо ручной сигмоиды плюс BCE — это численно стабильно (избегает $\log 0$).

    Где врут и что перепроверять: LLM уверенно предлагают устаревшие паттерны (`Variable` из PyTorch 0.x, `model.cuda()` вместо `.to(device)`), путают, нужен ли `zero_grad` до или после `backward`, и любят галлюцинировать несуществующие аргументы оптимизаторов. Всегда сверяй API с официальной документацией PyTorch и запускай код — нейросеть на текст не запускает тензоры за тебя. Числа сходимости («такая сеть даст 0.95 ROC-AUC») — чистая выдумка, проверяй на своих данных.

!!! note "Стек 2026"

    Актуально: PyTorch 2.x как стандарт де-факто; `torch.compile` для ускорения; AdamW как дефолтный оптимизатор; mixed precision (`torch.autocast`, bfloat16) для экономии VRAM; Lightning или чистый PyTorch для структуры; GELU/SiLU в современных архитектурах. Для табличных данных нейросети по-прежнему часто проигрывают градиентному бустингу (XGBoost/LightGBM/CatBoost) — это не устарело, это правда; нейросети выигрывают на тексте, картинках, последовательностях и при наличии полезных эмбеддингов.

    Устарело относительно старых курсов: TensorFlow 1.x с `Session`/`feed_dict`; Keras как первый выбор для исследований; ручная инициализация и ручной backprop в продакшене; sigmoid/tanh как скрытые активации в глубоких сетях; SGD без momentum как дефолт.

!!! warning "Ловушки"

    - **Забыл `zero_grad()`** — градиенты накапливаются, обучение ведёт себя хаотично. Классика.
    - **Не переключил `model.eval()` на инференсе** — dropout и batchnorm работают в train-режиме, метрики плавают и занижены.
    - **Не масштабировал признаки** — сети чувствительны к масштабу входа сильнее, чем деревья. Депозиты в тысячах и «дни» в единицах без стандартизации ломают сходимость. Нормализуй вход.
    - **Утечка данных** — посчитал среднее/std для нормализации по всему датасету, включая валидацию. Fit скейлера только на трейне.
    - **NaN в лоссе** — почти всегда слишком большой learning rate или ручная сигмоида плюс `log` вместо `BCEWithLogitsLoss`. Понизь lr, используй стабильный лосс, добавь gradient clipping.
    - **Сравнение train/val лосса в неравных условиях** — dropout активен на трейне, поэтому train-лосс может казаться хуже val-лосса в моменте. Не паникуй раньше времени.
    - **Дисбаланс классов** — фрод и депозиты редки. Сеть выучивает «всегда предсказывай мажорный класс». Используй `pos_weight` в лоссе или взвешенный сэмплинг.
