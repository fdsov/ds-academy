# M18.12 · Практика

### Быстрый квиз

Выбери вариант — ответ подсветится сразу.

```quiz
Почему стопка из нескольких линейных слоёв без функций активации между ними эквивалентна одному линейному слою?
[ ] Потому что веса инициализируются одинаковыми значениями
[x] Потому что композиция линейных преобразований снова линейна: произведение матриц — одна матрица
[ ] Потому что градиент затухает и лишние слои не обучаются
[ ] Потому что dropout зануляет промежуточные слои
> $W_2(W_1 x) = (W_2 W_1)x = W'x$ — вся глубина схлопывается в одно матричное умножение, пока между слоями нет нелинейности.
---
В чём суть проблемы «умирающего ReLU»?
[ ] ReLU насыщается при больших положительных значениях входа
[x] При постоянно отрицательном входе производная ReLU равна нулю, и нейрон перестаёт учиться
[ ] ReLU всегда выдаёт отрицательные значения и ломает softmax
[ ] ReLU требует слишком много вычислений по сравнению с сигмоидой
> Для $z<0$ производная ReLU равна 0: если нейрон стабильно получает отрицательный вход, градиент через него всегда нулевой. Leaky ReLU даёт малый наклон слева, чтобы это смягчить.
---
Что по смыслу вычисляет backpropagation?
[ ] Прямой проход данных от входа к выходу сети
[ ] Нормализацию выходов каждого слоя по батчу
[x] Градиенты лосса по всем весам за один обратный проход через цепное правило
[ ] Случайное отключение части нейронов для регуляризации
> Backprop эффективно считает частные производные лосса по каждому весу, распространяя «ошибку слоя» от выхода к входу по цепному правилу.
---
Почему mini-batch градиентный спуск стал стандартом по сравнению с batch и чистым SGD?
[x] Он сочетает стабильность оценки градиента с эффективной векторизацией на GPU
[ ] Он всегда находит глобальный минимум лосса
[ ] Он считает градиент точно по всему датасету за один шаг
[ ] Он не требует выбирать learning rate
> Batch точен, но дорог и не влезает в память; SGD быстр, но шумен. Mini-batch (32–512 примеров) — золотая середина: стабильнее SGD и использует векторизацию GPU.
---
Чем AdamW предпочтительнее обычного Adam при использовании weight decay?
[ ] AdamW не использует momentum, поэтому сходится быстрее
[ ] AdamW не требует bias-коррекции моментов
[x] AdamW корректно отделяет регуляризацию (weight decay) от адаптивного шага
[ ] AdamW работает только на CPU и потому стабильнее
> В обычном Adam weight decay смешивается с адаптивным масштабированием и работает неправильно; AdamW применяет его отдельно, поэтому это дефолт для весовой регуляризации.
---
Почему нельзя инициализировать все веса нулями?
[ ] Нулевые веса вызывают NaN в лоссе на первом же шаге
[ ] Нулевые веса делают активации слишком большими и взрывают градиент
[x] Все нейроны слоя становятся идентичными, получают одинаковый градиент и навсегда остаются клонами
[ ] Нулевые веса несовместимы с оптимизатором Adam
> Это проблема симметрии: при нулевой инициализации нейроны слоя неотличимы и обновляются одинаково, поэтому сеть не способна выучить разные паттерны.
---
Как переобучение проявляется на кривых train/val лосса?
[ ] Оба лосса синхронно растут с самого начала обучения
[ ] Val-лосс падает быстрее train-лосса на всём протяжении
[x] Train-лосс падает почти в пол, а val-лосс перестаёт снижаться и начинает расти — растёт gap между ними
[ ] Train-лосс застывает на месте, а val-лосс монотонно падает
> Сеть начинает запоминать шум трейна: ошибка на обучении продолжает падать, а на новых данных растёт. Early stopping ловит момент, когда val-лосс развернулся вверх.
```

**Разминка.** Реализуй один нейрон с сигмоидой на чистом NumPy: forward pass для одного примера. Затем вручную выпиши производную лосса BCE по весу через цепное правило и сверь с численным градиентом (конечная разность).

**Основное.** Собери MLP из двух скрытых слоёв на синтетических табличных данных (например, `make_classification` или реальный churn-датасет). Реализуй полный цикл обучения в PyTorch: DataLoader, AdamW, валидация каждую эпоху, early stopping по val-лоссу. Построй кривые train/val лосса и определи момент переобучения.

**Челлендж.** Возьми тот же датасет и проведи контролируемые эксперименты: (1) убери все активации — убедись, что сеть деградирует до линейной модели по качеству; (2) сравни ReLU, tanh и GELU; (3) сравни SGD, SGD+momentum и Adam по скорости сходимости; (4) добавляй dropout с $p \in \{0, 0.2, 0.5\}$ и смотри на gap между train и val. Зафиксируй seed, меняй один фактор за раз.

**Мини-проект.** Построй скоринг вероятности депозита (или churn) на реальных продуктовых данных. Полный пайплайн: разделение train/val/test без утечки, нормализация признаков (fit на трейне), MLP с регуляризацией, подбор learning rate и архитектуры по валидации, финальная оценка на test через ROC-AUC и PR-AUC. Сравни результат с baseline-логрегрессией и LightGBM. Сделай честный вывод: победила ли нейросеть на твоих табличных данных, и если нет — почему (это нормальный и важный результат).
