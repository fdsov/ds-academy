# M20.8 · Эмбеддинги для поиска, кластеризации и классификации

### Интуиция

Один раз превратив тексты в векторы, ты получаешь швейцарский нож. Поиск: найди документы, чей вектор ближе всего к вектору запроса. Кластеризация: сгруппируй похожие обращения, чтобы увидеть топ-темы жалоб без ручной разметки. Классификация: обучи лёгкую модель поверх векторов. Всё это — без единого вызова дорогой генеративной LLM на этапе работы.

### Формально

Семантический поиск. Заранее считаешь эмбеддинги всех документов и складываешь в индекс. Для запроса $q$ считаешь $E(q)$ и ищешь top-$k$ документов по косинусной близости. На больших объёмах точный перебор медленный, поэтому используют приближённый поиск ближайших соседей (ANN), например алгоритм HNSW — граф, по которому поиск идёт за логарифмическое время с небольшой потерей точности.

Кластеризация. Применяешь к векторам обычные методы (см. модуль про unsupervised): K-Means по косинусной метрике или HDBSCAN на снижённой размерности (UMAP сверху). Результат — автоматические темы. Гемблинг-кейс: кластеризовал 50K тикетов, увидел, что 18% — про задержку вывода у конкретного платёжного провайдера. Это прямой сигнал продукту.

Классификация. Эмбеддинг как признаки для логрегрессии, kNN или градиентного бустинга. Дёшево, быстро, интерпретируемо, не требует GPU на инференсе.

!!! question "Проверь себя"

    1. Какие три прикладные задачи закрывают эмбеддинги без вызова генеративной LLM на этапе работы?
    2. Зачем при семантическом поиске на больших объёмах нужен ANN (например HNSW) вместо точного перебора?
    3. Почему эмбеддинг-классификатор поверх логрегрессии часто выигрывает у fine-tuning по цене?

??? success "Ответы"

    1. Семантический поиск, кластеризация (автоматические темы) и классификация поверх векторов.
    2. Точный перебор косинусной близости по всем документам медленный; ANN (HNSW — граф) ищет за примерно логарифмическое время с небольшой потерей точности.
    3. Эмбеддинг уже несёт "понимание языка" от большой модели; лёгкий классификатор сверху учится за секунды на сотне примеров, без GPU и разметочной инфраструктуры.

!!! example "Мини-задание"

    Собери 15-20 коротких тикетов с метками (вывод / бонус / тех / платёж / KYC), прогони через embedding-модель и обучи LogisticRegression поверх векторов. Проверь предсказание на 2-3 новых фразах. Засеки, сколько секунд заняло обучение. 5 минут.
