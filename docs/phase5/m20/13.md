# M20.13 · Агенты и tool use кратко

### Интуиция

Обычная LLM только генерирует текст. Агент — это LLM, которой дали инструменты (функции) и право решать, когда их вызвать. Спросил "сколько FTD вчера на рынке UZ" — агент сам решает вызвать функцию `run_sql`, получает число, вставляет в ответ. Это превращает болтливый текстовый генератор в исполнителя задач.

### Формально

Tool use (function calling): ты описываешь модели доступные функции (имя, параметры, JSON-схема). Модель в ответ возвращает не текст, а структурированный запрос на вызов функции с аргументами. Твой код исполняет функцию, возвращает результат модели, она продолжает. Агент — это цикл "модель думает → выбирает инструмент → получает результат → думает дальше" до достижения цели (паттерн ReAct: reasoning + acting). К 2026 это стандартизировано протоколом MCP (Model Context Protocol), который унифицирует подключение инструментов и источников данных к моделям. Рядом вырос протокол A2A (Agent2Agent, v1.0 в начале 2026) — слой для общения агентов между собой: MCP связывает агента с инструментами, A2A — агентов друг с другом в multi-agent системах. Предостережение: агенты мощны, но хрупки и дороги (много вызовов LLM, накопление ошибок по шагам). Для прода начинай с минимального числа инструментов и жёстких guardrails, особенно если инструмент пишет в боевые системы.
