## Что отрабатываем

Модуль M21 учит одному навыку, который и есть твоя ценность рядом с AI: принимать работу ненадёжного джуниора через code review. Теория описала пять классов проблем (статистический вывод, галлюцинации API, неверные формулы, утечка PII, ложная уверенность) и принцип «AI пишет — ты проверяешь и отвечаешь». Здесь ты прогоняешь этот принцип руками.

Мы не зовём настоящую LLM (хотя ты можешь). Вместо этого мы берём готовый файл `ai_draft.py` — реалистичный «ответ модели» с тремя намеренно зашитыми ошибками: несуществующий аргумент API, неверная формула размера выборки и выдуманное p-value из текста. Твоя задача — найти каждую, доказать запуском кода и независимым пересчётом, исправить и записать в **лог проверки**. Лог проверки и есть артефакт воркшопа: таблица «что проверял → как поймал → за сколько → чем заменил».

Отрабатываем конкретно:

- верификацию вывода LLM по чек-листу M21.5 (код запускай, числа пересчитывай, формулы сверяй с первоисточником, API сверяй с docs);
- распознавание трёх из пяти классов галлюцинаций на живом коде;
- паттерн «сгенерируй тесты к коду» и «критика собственного вывода» из M21.6;
- дисциплину: число из текста модели недействительно, пока не получено кодом.

Артефакт на выходе: `verification_log.md` с пойманными ошибками и `ai_draft_fixed.py`, проходящий тесты.

## Бизнес-кейс

Этот аудит существует не ради чистоты кода, а потому что на другом конце AI-черновика стоит реальное продуктовое решение и реальный бюджет. Здесь ты ловишь галлюцинации не абстрактно, а перед тем, как они превратятся в ошибочный раскат фичи.

!!! example "Ситуация"

    Ты — data scientist в growth-команде iGaming-продукта. Продакт-оунер платёжного флоу приносит ответ AI-ассистента: «сравнил FTD двух групп, p = 0.021, разница значима — раскатывай вариант B на всю базу», и там же оценка нужной выборки около 53 игроков на группу. Он просит твоё «ок» сегодня, чтобы завтра включить B в проде. Твоя подпись под этим ревью — и есть зона ответственности.

    - На кону: раскат изменения платёжной формы на весь трафик (ориентировочно 80 тыс новых игроков в месяц). Если эффект мнимый, ты вслепую ухудшишь FTD-конверсию — при базовых ~3% и среднем депозите ~50 у.е. цена ошибочного раската измеряется сотнями тысяч ₽ недополученного FTD в месяц.
    - Заниженный в ~100 раз размер выборки (53 вместо ~5290 на группу) означает решение по шуму: тест выглядит «значимым» просто потому, что недонабран, — и на этом шуме хотят раскатить фичу на всех.
    - Ограничение: работаешь только на синтетике, без доступа к реальным PII игроков; времени — один рабочий день, ответ нужен до запуска.

## Ход работы

### Шаг 1: получить «ответ модели» с зашитыми ошибками

Зачем: чтобы тренировать ловлю галлюцинаций, нужен правдоподобный неверный код — ровно то, что выдаёт уверенная LLM. Создай файл-черновик дословно. Три ошибки спрятаны, но не подсвечены.

```python
# ai_draft.py  — "сгенерировано ассистентом", НЕ проверять глазами на доверии
import polars as pl

def deposit_stats(df: pl.DataFrame) -> pl.DataFrame:
    # средний и медианный депозит по странам
    return (
        df.group_by("country")
          .agg(
              pl.col("deposit").mean().alias("avg_dep"),
              pl.col("deposit").median(interpolation="linear").alias("med_dep"),
          )
          .sort("avg_dep", descending=True)
    )

def sample_size_per_group(p1: float, p2: float) -> float:
    # размер выборки на группу, alpha=0.05, power=0.8
    z_a, z_b = 1.96, 0.84
    num = (z_a + z_b) ** 2 * (p1 * (1 - p1) + p2 * (1 - p2))
    den = (p1 - p2)
    return num / den

# Вывод модели текстом (как в чате):
#   "Сравнил FTD двух групп, p = 0.021 — разница значима, раскатывай B."
AI_CLAIMED_PVALUE = 0.021
```

Что получилось: три класса ошибок из M21.4 ждут проверки. Не ищи их глазами — лови методом.

### Шаг 2: запусти код и поймай галлюцинацию API

Зачем: чек-лист M21.5 начинается с «код запускай всегда». Галлюцинация API ловится мгновенно — несуществующий метод или аргумент не запустится.

```bash
uv run python -c "import polars as pl; from ai_draft import deposit_stats; print(deposit_stats(pl.read_parquet('bets.parquet')))"
```

Падает с ошибкой про `median`: аргумента `interpolation=` у `pl.Expr.median()` в polars нет (это привычка из numpy/pandas-`quantile`, которую модель перенесла «по аналогии»). Сверь с документацией текущей версии, не со вторым промптом.

```bash
uv run python -c "import polars as pl; help(pl.Expr.median)"
```

Исправь — у медианы нет параметра:

```python
pl.col("deposit").median().alias("med_dep"),
```

Что получилось: класс «галлюцинация API» пойман за секунды через запуск. Запиши в лог время и способ.

!!! question "Проверь себя"

    1. Почему этот класс ошибки самый дешёвый в обнаружении?
    2. В чём остаётся скрытая опасность галлюцинаций API, даже когда код запустился?

??? success "Ответы"

    1. Несуществующий метод/аргумент вызывает исключение при запуске — объективная проверка вместо «выглядит правильно». Не нужно знание предметной области, только прогон.
    2. Иногда галлюцинированный вызов случайно совпадает с реальным, но делающим не то (другой default, другое поведение) — тогда код запустится и тихо посчитает неверно. Поэтому за запуском всё равно идут тесты на известных входах.

### Шаг 3: сверь формулу с первоисточником, а не с чатом

Зачем: неверные формулы (M21.4) запускаются без ошибок и выглядят правдоподобно. Проверяются только тем, кто знает вывод. Формула размера выборки в `ai_draft.py` потеряла квадрат в знаменателе: должно быть $(p_1-p_2)^2$, а стоит $(p_1-p_2)$.

Эталон из теории модуля:

$$
n = \frac{(z_{1-\alpha/2} + z_{1-\beta})^2 \, \big(p_1(1-p_1) + p_2(1-p_2)\big)}{(p_1 - p_2)^2}
$$

Пересчитай руками точку опоры из мини-задания M21.4: $p_1=0.03$, $p_2=0.04$, $z=1.96$ и $0.84$. Числитель $\approx 7.84 \cdot (0.0291+0.0384) = 7.84 \cdot 0.0675 \approx 0.529$; знаменатель $(0.01)^2 = 0.0001$; итог $n \approx 5290$ на группу. Это твой порядок величины.

```python
# проверка против сломанной версии
from ai_draft import sample_size_per_group
print("AI version:", sample_size_per_group(0.03, 0.04))   # ~52.9 — абсурдно мало
```

AI-версия даёт ~53 игрока на группу вместо ~5290 — ошибка в 100 раз, то есть недели трафика на ветер. Исправь знаменатель:

```python
den = (p1 - p2) ** 2
```

Что получилось: класс «неверная формула» пойман независимым пересчётом по учебнику. Запиши эталонное число в лог — это твоя константа на будущее.

!!! question "Проверь себя"

    1. Почему второй промпт к той же модели не годится для проверки формулы?
    2. Как сломанная формула прошла бы Шаг 2 (запуск)?

??? success "Ответы"

    1. Модель не выражает калиброванной неуверенности и воспроизведёт ту же ошибку тем же уверенным тоном — замкнутый круг. Разрывает его только независимый первоисточник: учебник, документация, оригинальная статья.
    2. Запуск проверяет синтаксис и существование API, а не математику. Формула без квадрата — валидный Python, исключения нет; ошибка видна лишь при сверке вывода или пересчёте на известных числах.

### Шаг 4: пересчитай выдуманное p-value кодом

Зачем: центральная дисциплина M21.4 — число из текста LLM (`AI_CLAIMED_PVALUE = 0.021`) недействительно, пока не получено запущенным кодом. Статистику считаем явным инструментом (statsmodels), а не «на словах».

Сначала смоделируем вторую группу B с чуть иной конверсией, потом посчитаем z-тест на пропорциях честно.

```python
# verify_stats.py
import polars as pl
import numpy as np
from statsmodels.stats.proportion import proportions_ztest

a = pl.read_parquet("bets.parquet")
rng = np.random.default_rng(7)
b_ftd = rng.binomial(1, 0.034, a.height)        # группа B

count = [int(a["ftd"].sum()), int(b_ftd.sum())]
nobs  = [a.height, a.height]
stat, pval = proportions_ztest(count, nobs)
print(f"A FTD={count[0]}/{nobs[0]}  B FTD={count[1]}/{nobs[1]}")
print(f"z = {stat:.3f}, p = {pval:.4f}")

from ai_draft import AI_CLAIMED_PVALUE
print(f"AI claimed p = {AI_CLAIMED_PVALUE} | recomputed p = {pval:.4f} | match: {abs(pval-AI_CLAIMED_PVALUE)<0.005}")
```

Что получилось: реальное `p` приходит из statsmodels и почти наверняка не совпадает с «0.021» из текста — оно было сгенерированным токеном, а не вычислением. Даже если случайно близко, твой вывод теперь опирается на расчёт, а не на тон модели. Зафиксируй оба числа в логе.

!!! tip "Проверь и предпосылку, не только число"

    z-тест на пропорциях предполагает достаточное число успехов в каждой группе (грубо, $np \ge 10$ и $n(1-p) \ge 10$) и независимость наблюдений. При FTD-конверсии ~3% на 5000 это ~150 успехов — нормально. Но если бы выборка была 300, нормальное приближение поплыло бы, и нужен точный тест (Fisher). M21 прямо предупреждает: сеньоры горят на t-тесте для пропорций и игнорировании предпосылок. Проверка предпосылки — твоя, не модели.

### Шаг 6: закрой контур тестами

Зачем: паттерн «сгенерируй тесты к коду» (M21.6) + правило «прошёл тесты на известных входах» (M21.5). Тесты фиксируют исправления, чтобы регресс не вернулся.

```python
# test_fixed.py
import math
from ai_draft import sample_size_per_group, deposit_stats
import polars as pl

def test_sample_size_order_of_magnitude():
    n = sample_size_per_group(0.03, 0.04)
    assert 5000 < n < 5600          # эталон ~5290

def test_deposit_stats_runs():
    df = pl.read_parquet("bets.parquet")
    out = deposit_stats(df)
    assert out.height == df["country"].n_unique()
    assert out["med_dep"].min() > 0

def test_empty_input():
    empty = pl.DataFrame(schema={"country": pl.Utf8, "deposit": pl.Float64})
    assert deposit_stats(empty).is_empty()
```

```bash
uv run pytest -q
```

Что получилось: исправленный `ai_draft.py` проходит тесты, включая edge case с пустым вводом, который модель проигнорировала. Контур замкнут: черновик от AI → запуск → пересчёт → лог → тесты.

## Бизнес-вывод

Лог проверки — это технический артефакт. Стейкхолдеру нужен не он, а решение. Переведи результат аудита в ответ продакт-оунеру на языке решений и денег, а не p-value.

- [ ] Рекомендация: не раскатывать вариант B по текущему AI-ответу — заявленный «p = 0.021» не воспроизводится из кода, а оценка выборки занижена примерно в 100 раз; решение по такому тесту — это решение по шуму.
- [ ] Эффект в деньгах: остановка раската по мнимому эффекту защищает ~80 тыс игроков/мес от слепого ухудшения FTD; цена ошибки — сотни тысяч ₽ недополученного FTD в месяц плюс недели трафика, слитые на недонаборном тесте.
- [ ] Риски и допущения: цифры иллюстративны и посчитаны на синтетике; вывод верен при соблюдении предпосылок z-теста (np ≥ 10, независимость наблюдений); на малой выборке переходим на точный тест Фишера.
- [ ] Следующий шаг: добрать выборку до ~5290 на группу, пересчитать p честным `proportions_ztest`, и только после этого возвращаться к решению о раскате B.
- [ ] Как подать: говори решениями, а не метриками — «запуск откладываем на N дней до корректного теста, иначе риск минус сотни тыс ₽/мес», а не «у меня p получился другой».
