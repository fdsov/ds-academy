Покажем правильный цикл: AI генерирует черновик, ты пишешь верификацию. Ниже — функция retention и обязательная обвязка проверок, которую ты добавляешь сам, не доверяя её AI слепо.

```python
import polars as pl


def n_day_retention(events: pl.DataFrame, max_day: int = 30) -> pl.DataFrame:
    """Classic N-day retention по когортам даты первого депозита.

    Retention[N] = доля игроков когорты с хотя бы одним login
    ровно через N дней после первого депозита.
    """
    if events.is_empty():
        return pl.DataFrame({"day": [], "retention": []})

    first_dep = (
        events.filter(pl.col("event_type") == "deposit")
        .group_by("player_id")
        .agg(pl.col("event_date").min().alias("cohort_date"))
    )

    logins = events.filter(pl.col("event_type") == "login").select(
        "player_id", pl.col("event_date").alias("login_date")
    )

    joined = first_dep.join(logins, on="player_id", how="left").with_columns(
        (pl.col("login_date") - pl.col("cohort_date")).dt.total_days().alias("day")
    )

    cohort_size = first_dep.height
    retention = (
        joined.filter(pl.col("day").is_between(1, max_day))
        .group_by("day")
        .agg(pl.col("player_id").n_unique().alias("retained"))
        .with_columns((pl.col("retained") / cohort_size).alias("retention"))
        .sort("day")
        .select("day", "retention")
    )
    return retention
```

Теперь верификация — это твоя зона ответственности, её AI делать за тебя не должен:

```python
import polars as pl
from datetime import date


def test_retention_basics():
    # один игрок, депозит 1-го, login на день 1 и день 3
    df = pl.DataFrame(
        {
            "player_id": ["p1", "p1", "p1"],
            "event_date": [date(2026, 3, 1), date(2026, 3, 2), date(2026, 3, 4)],
            "event_type": ["deposit", "login", "login"],
        }
    )
    out = n_day_retention(df, max_day=5)
    d = dict(zip(out["day"].to_list(), out["retention"].to_list()))
    assert d[1] == 1.0   # login через 1 день есть
    assert d[3] == 1.0   # login через 3 дня есть
    assert d.get(2) is None  # на день 2 login отсутствует


def test_empty_input():
    empty = pl.DataFrame(
        schema={"player_id": pl.Utf8, "event_date": pl.Date, "event_type": pl.Utf8}
    )
    assert n_day_retention(empty).is_empty()
```

Статистику — отдельным, явным инструментом, а не «на словах» из чата. Для A/B по конверсии FTD:

```python
from statsmodels.stats.proportion import proportions_ztest

# group A: 120 FTD из 4000; group B: 150 из 3950
count = [120, 150]
nobs = [4000, 3950]
stat, pval = proportions_ztest(count, nobs)
print(f"z = {stat:.3f}, p = {pval:.4f}")
```

Число `p` приходит из statsmodels, а не из текста LLM. Это и есть принцип в действии.

!!! tip "AI-копилот: где нейросети помогают"

    Рабочая связка 2026: Claude Code или Cursor в IDE для агентной работы с репозиторием (читает контекст файлов, правит несколько файлов, гоняет тесты), Copilot для строчного автодополнения, чат-ассистент для объяснений и черновиков. У Copilot с 2025-26 появился и agent mode: он берёт issue и доводит до pull request самостоятельно, так что это уже не только построчное автодополнение. Главное преимущество агентных инструментов в том, что они видят твой реальный код и версии из окружения, что снижает галлюцинации API.

    Пример сильного промпта для рефакторинга легаси-скрипта расчёта GGR:

    «Вот скрипт ggr_legacy.py. Шаг 1: объясни словами, какие бизнес-корректировки (бонусы, налоги, возвраты) в нём зашиты и где. Шаг 2: предложи разбиение на чистые функции с type hints и docstring, не меняя бизнес-логику. Шаг 3: сгенерируй pytest на трёх известных мне кейсах, которые я приложу. Не трогай формулу GGR без явного подтверждения.»

    Где врут и что перепроверять обязательно: любые конкретные числа из текста (никогда не результат, пока не посчитан кодом); выбор и интерпретацию статтестов; формулы (сверяй с учебником); вызовы свежих API (сверяй с docs текущей версии); каузальные формулировки. И железно: никаких реальных PII игроков в облачный промпт.

!!! note "Стек 2026"

    Актуально: агентные IDE-ассистенты (Claude Code, Cursor) как основной режим, а не только автодополнение; модели с большим контекстным окном, позволяющие подать целый модуль кода; MCP-интеграции, дающие ассистенту доступ к схеме БД и инструментам; enterprise-инстансы с zero-retention и DPA для compliance-чувствительных команд; polars как быстрая альтернатива pandas, который AI всё ещё генерирует по умолчанию.

    Устарело относительно курсов 2021-2023: представление об AI только как о «умном автодополнении одной строки»; совет «никогда не используй сгенерированный код» (нереалистично) — заменён на «всегда верифицируй»; идея, что облачный чат безопасен для любых данных; вера, что более новая модель не галлюцинирует API (галлюцинирует, особенно на свежих релизах).

!!! warning "Ловушки"

    - Принять число из текста LLM за результат расчёта. p-value, среднее, доля из текста — всегда пересчитывай кодом.
    - Слепо доверять выбору статистического теста. Сеньоры горят на t-тесте для пропорций, игнорировании множественных сравнений и предпосылок.
    - Вставить реальные данные игроков в облачный чат. Compliance-катастрофа, возможна потеря лицензии.
    - Принять каузальный вывод модели из наблюдательных данных за истину. Корреляция — не причина, отвечаешь ты.
    - Не запустить сгенерированный код «потому что выглядит правильно». Edge cases (пусто, NaN, дубликаты) AI почти всегда игнорирует.
    - Просить вывод из данных, которых не дал. Получишь правдоподобную выдумку.
    - Терять навык. Если делегировать всё подряд без понимания, атрофируется способность поймать ошибку AI — а именно она и есть твоя ценность.
