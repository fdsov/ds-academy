conv = nn.Sequential(

Self-attention руками, чтобы увидеть формулу в коде.

```python
import torch
import torch.nn.functional as F

def self_attention(x):  # x: (batch, seq, dim)
    d = x.size(-1)
    q = k = v = x  # упрощение: без обучаемых проекций
    scores = q @ k.transpose(-2, -1) / d ** 0.5  # (batch, seq, seq)
    weights = F.softmax(scores, dim=-1)
    return weights @ v

out = self_attention(torch.randn(2, 5, 16))
print(out.shape)  # torch.Size([2, 5, 16])
```

Дообучение предобученного трансформера через Hugging Face — каркас 2026.

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification, Trainer, TrainingArguments
from datasets import load_dataset

model_name = "distilbert-base-multilingual-cased"
tok = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=2)

ds = load_dataset("csv", data_files="support_tickets.csv")["train"].train_test_split(test_size=0.2)

def prep(batch):
    return tok(batch["text"], truncation=True, padding="max_length", max_length=128)

ds = ds.map(prep, batched=True)

args = TrainingArguments(output_dir="out", per_device_train_batch_size=16,
                         num_train_epochs=3, eval_strategy="epoch", learning_rate=2e-5)

trainer = Trainer(model=model, args=args, train_dataset=ds["train"], eval_dataset=ds["test"])
trainer.train()
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM-ассистент (Claude, Cursor, Copilot) силён на этой теме в трёх вещах: генерация каркаса обучения, объяснение незнакомого слоя, отладка ошибок формы тензоров.

    Пример рабочего промпта: "Вот мой PyTorch-модуль с Conv2d и Linear. На входе тензор (32, 3, 128, 128). Падает с ошибкой несовпадения размеров на первом Linear. Посчитай форму после каждого слоя, найди, где я ошибся в in_features, и дай исправленный код. Объясни формулу выхода свёртки, которую ты применил."

    Где врут и что перепроверять обязательно. Первое — гиперпараметры: модель уверенно назовёт learning rate, batch size, число эпох, но это догадки, их валидируют экспериментом и кривой обучения, а не доверием. Второе — устаревший API: ассистент часто выдаёт старые имена аргументов Hugging Face (например `evaluation_strategy` вместо `eval_strategy`), всегда сверяй с актуальной докой версии. Третье — выдуманные модели и метрики: может назвать несуществующую модель на Hub или приписать ей вымышленный score, проверяй по карточке модели. Четвёртое — математика внимания: путает, по какой оси идёт softmax, и роли Q/K/V, проверяй формы вручную. Никогда не пускай сгенерированный train-цикл в дорогой прогон, не проверив на крошечном батче.

!!! note "Стек 2026"

    Актуально: PyTorch 2.x как доминирующий фреймворк, `torch.compile` для ускорения; Hugging Face Transformers + Datasets + PEFT как стандартный пайплайн дообучения; LoRA/QLoRA для параметр-эффективного fine-tuning; FlashAttention и вариации для дешёвого длинного контекста; RoPE и относительные позиционные кодировки вместо синусоид; Vision Transformer и ConvNeXt конкурируют с классическими CNN; диффузионные и flow-matching модели для генерации.

    Устарело относительно старых курсов: чистые RNN/LSTM как основной инструмент для текста — их вытеснили трансформеры; TensorFlow 1.x со статическими графами и sessions; обучение крупных моделей с нуля под каждую задачу вместо fine-tuning; ручная аугментация и feature engineering там, где предобученные представления решают задачу; абсолютные синусоидальные позиции как единственный вариант.

!!! warning "Ловушки"

    - Путать stride и padding при расчёте формы выхода — приводит к несовпадению размеров на стыке свёрточной и полносвязной частей. Считай форму на бумаге по формуле перед запуском.
    - Применять обычную RNN на длинных последовательностях и удивляться, что модель не ловит далёкие зависимости — это затухающий градиент, бери LSTM/GRU или трансформер.
    - Забыть позиционное кодирование в трансформере с нуля — сеть станет инвариантной к порядку и потеряет смысл последовательности. Тихий баг: метрики просто хуже, ошибки нет.
    - Дообучать с слишком большим learning rate — катастрофическое забывание (catastrophic forgetting), модель теряет предобученные знания. Для fine-tuning lr обычно на порядок-два меньше, чем при обучении с нуля (порядка 1e-5 ... 5e-5).
    - Утечка данных через padding и неправильную маску внимания — на чём горят и сеньоры: токены паддинга начинают влиять на результат, если не передать attention mask.
    - Оценивать CNN-классификатор по accuracy на несбалансированных данных (антифрод, отток) — при дисбалансе 1:100 модель, всегда предсказывающая "норма", даст 99% accuracy и ноль пользы. Смотри precision/recall/PR-AUC (см. модуль про метрики).
    - Квадратичная память внимания: наивно подать последовательность в десятки тысяч токенов — упрёшься в память GPU. Это $O(n^2)$, планируй длину контекста заранее.
