# M19.10 · Практика

### Быстрый квиз

Выбери вариант — ответ подсветится сразу.

```quiz
Что такое индуктивное смещение (inductive bias) архитектуры нейросети?
[ ] Ошибка, которая накапливается из-за неудачной инициализации весов
[x] Заранее зашитые допущения о структуре данных, сужающие пространство гипотез
[ ] Систематическое отклонение предсказаний модели от истинных меток
[ ] Дополнительный слой регуляризации против переобучения
> Архитектура — это материализованное индуктивное смещение: чем точнее оно совпадает с истинной структурой данных, тем меньше примеров нужно для обобщения.
---
Какое свойство свёртки обеспечивает разделение весов (weight sharing)?
[ ] Устойчивость к шуму во входных данных
[x] Инвариантность к сдвигу: признак ловится в любом месте кадра
[ ] Инвариантность к повороту и масштабу объекта
[ ] Автоматический подбор размера ядра под задачу
> Один и тот же фильтр применяется ко всем позициям, поэтому выученный признак распознаётся где угодно, а число параметров резко падает.
---
Откуда берётся затухающий градиент в обычной (vanilla) RNN?
[ ] Из-за слишком большого learning rate на длинных последовательностях
[x] Градиент многократно умножается на $W_h$ и производную $\tanh$, и при спектральном радиусе меньше 1 экспоненциально стремится к нулю
[ ] Из-за того, что pooling стирает информацию о прошлом
[ ] Из-за отсутствия позиционного кодирования в рекуррентном слое
> Сигнал от далёкого прошлого не доходит: произведение по длинной цепочке гаснет, и сеть не учит далёкие зависимости.
---
Какой элемент LSTM смягчает затухание градиента?
[ ] Выходной гейт, ограничивающий значения скрытого состояния
[x] Аддитивное обновление состояния ячейки $C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$
[ ] Замена $\tanh$ на ReLU внутри ячейки
[ ] Удвоение размерности скрытого состояния
> При гейте забывания близком к 1 градиент течёт по линии памяти почти без умножения на малую матрицу — затухание резко ослабляется.
---
Зачем в scaled dot-product attention делят на $\sqrt{d_k}$?
[ ] Чтобы нормировать веса внимания к сумме, равной 1
[x] Чтобы при больших размерностях softmax не уходил в насыщение
[ ] Чтобы снизить квадратичную сложность $O(n^2)$ по длине
[ ] Чтобы убрать необходимость в позиционном кодировании
> Деление масштабирует логиты; к сумме 1 их приводит уже сам softmax, который берут по строкам матрицы $QK^\top$.
---
Почему чистому self-attention нужно позиционное кодирование?
[ ] Иначе не получится посчитать матрицу $QK^\top$
[x] Softmax симметричен к перестановке, и без позиций последовательность для сети — мешок элементов без порядка
[ ] Без него взрывается число параметров multi-head attention
[ ] Оно заменяет остаточные связи и LayerNorm в блоке
> Внимание само по себе не знает порядка; вектор позиции прибавляют к эмбеддингу, чтобы вернуть сети информацию о следовании элементов.
---
В чём идея LoRA при дообучении предобученной модели?
[ ] Учить модель с нуля, но на меньшем датасете
[x] Заморозить исходные веса и учить крошечные низкоранговые добавки
[ ] Обрезать большинство слоёв, оставив только классификационную голову
[ ] Сжимать веса до 4 бит без дообучения
> Адаптеры весят мегабайты вместо гигабайтов, дообучение помещается на одну GPU, а базовые веса остаются нетронутыми.
```

Разминка. Для входа 32x32x3, свёртки с 16 фильтрами 5x5, stride 1, padding 0 посчитай руками форму выхода и число параметров слоя. Затем проверь кодом на PyTorch, что совпало.

Основное. Собери небольшую CNN на 3-4 свёрточных блока и обучи её на CIFAR-10 (или на своём датасете изображений). Построй кривые обучения train/val, найди точку переобучения, добавь dropout и аугментацию, покажи, как сдвинулась кривая. Отдельно реализуй Conv1D-модель, которая по последовательности из 20 ставок игрока предсказывает вероятность попытки фрода на следующем шаге (синтетические данные допустимы) — сравни её с LSTM на той же задаче по PR-AUC.

Челлендж. Реализуй один блок трансформера-энкодера с нуля на PyTorch: multi-head self-attention + residual + LayerNorm + FFN. Проверь на игрушечной задаче (например, определить, есть ли в последовательности повтор элемента). Затем визуализируй веса внимания одной головы тепловой картой и проинтерпретируй, на что она смотрит.

Мини-проект. Возьми реальную текстовую задачу из продукта: классификация обращений в поддержку по теме (платежи, бонусы, технические, верификация) или тональности. Собери и разметь хотя бы 1000-2000 примеров. Дообучи предобученный мультиязычный энкодер с Hugging Face, сравни три варианта: обучение головы поверх замороженной модели, полный fine-tuning, LoRA. Сведи в таблицу качество (macro-F1), время обучения и размер артефакта. Сделай вывод, какой режим выбрать под твой объём данных, и обоснуй через перенос обучения.
