# M7.10 · Измерения качества данных

Чтобы управлять качеством, его надо измерять. Индустриальный стандарт — шесть измерений (dimensions). Для каждого можно посчитать число от 0 до 1 и завести на него SLA.

Полнота (completeness): доля заполненных значений там, где они обязаны быть. $\text{Completeness} = 1 - \frac{\text{число пропусков}}{\text{число обязательных ячеек}}$.

Точность (accuracy): соответствие реальности. Самое трудноизмеримое, требует эталона (сверка суммы депозитов в логе с балансом платёжного провайдера).

Согласованность (consistency): данные не противоречат сами себе и другим источникам (дата депозита не раньше регистрации; сумма в логе сходится с витриной).

Своевременность (timeliness): данные свежие и приходят вовремя. Для антифрода задержка в час обесценивает данные; для месячного отчёта — нет.

Уникальность (uniqueness): нет дублей сущностей. $\text{Uniqueness} = \frac{\text{число уникальных сущностей}}{\text{число записей}}$.

Валидность (validity): значения в допустимом формате и диапазоне (валюта из справочника ISO, возраст 18-120 для гемблинга).

Эти шесть метрик превращают расплывчатое «данные плохие» в конкретику: «полнота поля country упала с 0.99 до 0.71 после релиза мобильного клиента». С этим уже можно идти к инженерам.

!!! question "Проверь себя"

    1. Чем полнота (completeness) отличается от валидности (validity) — приведите по примеру на логе ставок.
    2. Почему своевременность (timeliness) — относительная метрика, зависящая от задачи?
    3. Какую пользу даёт перевод «данные плохие» в число по конкретному измерению?

??? success "Ответы"

    1. Полнота — доля заполненных обязательных ячеек (например, `user_id` не пуст); валидность — соответствие формату/диапазону (валюта из справочника ISO, возраст 18-120). Поле может быть заполнено, но невалидно.
    2. Для антифрода задержка в час обесценивает данные, а для месячного отчёта та же задержка несущественна; порог свежести задаётся требованием к использованию.
    3. Число с SLA превращает расплывчатую жалобу в конкретику («полнота `country` упала с 0.99 до 0.71 после релиза мобильного клиента»), с которой можно идти к инженерам и отслеживать регрессии.
