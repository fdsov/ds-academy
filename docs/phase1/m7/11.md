# M7.11 · Воспроизводимый пайплайн очистки

Разовая чистка в ноутбуке — это технический долг. Завтра придут новые данные, и всё надо повторять руками, забывая шаги. Профессиональный артефакт — пайплайн: набор чистых детерминированных функций, прогоняющих сырые данные через фиксированную последовательность шагов с валидацией на входе и выходе.

Принципы. Идемпотентность: повторный прогон на тех же данных даёт тот же результат (важно для дозагрузок). Детерминизм: никаких «почистил на глаз», только воспроизводимые правила. Никогда не мутировать сырьё: сырые данные неприкосновенны (raw zone), чистка пишет новый слой. Логирование: каждый шаг сообщает, сколько строк затронул. Валидация на границах: контракт на входе и на выходе. Версионирование: и код пайплайна, и сами данные (DVC, lakeFS или просто партиционирование по дате загрузки).

!!! note "Medallion-архитектура"

    Индустриальный паттерн организации слоёв — bronze/silver/gold. Bronze — сырьё как пришло, неизменное. Silver — очищенное, типизированное, провалидированное. Gold — агрегированные витрины под конкретные задачи (например, таблица retention по когортам). Ваш пайплайн очистки — это переход bronze -> silver.
