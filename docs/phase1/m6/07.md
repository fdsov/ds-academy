# M6.7 · Добавление и изменение колонок, assign

### Интуиция

Новый столбец — это просто результат операции над существующими. "GGR = ставки минус выплаты", "это VIP, если депозит больше 10000". Векторно: операция применяется ко всему столбцу сразу, без циклов.

Ключевая мысль: в pandas ты почти никогда не пишешь `for i in range(len(df))`, чтобы посчитать новую колонку. Вместо этого ты описываешь формулу над целыми столбцами, а pandas (поверх NumPy) прогоняет её разом по всему массиву на уровне C. Это и быстрее в десятки-сотни раз, и читается ближе к математической записи. Думай о колонке не как о списке значений, по которым надо пройтись, а как о едином векторе, к которому применяется операция.

Аналогия с Excel: новая колонка — это как формула `=B2-C2`, "протянутая" вниз на все строки. Только в pandas ты пишешь формулу один раз для всего столбца, а не тянешь её мышкой.

### Формально

```python
df["ggr"] = df["bets"] - df["payouts"]
df["is_vip"] = df["total_deposit"] > 10_000

import numpy as np
df["segment"] = np.where(df["total_deposit"] > 10_000, "vip", "regular")

# несколько условий
df["tier"] = pd.cut(df["total_deposit"], bins=[0, 100, 1000, 10_000, np.inf],
                    labels=["bronze", "silver", "gold", "vip"])
```

Разберём каждую строку:

- `df["ggr"] = df["bets"] - df["payouts"]` — поэлементное вычитание двух столбцов. Pandas выравнивает их по индексу и считает разность для каждой строки. Если столбца `ggr` ещё нет — он создаётся; если есть — перезаписывается.
- `df["is_vip"] = df["total_deposit"] > 10_000` — сравнение возвращает булев столбец (`True`/`False`). Такие маски потом удобно использовать для фильтрации и для подсчёта долей (`df["is_vip"].mean()` даст долю VIP-игроков).
- `np.where(условие, A, B)` — векторный тернарный оператор: где условие истинно, берётся `A`, иначе `B`. Это замена `if/else` для целого столбца.
- `pd.cut(...)` — режет непрерывную величину на интервалы (бины) и присваивает каждому интервалу метку. `np.inf` как правая граница ловит всё, что больше 10000.

#### Разобранный численный пример

Пусть есть мини-таблица депозитов и оборотов игроков:

```python
import pandas as pd
import numpy as np

df = pd.DataFrame({
    "player": ["a", "b", "c", "d"],
    "bets": [1000, 50, 8000, 300],
    "payouts": [900, 80, 7000, 200],
    "total_deposit": [50, 120, 15_000, 1_500],
})

df["ggr"] = df["bets"] - df["payouts"]
df["margin"] = (df["bets"] - df["payouts"]) / df["bets"]
df["segment"] = np.where(df["total_deposit"] > 10_000, "vip", "regular")
df["tier"] = pd.cut(df["total_deposit"], bins=[0, 100, 1000, 10_000, np.inf],
                    labels=["bronze", "silver", "gold", "vip"])
```

Что получится:

| player | bets | payouts | ggr  | margin | segment | tier   |
|--------|------|---------|------|--------|---------|--------|
| a      | 1000 | 900     | 100  | 0.10   | regular | bronze |
| b      | 50   | 80      | -30  | -0.60  | regular | silver |
| c      | 8000 | 7000    | 1000 | 0.125  | vip     | vip    |
| d      | 300  | 200     | 100  | 0.333  | regular | gold   |

Обрати внимание на игрока `b`: его `ggr = -30` (казино в минусе по нему за этот период — игрок выиграл больше, чем поставил), а `margin` отрицательная. Это нормально на коротких окнах: GGR положителен в среднем по большой массе игроков, а на отдельном игроке за день может быть любым. Игрок `b` при депозите 120 попал в `tier = silver` (бин 100–1000), хотя по обороту он мелкий — `tier` тут считается по депозиту, а не по ставкам, важно не путать, что именно ты биннишь.

`assign` — это способ добавить столбцы, **возвращая новый DataFrame**, что идеально для method chaining (см. ниже):

```python
df2 = df.assign(
    ggr=lambda d: d["bets"] - d["payouts"],
    margin=lambda d: (d["bets"] - d["payouts"]) / d["bets"],
)
```

Лямбда внутри `assign` получает текущий DataFrame, поэтому можно ссылаться на столбцы, созданные в той же цепочке. Это делает код декларативным: видно, что каждый шаг порождает новое состояние, ничего не мутируется по дороге.

#### Зачем лямбда, а не просто значение

В `assign` можно передать столбец двумя способами:

```python
# через лямбду — ссылается на "тот df, что приходит в эту точку цепочки"
df.assign(ggr=lambda d: d["bets"] - d["payouts"])

# напрямую выражением — ссылается на исходный df из внешней области
df.assign(ggr=df["bets"] - df["payouts"])
```

Разница принципиальна внутри длинной цепочки. Если перед `assign` ты сделал фильтр или другой `assign`, то лямбда `lambda d: ...` увидит уже изменённый промежуточный DataFrame, а прямое выражение `df["bets"]` будет ссылаться на старый внешний `df` с другим набором строк — и pandas либо выровняет по индексу не так, как ты ждёшь, либо упадёт. Внутри цепочек почти всегда нужна лямбда.

Также `assign` умеет ссылаться на колонку, созданную в том же вызове, если порядок аргументов правильный:

```python
df.assign(
    ggr=lambda d: d["bets"] - d["payouts"],
    margin=lambda d: d["ggr"] / d["bets"],   # ggr уже существует к этому моменту
)
```

#### Method chaining: пример из аналитики

```python
report = (
    df
    .assign(ggr=lambda d: d["bets"] - d["payouts"])
    .assign(margin=lambda d: d["ggr"] / d["bets"])
    .query("bets > 0")
    .assign(segment=lambda d: np.where(d["total_deposit"] > 10_000, "vip", "regular"))
)
```

Цепочка читается сверху вниз как последовательность преобразований: посчитали GGR, посчитали маржу, выкинули строки с нулевыми ставками (иначе деление дало бы `inf`), разметили сегменты. Исходный `df` при этом не меняется — удобно для воспроизводимости и отладки.

### Пограничные случаи и типичные ошибки

- **`SettingWithCopyWarning`.** Если присвоить колонку срезу (`subset = df[df["bets"] > 0]; subset["x"] = ...`), pandas предупредит, что неясно — меняешь ты копию или оригинал. Решение: либо явная копия `subset = df[df["bets"] > 0].copy()`, либо считать через `assign`/`.loc[mask, "col"] = ...` на исходном df.
- **Деление на ноль.** `margin = ggr / bets` при `bets == 0` даст `inf` или `NaN`, а не ошибку (pandas/NumPy не падают на делении на ноль для float). Фильтруй такие строки заранее (`.query("bets > 0")`) или используй `np.where(d["bets"] > 0, d["ggr"]/d["bets"], np.nan)`.
- **Выравнивание по индексу.** При `df["x"] = some_series` pandas выравнивает по индексу, а не по позиции. Если у `some_series` другой индекс — на несовпавших местах появятся `NaN`. Чтобы присвоить по позиции, бери `.values` (`df["x"] = some_series.values`).
- **`pd.cut` и границы.** По умолчанию бины включают правую границу и исключают левую (`(0, 100]`). Значение `0` не попадёт ни в один бин с `bins=[0, ...]` и станет `NaN`. Если ноль важен — сдвинь левую границу (`bins=[-np.inf, 100, ...]`) или поставь `include_lowest=True`.
- **`np.where` и типы.** `np.where` возвращает массив одного типа. Если ветки разнотипны (число и строка), результат станет `object` — это нормально для меток, но проверяй dtype, если ждал число.
- **`assign` не мутирует исходный df.** `df.assign(x=...)` возвращает новый объект; если не присвоить результат (`df = df.assign(...)` или взять в цепочку), изменения "потеряются". Прямое `df["x"] = ...`, наоборот, меняет df на месте.

!!! tip "Когда что выбирать"

    `df["col"] = ...` — быстро и привычно для разовых правок и интерактивной работы. `assign` — когда строишь pipeline из нескольких шагов и хочешь, чтобы исходные данные остались нетронутыми, а код читался как последовательность преобразований.

!!! warning "Проверь себя"

    Чем `df.assign(x=...)` отличается от `df["x"] = ...` по влиянию на исходный DataFrame? (Подсказка: один возвращает новый объект, другой меняет на месте.)
