```quiz
Чем `Series` отличается от `DataFrame`?
[x] `Series` — один размеченный столбец (значения + индекс), `DataFrame` — таблица из нескольких `Series` с общим индексом строк

[ ] `Series` хранит только числа, а `DataFrame` только строки
[ ] Это синонимы, разницы нет
> `Series` — одномерный размеченный массив (один столбец), `DataFrame` — двумерная структура из нескольких `Series` с общим индексом строк.
---
По правилу 2026 года, какой формат выбрать для хранения и работы с данными?
[ ] CSV, потому что его читает человек
[x] Parquet — бинарный колоночный формат со сжатием и встроенной схемой
[ ] JSON, потому что он гибкий
[ ] Excel, потому что привычно
> CSV — для обмена с людьми; Parquet — для хранения и работы: он сжат, помнит типы и читается колоночно в разы быстрее.
---
Чем `loc` отличается от `iloc`?
[x] `loc` ищет по меткам (label), `iloc` — по позициям (integer position)
[ ] `loc` ищет по позициям, `iloc` — по меткам
[ ] Оба ищут только по числовым номерам строк
[ ] `loc` для строк, `iloc` для столбцов
> Мнемоника: l = label (метка), i = integer position (позиция). Когда метки случайно числа, их легко перепутать.
---
Какой из трёх режимов `groupby` возвращает результат той же длины, что и группа, размножая групповую характеристику обратно на каждую строку?
[ ] `agg`
[x] `transform`
[ ] `apply`
[ ] `pivot`
> `agg` сворачивает группу в одно число, а `transform` возвращает результат той же длины — например, "доля депозита игрока от суммы по его стране".
---
Что происходит при `left join`, если ключ в правой таблице не уникален?
[ ] Pandas удалит лишние строки слева
[x] Строки слева размножатся, и суммы (например GGR) раздуются кратно
[ ] Join упадёт с ошибкой автоматически
[ ] Правая таблица обрежется до уникальных ключей
> Это главный тихий баг аналитики. Лекарство: указывать `validate=` и сверять `len(df)` до и после merge.
---
Что значит "lazy" режим в Polars и почему он быстрее eager-подхода pandas?
[x] Polars строит план всего запроса, оптимизирует его (pushdown, переупорядочивание) и исполняет только на `.collect()`
[ ] Polars выполняет каждую строку кода сразу, как только её встретит
[ ] Lazy означает, что Polars пропускает часть данных
[ ] Это режим, в котором Polars работает на одном ядре
> Eager (pandas) исполняет немедленно, не видя пайплайн целиком; lazy (Polars) строит и оптимизирует план до исполнения — отсюда projection/predicate pushdown и параллелизм.
---
Почему `apply` в `groupby` бывает медленным на большом числе групп?
[ ] Он копирует всю таблицу в память на каждом шаге
[x] Он вызывает питоновскую функцию для каждой группы — интерпретируемый цикл без векторизации
[ ] Он всегда сортирует данные перед вычислением
[ ] Он работает только в один поток на уровне диска
> Встроенные агрегаты (`sum`, `mean`, `nunique`) реализованы в C/Cython; `apply` — последнее средство, сначала ищи агрегат, `transform` или векторную операцию.
```
