# M6.8 · groupby: split-apply-combine во всю глубину

### Интуиция

`groupby` — самая важная операция в аналитике. Идея в трёх словах: **разбей, примени, собери** (split-apply-combine). Разбей таблицу на группы (например, по игроку или по каналу), к каждой группе примени функцию (сумма, среднее, своя логика), собери результаты обратно в таблицу.

"Сколько каждый игрок задепозитил?" — группируем по `player_id`, к каждой группе применяем сумму `deposit`, собираем. "Какой средний чек по каналам?" — группируем по `channel`, применяем среднее. Это паттерн, который покрывает 80% аналитических задач.

### Формально

Парадигму split-apply-combine формализовал Хэдли Уикхем (2011). Pandas `groupby` реализует её буквально: объект `GroupBy` — это ленивое отображение "ключ группы → набор позиций строк". Реальное вычисление происходит, когда ты вызываешь агрегацию.

```python
# базовая агрегация
g = df.groupby("channel")["deposit"].sum()

# несколько агрегатов сразу, с именами столбцов (named aggregation)
report = df.groupby("channel").agg(
    players=("player_id", "nunique"),
    total_dep=("deposit", "sum"),
    avg_dep=("deposit", "mean"),
    ggr=("ggr", "sum"),
)

# группировка по нескольким ключам
df.groupby(["country", "channel"])["ggr"].sum()
```

### Три режима применения: agg, transform, apply

Здесь начинается то, что отличает джуна от мидла. У `groupby` есть три способа "применить" функцию, и они дают результаты разной формы.

`agg` (агрегация) — сворачивает каждую группу в **одно число**. На входе группа из N строк, на выходе одна строка. Результат короче исходной таблицы.

`transform` — применяет функцию к группе, но возвращает результат **той же длины, что группа**. Используется, когда нужно вернуть групповую характеристику обратно к каждой строке. Классика — "доля депозита игрока от суммы по его стране":

```python
df["country_total"] = df.groupby("country")["deposit"].transform("sum")
df["share_in_country"] = df["deposit"] / df["country_total"]
```

Здесь `transform("sum")` посчитал сумму по стране и "размножил" её обратно на каждую строку этой страны. Через `agg` так нельзя — он схлопнул бы таблицу.

`apply` — самый гибкий и самый опасный. Принимает произвольную функцию, получающую целый sub-DataFrame группы, и может вернуть что угодно. Мощно, но медленно.

```python
def second_purchase_gap(g):
    g = g.sort_values("ts")
    if len(g) < 2:
        return pd.NA
    return (g["ts"].iloc[1] - g["ts"].iloc[0]).days

df.groupby("player_id").apply(second_purchase_gap)
```

### Почему apply бывает медленным

`apply` вызывает питоновскую функцию для **каждой группы** по отдельности — это интерпретируемый цикл под капотом, теряющий всю векторизацию. Если групп миллион, ты делаешь миллион питоновских вызовов. Встроенные агрегаты (`sum`, `mean`, `count`, `nunique`) реализованы в C/Cython и работают на порядки быстрее.

!!! warning "Правило производительности"

    Прежде чем писать `apply`, спроси: можно ли выразить это через встроенный агрегат, `transform`, или векторную операцию? В 90% случаев — да. `apply` — последнее средство, а не первое. На больших данных разница между `agg("sum")` и `apply(lambda g: g.sum())` достигает 100x.

!!! question "Проверь себя"

    1. Расшифруй split-apply-combine на примере "средний депозит по каналам".
    2. Чем `agg` отличается от `transform` по форме результата?
    3. Почему "доля депозита игрока от суммы по его стране" делается через `transform`, а не через `agg`?
    4. Почему `apply` на большом числе групп медленный, и что пробовать перед ним?

??? success "Ответы"

    1. Split — разбить таблицу по `channel`; apply — к каждой группе применить `mean` депозита; combine — собрать результаты в таблицу "канал → средний депозит".
    2. `agg` сворачивает группу в одно число (результат короче исходной таблицы); `transform` возвращает результат той же длины, что группа, — групповая характеристика "размножается" обратно на каждую строку.
    3. Нужно вернуть сумму по стране к каждой строке этой страны (длина сохраняется); `agg` схлопнул бы таблицу до одной строки на страну.
    4. `apply` вызывает питоновскую функцию для каждой группы — интерпретируемый цикл без векторизации. Сначала ищи встроенный агрегат (`sum`, `mean`, `nunique`), `transform` или векторную операцию.

!!! example "Мини-задание"

    На маленьком `DataFrame` (`country`, `deposit`) посчитай двумя способами: (а) `groupby("country")["deposit"].sum()` через `agg` и (б) `groupby("country")["deposit"].transform("sum")`. Сравни форму (число строк) результатов и объясни словами, почему они разные. 4 минуты.
