# M6.6 · Фильтрация

### Интуиция

Фильтрация — это "оставь только строки, удовлетворяющие условию". Ты строишь маску — столбец из True/False той же длины, что и таблица, — и pandas оставляет строки, где True. Думай об этом как о включённой/выключенной лампочке напротив каждой строки.

Ключевая идея, которую новички часто пропускают: **маска и сама таблица выровнены по индексу**. Когда ты пишешь `df["amount"] >= 50`, pandas не "пробегает циклом" — он за один проход (векторизованно) сравнивает весь столбец с числом и возвращает булеву Series с тем же индексом, что у `df`. Потом `df[mask]` сопоставляет строки по индексу, а не по позиции. Поэтому маску, построенную по одному DataFrame, нельзя бездумно применять к другому: индексы должны совпадать.

На пальцах: представь таблицу депозитов. Колонка `amount` — это `[120, 30, 75, 10]`. Условие `amount >= 50` превращается в `[True, False, True, False]`. `df[mask]` оставляет первую и третью строки. Всё. Никаких циклов, никаких `if` по строкам.

### Формально

```python
mask = (df["channel"] == "facebook") & (df["amount"] >= 50)
fb = df[mask]

# несколько значений
df[df["country"].isin(["UZ", "KZ", "RU"])]

# диапазон
df[df["amount"].between(10, 1000)]

# текст
df[df["promo"].str.startswith("WELCOME")]

# современный способ — query, читается как SQL
df.query("channel == 'facebook' and amount >= 50")
```

Критично: логические операторы — это `&`, `|`, `~` (не `and`, `or`, `not`), и каждое условие обязательно в скобках из-за приоритета операций. `df[df.a > 0 & df.b > 0]` без скобок интерпретируется неверно и упадёт.

### Почему именно `&`, `|`, `~`, а не `and`, `or`, `not`

`and`/`or` в Python работают с *одним* булевым значением и пытаются привести объект к True/False. Но Series из тысячи True/False нельзя свести к одному булеву — что это, "истина" или "ложь"? Поэтому pandas бросает `ValueError: The truth value of a Series is ambiguous`. Операторы `&`, `|`, `~` — поэлементные (bitwise): они сравнивают маски *по позициям* и возвращают новую маску. Именно это нам нужно.

Про скобки: у `&` приоритет *выше*, чем у `>` и `==`. То есть `df.a > 0 & df.b > 0` Python читает как `df.a > (0 & df.b) > 0` — бессмыслица, падение. Скобки `(df.a > 0) & (df.b > 0)` форсируют правильный порядок. Правило простое: **каждое условие — в свои скобки, всегда.**

### Разбор на числах

```python
import pandas as pd

df = pd.DataFrame({
    "player_id": [1, 2, 3, 4, 5],
    "channel":   ["facebook", "google", "facebook", "facebook", "organic"],
    "country":   ["UZ", "RU", "KZ", "UZ", "DE"],
    "amount":    [120, 30, 75, 10, 200],
})

mask = (df["channel"] == "facebook") & (df["amount"] >= 50)
# channel == facebook -> [True, False, True, True, False]
# amount   >= 50      -> [True, False, True, False, True]
# &                   -> [True, False, True, False, False]

df[mask]
#    player_id   channel country  amount
# 0          1  facebook      UZ     120
# 2          3  facebook      KZ      75
```

Обрати внимание на индекс результата: остались строки `0` и `2`, индекс **не пересобирается** в `0,1`. Если он мешает — `df[mask].reset_index(drop=True)`.

### Пример из гемблинга: "качественный FTD"

Допустим, продуктовая задача — найти первые депозиты (FTD) от $50 из платных каналов, чтобы оценить долю "качественного" трафика. В долларах порог $50 выбран как граница, ниже которой LTV обычно не отбивает CAC.

```python
paid = ["facebook", "google", "tiktok"]
quality_ftd = df[
    df["channel"].isin(paid)
    & (df["amount"] >= 50)
    & df["country"].isin(["UZ", "KZ", "RU"])
]

share = len(quality_ftd) / len(df)
print(f"Доля качественных FTD: {share:.1%}")
```

Это типичный аналитический срез: канал И сумма И гео. Каждое условие — отдельная бизнес-гипотеза, а `&` собирает их в один фильтр.

### Пограничные случаи и типичные ошибки

- **Пропуски (NaN) проваливают любое сравнение.** `df["amount"] >= 50` для строки с `NaN` даёт `False`, а не ошибку, — такие строки молча выпадают из фильтра. Если NaN важны, обрабатывай отдельно: `df["amount"].fillna(0)` или `df[df["amount"].isna()]`. Отрицание тоже коварно: `~(df["amount"] >= 50)` вернёт `True` для NaN-строки, потому что `False` инвертируется в `True`.
- **`isin` против цепочки `==`.** `df[df.country == "UZ" | df.country == "KZ"]` — двойная ловушка (нет скобок + читается тяжело). Пиши `df[df["country"].isin(["UZ", "KZ"])]`.
- **`between` включает оба конца** по умолчанию (`inclusive="both"`). Если нужен полуинтервал — `between(10, 1000, inclusive="left")`.
- **`.str`-методы и NaN.** `df["promo"].str.startswith("WELCOME")` вернёт `NaN` для пропусков, а `NaN` в маске ломает индексацию. Защита: `df["promo"].str.startswith("WELCOME", na=False)` — пропуски трактуются как `False`.
- **`query` и переменные.** Чтобы подставить значение из Python внутрь `query`, используй префикс `@`: `df.query("amount >= @threshold")`. Без `@` pandas будет искать колонку с именем `threshold`.
- **Фильтр возвращает копию-вид, не модифицируй его на месте.** `df[mask]["amount"] = 0` — частый источник `SettingWithCopyWarning` и тихих багов. Для записи по условию используй `df.loc[mask, "amount"] = 0`.

### Когда что выбирать

- Простой одно-двухусловный фильтр в скрипте — булева маска, она прозрачна и отлаживается построчно.
- Много условий или читаемость для не-питониста — `query` (синтаксис как SQL, меньше скобок).
- Проверка вхождения в список значений — `isin`.
- Числовой/датовый диапазон — `between`.
