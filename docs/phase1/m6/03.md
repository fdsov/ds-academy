# M6.3 · Series и DataFrame: атомы pandas

### Интуиция

`Series` — это один столбец с подписями. Представь колонку в Excel, но у каждой ячейки есть имя (метка строки), а не только номер. `DataFrame` — это таблица: набор `Series`, склеенных по общему индексу строк. Если ты держишь в голове "Series = столбец, DataFrame = таблица из столбцов с общим индексом" — ты уже понял базу.

Ключевое отличие от Excel и от обычного списка Python: у строк есть **индекс** — отдельная ось меток. Это не просто номера 0, 1, 2. Индексом может быть дата, ID игрока, что угодно. Индекс — главная фишка pandas и одновременно главный источник путаницы для новичков.

### Формально

`Series` — это одномерный размеченный массив: пара (значения, индекс), где значения хранятся в типизированном массиве (поверх NumPy или, в новых версиях, Apache Arrow), а индекс — это отдельный объект `Index`, отображающий метки в позиции. `DataFrame` — двумерная размеченная структура: словарь из `Series` с общим объектом-индексом строк и объектом-индексом столбцов.

```python
import pandas as pd

deposits = pd.Series([100, 250, 50], index=["u1", "u2", "u3"], name="deposit")
deposits["u2"]  # доступ по метке -> 250
deposits.iloc[1]  # доступ по позиции -> 250
```

Важно сразу зафиксировать: метка и позиция — это разные вещи. `deposits["u2"]` ищет по метке, `deposits.iloc[1]` — по порядковому номеру. Когда метки — это случайно числа (0, 1, 2), новички их путают, и это порождает классические баги.

!!! note "dtype — это не мелочь"

    У каждого столбца есть тип (`int64`, `float64`, `object`, `category`, `datetime64[ns]`, `boolean`). Тип определяет и скорость, и корректность. В pandas 2.x столбец строк по умолчанию хранился как `object` — медленный питоновский тип, а Arrow был опцией; в pandas 3.0 дефолтом стал выделенный строковый dtype на PyArrow, так что строки сразу быстрее и компактнее. Для повторяющихся строк (канал, страна, статус) почти всегда лучше `category`. Бэкенд Arrow (`dtype_backend="pyarrow"`) даёт честные nullable-типы и заметно быстрее старого NumPy-бэкенда.

!!! question "Проверь себя"

    1. Чем `Series` отличается от `DataFrame` одним предложением?
    2. Что вернёт `deposits["u2"]` и что `deposits.iloc[1]` — по чему ищет каждый из них?
    3. Почему столбец строк лучше держать как `category`, а не как `object`?
    4. Что такое индекс в pandas и чем он отличается от обычной нумерации строк?

??? success "Ответы"

    1. `Series` — один размеченный столбец (значения + индекс); `DataFrame` — таблица из нескольких `Series` с общим индексом строк.
    2. `deposits["u2"]` ищет по метке, `deposits.iloc[1]` — по позиции; в данном примере оба вернут 250.
    3. `object` хранит полноценные питоновские строки (медленно и тяжело); `category` хранит словарь уникальных значений один раз, а в ячейках держит компактные целочисленные коды — меньше памяти, быстрее группировки.
    4. Индекс — отдельная ось меток строк (дата, ID игрока, что угодно), а не просто номера 0,1,2; по нему работает выравнивание и доступ через `loc`.
