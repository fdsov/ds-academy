# M5.6 · Индексация, срезы, маски

### Базовые срезы

Синтаксис как у списков, но по каждой оси через запятую. `A[i, j]` — элемент, `A[i]` — целая строка, `A[:, j]` — целый столбец.

```python
A = np.arange(12).reshape(3, 4)
A[1, 2]      # 6 — строка 1, столбец 2
A[0]         # [0 1 2 3] — первая строка
A[:, 1]      # [1 5 9] — второй столбец
A[0:2, 1:3]  # подматрица
```

Полезно держать в голове геометрию. Двумерик `A` из примера выглядит так:

```text
        col0 col1 col2 col3
row0 [   0    1    2    3 ]
row1 [   4    5    6    7 ]
row2 [   8    9   10   11 ]
```

Тогда `A[1, 2]` — пересечение строки 1 и столбца 2, то есть `6`. Запись `A[0:2, 1:3]` берёт строки `0,1` и столбцы `1,2`, давая блок `[[1 2] [5 6]]`. Двоеточие `:` без чисел означает «вся ось целиком», поэтому `A[:, 1]` — весь столбец 1. Срез работает по правилу полуинтервала `[start:stop)`: `stop` не включается, ровно как в обычных списках Python и в `range`.

Срез принимает и третий параметр — шаг: `A[::2]` берёт каждую вторую строку, `A[:, ::-1]` разворачивает столбцы. Отрицательные индексы считают с конца: `A[-1]` — последняя строка, `A[:, -1]` — последний столбец.

Важнейший факт: базовый срез возвращает **view**, а не копию. Изменение среза меняет исходный массив. Это отличается от поведения списков Python, где срез копирует.

```python
A = np.arange(12).reshape(3, 4)
row = A[1]          # view на строку 1
row[0] = 999
A[1, 0]             # 999 — оригинал изменился!
```

Почему так сделано: NumPy хранит данные одним непрерывным блоком в памяти, а view — это просто другой «взгляд» на тот же блок (другие offset и shape, те же байты). Копировать мегабайты при каждом срезе было бы расточительно. Если копия нужна явно — пиши `A[1].copy()`.

!!! warning "Типичная ошибка"

    Аналитик берёт срез данных, чтобы «поэкспериментировать», правит его — и портит исходный массив, не заметив этого. Если меняешь срез и хочешь сохранить оригинал нетронутым, всегда делай `.copy()`. Особенно коварно это в коде вида `subset = data[:100]; subset *= 2` — оригинальные первые 100 строк тоже удвоятся.

### Boolean-маски — главный инструмент аналитика

#### Интуиция

Маска — это массив из True/False той же длины, который говорит «этот элемент берём, этот нет». Сравнение массива с числом даёт сразу всю маску, без цикла. Это твой основной способ фильтровать данные: «все депозиты больше 10000», «все игроки с retention выше медианы».

Представь, что у тебя список депозитов и рядом — список «галочек», по одной на каждый депозит. Где галочка стоит (True) — депозит проходит фильтр, где не стоит (False) — отбрасывается. Маска и есть этот список галочек, только NumPy строит его за тебя одним сравнением и применяет векторизованно — без `for`, на скорости C.

#### Формально

Выражение `arr > 100` применяет оператор поэлементно и возвращает массив `dtype=bool` той же формы. Индексация массивом-маской `arr[mask]` возвращает одномерный массив из элементов, где маска True (всегда копия, не view). Условия комбинируются поэлементными операторами `&` (и), `|` (или), `~` (не) — обязательно в скобках, потому что приоритет операторов в Python отдаёт `&` ниже, чем сравнения.

```python
deposits = np.array([50, 15000, 300, 99000, 20])

big = deposits > 10_000           # [False True False True False]
deposits[big]                     # [15000 99000]

mask = (deposits > 100) & (deposits < 50_000)
deposits[mask]                    # [15000 300]

# подсчёт и доля без цикла
big.sum()                         # 2 — сколько крупных депозитов
big.mean()                        # 0.4 — их доля
```

Последняя строка — частый идиоматический приём: для boolean-массива `.mean()` даёт долю True, потому что True считается за 1, False за 0. «Какая доля игроков сделала повторный депозит» — это одна строка.

Разберём, почему скобки критичны. Возьмём `deposits > 100 & deposits < 50_000` без скобок. У `&` приоритет выше, чем у `>` и `<`, поэтому Python сначала вычислит `100 & deposits` (побитовое И двух массивов чисел — бессмыслица в этом контексте), а затем попытается сравнить результат — и в лучшем случае выдаст `ValueError`, в худшем тихо посчитает не то. Правильно: каждое сравнение в своих скобках — `(deposits > 100) & (deposits < 50_000)`.

И ещё одна частая ошибка — использовать `and`/`or` вместо `&`/`|`. Ключевые слова `and`/`or` работают с одним булевым значением, а не с массивом галочек, поэтому на массиве из >1 элемента они кидают `ValueError: The truth value of an array ... is ambiguous`. Для поэлементной логики всегда `&`, `|`, `~`.

##### Числовой пример: фильтрация когорты

Допустим, есть депозиты и FTD-флаг (первый ли это депозит игрока). Хотим долю и сумму крупных первых депозитов:

```python
deposits = np.array([50, 15000, 300, 99000, 20, 25000])
is_ftd   = np.array([1, 1, 0, 1, 0, 0], dtype=bool)

big_ftd = (deposits > 10_000) & is_ftd   # [F T F T F F]
big_ftd.mean()                            # 0.333 — доля крупных FTD
deposits[big_ftd].sum()                   # 114000 — их суммарный объём ($)
```

Здесь маску можно собирать из булева столбца `is_ftd` напрямую — он уже `bool`, оборачивать его в сравнение не нужно. Комбинируем числовое условие с готовым флагом через `&`. Так в одну строку получается ответ на вопрос «сколько денег принесли крупные первые депозиты», который без NumPy потребовал бы цикла с накоплением.

#### Присваивание по маске

Маску можно использовать слева от `=` — это меняет только выбранные элементы. Классический приём — клиппинг выбросов или замена аномалий:

```python
x = np.array([5, -3, 200, 8, -1])
x[x < 0] = 0          # обнулить отрицательные → [5 0 200 8 0]
x[x > 100] = 100      # обрезать сверху → [5 0 100 8 0]
```

В аналитике гемблинга так чистят, например, явно ошибочные ставки: `bets[bets > balance_cap] = balance_cap`.

### Fancy indexing

Индексация массивом целочисленных индексов выбирает элементы в произвольном порядке и с повторами. Возвращает копию.

```python
prices = np.array([10, 20, 30, 40, 50])
idx = np.array([0, 2, 2, 4])
prices[idx]                       # [10 30 30 50]

# в двух измерениях — пары координат
A = np.arange(12).reshape(3, 4)
A[[0, 1, 2], [1, 2, 3]]           # [1 6 11] — элементы (0,1),(1,2),(2,3)
```

Обрати внимание на двумерный случай: списки индексов **спариваются поэлементно**, а не образуют прямоугольник. `A[[0,1,2], [1,2,3]]` берёт точки `(0,1)`, `(1,2)`, `(2,3)`, а не всю подматрицу строк `0,1,2` и столбцов `1,2,3`. Если нужен именно прямоугольный блок — это уже срез `A[0:3, 1:4]` или приём с `np.ix_`.

Повторы в индексе разрешены (`idx = [0, 2, 2, 4]` дважды берёт элемент 2) — этим fancy indexing отличается от boolean-маски, где каждый элемент попадает в результат максимум один раз. Поэтому fancy indexing удобен для пересборки/перестановки: например, перемешать выборку по случайному порядку `data[np.random.permutation(len(data))]` или применить заранее посчитанный порядок сортировки `data[np.argsort(scores)]`.

#### Когда что выбирать

| Инструмент | Что даёт | View или копия |
|---|---|---|
| Базовый срез `A[1:3, :2]` | непрерывный блок | **view** |
| Boolean-маска `A[A > 0]` | элементы по условию | копия |
| Fancy indexing `A[[0,2,2]]` | элементы по списку индексов, с повторами/порядком | копия |

Правило памяти: только **базовый срез** возвращает view; маска и fancy indexing — всегда копия, потому что результат не описывается одним непрерывным куском памяти.

!!! question "Проверь себя"

    1. Базовый срез массива возвращает view или копию? Чем это отличается от срезов списков Python?
    2. Что вернёт `mask.mean()` для boolean-маски и почему?
    3. Почему условия в маске нужно оборачивать в скобки: `(a > 1) & (b < 2)`?
    4. Fancy indexing и boolean-маска возвращают view или копию?

??? success "Ответы"

    1. View — окно в ту же память; изменение среза меняет оригинал. Списки Python при срезе копируют.
    2. Долю элементов True: True считается за 1, False за 0, поэтому среднее равно доле True.
    3. У оператора `&` приоритет выше, чем у сравнений: без скобок `a > 1 & b < 2` распарсится как `a > (1 & b) < 2`.
    4. Копию (оба варианта).

!!! example "Мини-задание"

    Дан `deposits = np.array([50, 15000, 300, 99000, 20])`. Через одну boolean-маску посчитай долю депозитов больше 10000, их количество и сумму — без цикла. Проверь, что `(deposits > 10_000).mean()` даёт ту же долю, что и `.sum() / len`.
