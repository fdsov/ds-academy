# M5.6 · Индексация, срезы, маски

### Базовые срезы

Синтаксис как у списков, но по каждой оси через запятую. `A[i, j]` — элемент, `A[i]` — целая строка, `A[:, j]` — целый столбец.

```python
A = np.arange(12).reshape(3, 4)
A[1, 2]      # 6 — строка 1, столбец 2
A[0]         # [0 1 2 3] — первая строка
A[:, 1]      # [1 5 9] — второй столбец
A[0:2, 1:3]  # подматрица
```

Важнейший факт: базовый срез возвращает **view**, а не копию. Изменение среза меняет исходный массив. Это отличается от поведения списков Python, где срез копирует.

### Boolean-маски — главный инструмент аналитика

#### Интуиция

Маска — это массив из True/False той же длины, который говорит «этот элемент берём, этот нет». Сравнение массива с числом даёт сразу всю маску, без цикла. Это твой основной способ фильтровать данные: «все депозиты больше 10000», «все игроки с retention выше медианы».

#### Формально

Выражение `arr > 100` применяет оператор поэлементно и возвращает массив `dtype=bool` той же формы. Индексация массивом-маской `arr[mask]` возвращает одномерный массив из элементов, где маска True (всегда копия, не view). Условия комбинируются поэлементными операторами `&` (и), `|` (или), `~` (не) — обязательно в скобках, потому что приоритет операторов в Python отдаёт `&` ниже, чем сравнения.

```python
deposits = np.array([50, 15000, 300, 99000, 20])

big = deposits > 10_000           # [False True False True False]
deposits[big]                     # [15000 99000]

mask = (deposits > 100) & (deposits < 50_000)
deposits[mask]                    # [15000 300]

# подсчёт и доля без цикла
big.sum()                         # 2 — сколько крупных депозитов
big.mean()                        # 0.4 — их доля
```

Последняя строка — частый идиоматический приём: для boolean-массива `.mean()` даёт долю True, потому что True считается за 1, False за 0. «Какая доля игроков сделала повторный депозит» — это одна строка.

### Fancy indexing

Индексация массивом целочисленных индексов выбирает элементы в произвольном порядке и с повторами. Возвращает копию.

```python
prices = np.array([10, 20, 30, 40, 50])
idx = np.array([0, 2, 2, 4])
prices[idx]                       # [10 30 30 50]

# в двух измерениях — пары координат
A = np.arange(12).reshape(3, 4)
A[[0, 1, 2], [1, 2, 3]]           # [1 6 11] — элементы (0,1),(1,2),(2,3)
```

!!! question "Проверь себя"

    1. Базовый срез массива возвращает view или копию? Чем это отличается от срезов списков Python?
    2. Что вернёт `mask.mean()` для boolean-маски и почему?
    3. Почему условия в маске нужно оборачивать в скобки: `(a > 1) & (b < 2)`?
    4. Fancy indexing и boolean-маска возвращают view или копию?

??? success "Ответы"

    1. View — окно в ту же память; изменение среза меняет оригинал. Списки Python при срезе копируют.
    2. Долю элементов True: True считается за 1, False за 0, поэтому среднее равно доле True.
    3. У оператора `&` приоритет выше, чем у сравнений: без скобок `a > 1 & b < 2` распарсится как `a > (1 & b) < 2`.
    4. Копию (оба варианта).

!!! example "Мини-задание"

    Дан `deposits = np.array([50, 15000, 300, 99000, 20])`. Через одну boolean-маску посчитай долю депозитов больше 10000, их количество и сумму — без цикла. Проверь, что `(deposits > 10_000).mean()` даёт ту же долю, что и `.sum() / len`.
