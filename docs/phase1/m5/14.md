# M5.14 · Работа с NaN

### Интуиция

NaN (Not a Number) — это специальное значение float, означающее «пропуск/неопределённость». В реальных данных оно вездесуще: игрок не сделал депозит, метрика не посчиталась, джойн не нашёл пару. NaN «заразен»: любая арифметика с ним даёт NaN, и обычные `sum`/`mean` тоже вернут NaN, если в данных есть хоть один пропуск.

Полезная аналогия «на пальцах»: NaN — это не ноль и не пустота, а честное «я не знаю». И это правильно, что он заразен. Если ты не знаешь депозит одного игрока из ста, то и точную сумму депозитов всех ста ты тоже не знаешь — знаешь только сумму известных. NaN заставляет тебя явно решить, что делать с незнанием, а не молча подставить ноль. Подмена NaN нулём — самая частая и самая дорогая ошибка новичка: ноль означает «точно знаю, что было нуль рублей», а это совсем другое утверждение.

### Формально

NaN определён стандартом IEEE 754 и обладает странным свойством: он не равен сам себе. Поэтому `nan == nan` — это False, и проверять надо через `np.isnan`.

```python
x = np.array([10.0, np.nan, 30.0, np.nan, 50.0])

x.mean()                # nan — заражено
np.isnan(x)             # [False True False True False]
x[~np.isnan(x)]         # [10. 30. 50.] — отфильтровать пропуски

np.nanmean(x)           # 30.0 — среднее, игнорируя NaN
np.nansum(x)            # 90.0
np.nanstd(x), np.nanmax(x)

# заполнить пропуски значением
np.where(np.isnan(x), 0.0, x)   # [10. 0. 30. 0. 50.]
```

Свойство «не равен сам себе» — не курьёз, а рабочий инструмент. Из него следуют два практичных приёма:

```python
# 1. посчитать число пропусков без isnan
np.isnan(x).sum()       # 2

# 2. найти пропуски «самопроверкой» x != x — тоже True только для NaN
mask = x != x           # [False True False True False]
```

Существует семейство `nan`-агрегаций: `nanmean`, `nansum`, `nanstd`, `nanmax`, `nanmin`, `nanmedian`. Используй их, когда пропуски ожидаемы и их нужно игнорировать, но всегда осознанно — иногда NaN означает «надо разобраться, а не замолчать».

### Численный пример: депозиты и заразность среднего

Разберём типичную аналитическую ловушку. Пусть у нас депозиты пяти игроков, у двоих депозит не зафиксирован (NaN):

```python
dep = np.array([100.0, np.nan, 300.0, np.nan, 50.0])

dep.mean()              # nan — обычное среднее сломалось
np.nanmean(dep)         # 150.0  = (100 + 300 + 50) / 3

# а если бы мы по ошибке подставили нули:
filled = np.where(np.isnan(dep), 0.0, dep)
filled.mean()           # 90.0   = (100 + 0 + 300 + 0 + 50) / 5
```

Разница огромна: `nanmean` делит на 3 (число известных значений), а замена нулём делит на 5. Средний депозит «150 рублей на активного» против «90 рублей на игрока» — это два разных бизнес-ответа. Правильный выбор зависит от вопроса: если считаешь ARPPU (средний доход на платящего), пропуск — это «не платил», и его надо исключать (`nanmean`). Если считаешь ARPU на всю когорту, и NaN означает «точно не вносил», то ноль уместен. Всегда формулируй, что именно означает пропуск в твоей задаче.

### Пограничные случаи и типичные ошибки

`np.nanmean` пустого среза (все элементы NaN) вернёт NaN и предупреждение `RuntimeWarning: Mean of empty slice` — это не ошибка, а сигнал «нечего усреднять». Типичный кейс в гемблинге: считаешь средний чек по сегменту, а в сегменте за этот день не было ни одного платежа.

```python
allnan = np.array([np.nan, np.nan])
np.nanmean(allnan)      # nan + RuntimeWarning
np.nansum(allnan)       # 0.0 — внимание: сумма пустого = 0, а не NaN
```

Ещё одна частая ошибка — путать NaN с `None` и с бесконечностью:

- `None` — это объект Python, он превращает массив в `dtype=object` и ломает векторные операции. NaN — настоящий float и работает быстро.
- `np.inf` (бесконечность) — это не NaN. `np.isnan(np.inf)` даёт False. Чтобы поймать и то, и другое разом, используй `np.isfinite`: оно False и для NaN, и для `±inf`.

```python
y = np.array([1.0, np.nan, np.inf, -np.inf, 5.0])
np.isnan(y)             # [False  True False False False]
np.isfinite(y)          # [ True False False False  True] — ловит и inf, и nan
```

Бесконечности тоже заразны и часто рождаются из деления на ноль (`bet / 0`), поэтому в продакшен-расчётах метрик фильтруй через `isfinite`, а не только `isnan`.

!!! note "NaN только во float"

    NaN существует только в типах с плавающей точкой. У целочисленных массивов NaN нет. Поэтому если в колонке возможны пропуски, она будет float (или object). pandas в M9 решает это через nullable-типы и `pd.NA`, но базовый NumPy NaN живёт лишь во float.

!!! warning "Не замалчивай пропуски по умолчанию"

    `nanmean` и компания удобны, но опасны как привычка: они тихо прячут проблему. Прежде чем игнорировать NaN, спроси себя — это ожидаемый пропуск (игрок не платил) или баг пайплайна (джойн потерял строки)? Во втором случае правильный ход — найти причину, а не усреднить вокруг дыры. Хорошая практика: логировать долю пропусков `np.isnan(x).mean()` и поднимать тревогу, если она вдруг выросла.
