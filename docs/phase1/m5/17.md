# M5.17 · Практика

### Быстрый квиз

Выбери вариант — ответ подсветится сразу.

```quiz
Почему NumPy-массив обрабатывается быстрее, чем список Python той же длины?
[ ] NumPy автоматически распараллеливает любой расчёт на все ядра процессора
[x] Числа лежат подряд плотной лентой с единым dtype, и цикл идёт на C без проверки типов
[ ] NumPy кэширует результаты предыдущих операций
[ ] Список Python всегда хранит данные на диске, а массив — в оперативной памяти
> В массиве элементы лежат непрерывным блоком с фиксированным dtype, поэтому операция превращается в тугой цикл на C (с SIMD), без создания объектов и проверки типов на каждой итерации.
---
Какие три метаданных вместе со ссылкой на блок памяти описывают ndarray?
[ ] size, ndim, nbytes
[x] dtype, shape, strides
[ ] index, columns, values
[ ] offset, length, capacity
> dtype задаёт тип элемента, shape — размеры по осям, strides — на сколько байт шагнуть вдоль каждой оси. Эта тройка позволяет интерпретировать одну ленту байтов как массив любой формы.
---
Почему float32 опасен для финансового свода, но приемлем для матрицы признаков в ML?
[ ] float32 не поддерживает отрицательные числа
[ ] float32 медленнее, чем float64, и поэтому копит ошибку
[x] float32 даёт лишь ~7 значащих цифр, и при суммировании миллионов сумм копится ошибка округления, а в ML важнее память и скорость GPU
[ ] float32 округляет все значения до целых
> float32 экономит память вдвое ценой точности (~7 цифр против ~15-16 у float64). Для финансовых сумм это накапливает ошибку, для матриц признаков в ML точность не критична, а память и скорость важны.
---
Что вернёт mask.mean() для boolean-маски и почему?
[x] Долю элементов True, потому что True считается за 1, а False за 0
[ ] Среднее значение исходного массива по элементам, где маска True
[ ] Всегда 0.5, так как маска состоит из True и False
[ ] Ошибку, потому что среднее не определено для bool
> Для boolean-массива True трактуется как 1, False как 0, поэтому среднее равно доле True. Это идиоматичный способ посчитать долю, например, повторных депозитов одной строкой.
---
Базовый срез NumPy-массива (например a[2:5]) возвращает view или копию?
[ ] Копию, как и срез списка Python
[x] View — окно в ту же память, изменение среза меняет оригинал
[ ] Зависит от dtype массива
[ ] View только для одномерных массивов, иначе копию
> Базовый срез возвращает view: новый объект-массив поверх той же ленты байтов. Изменение элементов среза меняет исходный массив — в отличие от среза списка Python, который копирует. Для независимости нужен .copy().
---
Зачем нужен keepdims=True при вычитании среднего из каждой строки матрицы?
[ ] Чтобы среднее считалось по столбцам, а не по строкам
[ ] Чтобы ускорить расчёт за счёт кэширования оси
[x] Чтобы сохранить схлопнутую ось как размер 1 и broadcasting сработал в нужном направлении
[ ] Чтобы результат остался целочисленным
> X.mean(axis=1, keepdims=True) даёт форму (n, 1), которая растягивается по столбцам. Без него результат (n,) выровняется справа к столбцам и посчитает не то.
---
Почему np.random.default_rng предпочтительнее старого np.random.seed + np.random.rand?
[ ] default_rng генерирует числа криптографически стойко
[ ] Старый интерфейс удалён в NumPy 2.x и больше не работает
[x] У default_rng нет глобального состояния: каждый генератор изолирован, нет скрытых зависимостей и надёжнее воспроизводимость
[ ] default_rng работает только с целыми числами
> Старый seed — это глобальное состояние, создающее скрытые зависимости между частями кода и проблемы в параллельных процессах. default_rng создаёт изолированный Generator (PCG64), каждый независим.
```

### Разминка

1. Создай массив чисел от 0 до 99 через `np.arange`, преобразуй в матрицу `(10, 10)`. Выведи: третью строку, последний столбец, центральный блок `4x4`. Замени `reshape` так, чтобы NumPy сам посчитал одну из осей через `-1`.
2. Дан массив депозитов `dep = rng.exponential(200, 1000)`. Найди долю депозитов больше 500, их количество и сумму. Сделай всё через boolean-маску, без цикла.
3. Сравни скорость суммы квадратов через Python-цикл и через `np.sum(arr ** 2)` для миллиона элементов. Измерь `time.perf_counter`, посчитай ускорение.

### Основное

4. Матрица `bets` формы `(500 игроков, 30 дней)`. Посчитай: суммарные ставки по дням (`axis`), средние по игрокам, день с максимальной активностью для каждого игрока (`argmax`). Нормируй матрицу z-score по столбцам через broadcasting.
5. В массиве метрик есть NaN (вставь искусственно ~5% пропусков через маску и присваивание `np.nan`). Посчитай среднее и стандартное отклонение, игнорируя пропуски, затем заполни пропуски медианой через `np.where`.
6. Реализуй векторную сегментацию игроков по LTV: `np.where`-каскадом или через `np.digitize` разбей на «whale / regular / churned» по двум порогам. Посчитай размер каждого сегмента.

### Челлендж

7. Напиши две версии попарных евклидовых расстояний между строками матрицы `X` формы `(n, d)`: наивную с двойным циклом и векторизованную через broadcasting (формула $\lVert a - b \rVert^2 = \lVert a \rVert^2 + \lVert b \rVert^2 - 2 a \cdot b$). Сравни скорость и проверь, что результаты совпадают (`np.allclose`). Оцени, при каком `n` векторная версия упрётся в память.

### Мини-проект: антифрод-скоринг на чистом NumPy

Сгенерируй синтетический лог из 2 млн ставок: `player_id`, `bet`, `win`, `timestamp`. Не используя pandas, только NumPy:

- посчитай GGR по каждому игроку через `np.bincount` с весами;
- построй матрицу признаков на игрока: суммарный объём, число ставок, средняя ставка, доля выигрышных, максимальная ставка;
- нормируй признаки z-score по столбцам (broadcasting);
- посчитай простой аномалийный скор как евклидову норму z-вектора каждого игрока (`np.linalg.norm` по `axis=1`);
- выведи топ-20 «подозрительных» игроков и обоснуй, почему этот скор может давать ложные срабатывания на хайроллерах (vip), которые легитимны.

Цель — весь пайплайн без единого Python-цикла по данным и с осознанным выбором dtype.
