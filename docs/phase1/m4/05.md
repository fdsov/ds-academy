# M4.5 · Агрегаты, GROUP BY, HAVING — свёртка данных

Интуиция. Агрегат — это когда из множества строк получается одно число. "Сколько всего депозитов" — COUNT. "На какую сумму" — SUM. GROUP BY — "посчитай это отдельно для каждой страны / каждого дня / каждого игрока".

Формально. Пять базовых агрегатных функций: `COUNT` (количество), `SUM` (сумма), `AVG` (среднее), `MIN` / `MAX` (минимум / максимум). С `GROUP BY` они считаются по группам:

```sql
SELECT country,
       COUNT(*)              AS deposits_cnt,
       COUNT(DISTINCT player_id) AS depositors,
       SUM(amount)           AS total_amount,
       AVG(amount)           AS avg_check
FROM deposits
GROUP BY country
HAVING SUM(amount) > 10000
ORDER BY total_amount DESC;
```

Тонкости, которые многие не знают:

- `COUNT(*)` считает все строки. `COUNT(column)` считает только строки где `column` не NULL. `COUNT(DISTINCT player_id)` — число уникальных игроков. Это три разных числа.
- `HAVING` — это `WHERE` для групп. `WHERE` фильтрует строки до группировки, `HAVING` — группы после. Хочешь "страны с суммой депозитов больше 10000" — это HAVING. Хочешь "только депозиты больше 100, потом сгруппировать" — это WHERE.
- `AVG` игнорирует NULL. Среднее по `[100, NULL, 200]` равно 150, а не 100. Это часто не то, что ты думаешь.

!!! question "Проверь себя"

    1. Чем отличаются `COUNT(*)`, `COUNT(column)` и `COUNT(DISTINCT player_id)`?
    2. В чём разница между `WHERE` и `HAVING` и почему "страны с суммой депозитов больше 10000" — это `HAVING`?
    3. Чему равно `AVG` по значениям `[100, NULL, 200]` и почему?

??? success "Ответы"

    1. `COUNT(*)` считает все строки; `COUNT(column)` — только строки, где `column` не NULL; `COUNT(DISTINCT player_id)` — число уникальных значений.
    2. `WHERE` фильтрует строки до группировки, `HAVING` — группы после агрегации. Сумма по стране известна только после `GROUP BY`, поэтому условие на неё ставится в `HAVING`.
    3. 150: `AVG` игнорирует NULL и делит сумму 300 на 2 непустых значения, а не на 3.
