# M4.1 · Зачем этот модуль

Данные в реальной компании не лежат в CSV-файле у тебя на столе. Они живут в базе данных: депозиты игроков, ставки, события входа в игру, история бонусов, выплаты. Эти таблицы измеряются десятками и сотнями миллионов строк. Питон с pandas не вытянет 200 миллионов ставок в память твоего ноутбука, а если и вытянет — ты будешь ждать минут двадцать вместо двух секунд.

SQL (Structured Query Language) — это язык, на котором ты разговариваешь с базой и просишь её посчитать ровно то, что нужно, прямо там, где лежат данные. База оптимизирована под это десятилетиями. Аналитик, который не умеет в SQL, — это переводчик без словаря: он зависит от других людей, которые "выгрузят ему данные". А значит, он не может проверить гипотезу за пять минут, не может построить когорту retention без чужой помощи, не может ответить на вопрос продакта "сколько у нас вчера было FTD из Узбекистана" немедленно.

Без SQL ломается вся работа data scientist'а на ранних этапах: сбор данных, EDA, построение признаков для модели, проверка качества данных, ad-hoc аналитика для бизнеса. Это базовый навык, такой же фундаментальный, как умение читать. И парадокс: SQL легко начать (за час напишешь первый SELECT), но по-настоящему глубоко им владеют единицы — оконные функции, планы запросов и логику NULL понимают плохо даже многие senior-аналитики.
