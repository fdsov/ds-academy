# M4.4 · SELECT, WHERE, ORDER BY, LIMIT — чтение данных

Интуиция. SELECT — это "покажи мне". WHERE — "только те строки, которые подходят под условие". ORDER BY — "отсортируй". LIMIT — "и хватит первых N штук".

Формально. Базовый запрос на выборку:

```sql
SELECT player_id, amount, deposit_ts
FROM deposits
WHERE amount >= 100
ORDER BY deposit_ts DESC
LIMIT 10;
```

Это значит: из таблицы `deposits` взять столбцы `player_id, amount, deposit_ts`, оставить только строки где сумма не меньше 100, отсортировать по времени от новых к старым (`DESC` — descending), показать первые 10.

Критично понимать порядок выполнения, потому что он не совпадает с порядком написания. SQL читается сверху вниз, но выполняется так:

1. `FROM` — берём источник
2. `WHERE` — фильтруем строки
3. `GROUP BY` — группируем
4. `HAVING` — фильтруем группы
5. `SELECT` — вычисляем столбцы и алиасы
6. `ORDER BY` — сортируем
7. `LIMIT` — отрезаем

Именно поэтому в `WHERE` нельзя сослаться на алиас из `SELECT` — на момент фильтрации строк его ещё не существует. Это первая вещь, которая отличает понимающего от зубрилы.

!!! question "Проверь себя"

    1. В каком порядке выполняются `FROM`, `WHERE`, `SELECT`, `ORDER BY`, `LIMIT`?
    2. Почему в `WHERE` нельзя сослаться на алиас, заданный в `SELECT`?
    3. Что делает `DESC` в `ORDER BY` и для чего нужен `LIMIT`?

??? success "Ответы"

    1. `FROM` → `WHERE` → `GROUP BY` → `HAVING` → `SELECT` → `ORDER BY` → `LIMIT`.
    2. На момент фильтрации строк в `WHERE` шаг `SELECT` ещё не выполнялся, поэтому алиаса физически не существует.
    3. `DESC` сортирует по убыванию (от больших к меньшим); `LIMIT N` оставляет только первые N строк результата.

!!! example "Мини-задание"

    Возьми запрос из примера выше. Не запуская его, выпиши на бумаге порядок шагов выполнения для этого конкретного запроса и отметь, на каком шаге отсекаются строки с `amount < 100`, а на каком остаются только 10 строк. Затем проверь себя по списку порядка выполнения.
