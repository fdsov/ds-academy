```python
import os, random
import numpy as np
import torch

def set_global_seed(seed: int = 42) -> None:
    os.environ["PYTHONHASHSEED"] = str(seed)
    random.seed(seed)
    np.random.seed(seed)
    torch.manual_seed(seed)
    torch.cuda.manual_seed_all(seed)
    # детерминизм важнее скорости при воспроизведении статьи
    torch.backends.cudnn.deterministic = True
    torch.backends.cudnn.benchmark = False

set_global_seed(42)
```

Корректная оценка с несколькими сидами и доверительным интервалом — то, чего не хватает в статьях, которые ты критикуешь.

```python
import numpy as np
from scipy import stats

def report_metric(scores: list[float], confidence: float = 0.95) -> str:
    arr = np.asarray(scores, dtype=float)
    mean = arr.mean()
    sem = stats.sem(arr)
    h = sem * stats.t.ppf((1 + confidence) / 2.0, df=len(arr) - 1)
    return f"{mean:.4f} ± {h:.4f} (95% CI, n={len(arr)})"

# результаты по 5 независимым сидам, а не один прогон
auc_scores = [0.812, 0.799, 0.821, 0.805, 0.817]
print(report_metric(auc_scores))  # 0.8108 ± 0.0107 (95% CI, n=5)
```

Защита от самой частой утечки — препроцессинг внутри pipeline, обучаемый только на train.

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import StratifiedKFold, cross_val_score

# StandardScaler внутри Pipeline => fit только на train-фолде,
# а не на всех данных. Это закрывает утечку через нормализацию.
pipe = Pipeline([
    ("scaler", StandardScaler()),
    ("clf", LogisticRegression(max_iter=1000)),
])

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(pipe, X, y, cv=cv, scoring="roc_auc")
print(report_metric(list(scores)))
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM в 2026 (Claude, Cursor, Copilot, специализированные research-ассистенты) полезны для чтения статей на трёх задачах: триаж ("дай 5 C по этому абстракту, стоит ли второй проход"), объяснение незнакомого куска method, и черновик related work для своего текста.

    Пример промпта для критического разбора:

    "Вот метод и эксперименты из статьи [текст]. Сыграй роль скептичного рецензента ICML. Найди: слабые бейзлайны, признаки cherry-picking, возможные утечки данных, отсутствие мер разброса. По каждому пункту — конкретная цитата из текста и почему это проблема. Не хвали."

    Где врут и что перепроверять обязательно: LLM галлюцинирует цифры, цитаты и несуществующие ссылки (особенно в bibtex — проверяй каждую по Google Scholar). Модель уверенно пересказывает вывод, которого в статье нет, и сглаживает оговорки авторов. Никогда не доверяй её оценке "значимо / незначимо" — это считается из данных, а не из текста. Любое заявленное число сверяй с таблицей в оригинале сам.

!!! note "Стек 2026"

    Актуально: arXiv + Papers with Code + OpenReview как тройка для поиска и проверки воспроизводимости; Semantic Scholar и Connected Papers для навигации по графу цитирований; Zotero + Obsidian для заметок; reproducibility checklists на топ-конференциях стали нормой; LLM-ассистенты для триажа потока.

    Устарело относительно старых курсов: ставка на журналы как главный канал (в ML это конференции); линейное чтение статьи целиком; доверие к единичному прогону без разброса; "взял из туториала, не читая первоисточник"; ручной поиск кода вместо Papers with Code; представление, что препринт = проверенная работа.

!!! warning "Ловушки"

    - Читать статью линейно от корки до корки. Время уходит, отбраковка не происходит. Используй три прохода.
    - Принимать число из abstract на веру без чтения раздела experiments и проверки бейзлайнов.
    - Верить SOTA-таблице без мер разброса. Улучшение в пределах шума сидов — не улучшение.
    - Игнорировать раздел Limitations или, хуже, не замечать его отсутствия.
    - Сеньорская ловушка: воспроизводить, не зафиксировав версии библиотек, и винить себя, когда числа не сошлись из-за смены версии фреймворка.
    - Сеньорская ловушка: утечка через препроцессинг (scaler/feature selection до сплита). На синтетике незаметна, в проде модель разваливается.
    - Доверять цитатам и цифрам, которые сгенерировал LLM, без сверки с первоисточником.
    - Гнаться за каждой свежей статьёй. Это путь к выгоранию без понимания. Глубоко — немногое, бегло — остальное.
