# M26.12 · Практика

### Быстрый квиз

Выбери вариант — ответ подсветится сразу.

```quiz
Почему истинный риск $R(h)$ нельзя посчитать напрямую?
[ ] Потому что функция потерь $\ell$ неизвестна
[x] Потому что распределение данных $\mathcal{D}$ неизвестно
[ ] Потому что выборка $S$ слишком мала
[ ] Потому что гипотеза $h$ не задана явно
> $R(h)$ — это матожидание потерь по всему распределению $\mathcal{D}$, а оно неизвестно; поэтому его заменяют средним по выборке (эмпирическим риском).
---
В чём главная опасность чистого принципа ERM?
[ ] Он не минимизирует ошибку на обучающей выборке
[ ] Он требует знания истинного распределения
[x] Низкий эмпирический риск можно получить запоминанием и он не обязан переноситься на истинный риск
[ ] Он применим только к линейным моделям
> ERM минимизирует ошибку на выборке, но её можно довести до нуля простым запоминанием ответов — это и есть проблема обобщения.
---
По правилу $O(1/\sqrt{n})$, во сколько раз нужно увеличить выборку, чтобы вдвое уменьшить член $\varepsilon$?
[ ] В 2 раза
[x] В 4 раза
[ ] В 8 раз
[ ] В 16 раз
> Так как $\varepsilon \propto 1/\sqrt{n}$, уменьшение $\varepsilon$ вдвое требует роста $\sqrt{n}$ вдвое, то есть $n$ в 4 раза.
---
Какое из трёх слагаемых ошибки нельзя уменьшить никакой моделью?
[ ] Bias²
[ ] Variance
[x] Неустранимый шум $\sigma^2$
[ ] Generalization gap
> $\sigma^2$ заложен в самих данных ($y = f(x) + \epsilon$), не зависит от модели и задаёт нижнюю границу ошибки.
---
Чему равна VC-размерность линейного классификатора в $\mathbb{R}^p$?
[ ] $p$
[x] $p + 1$
[ ] $2^p$
[ ] $p - 1$
> Линейный классификатор в $\mathbb{R}^p$ имеет $\text{VC} = p + 1$: прямая на плоскости ($p=2$) дробит 3 точки, но не 4.
---
Что утверждает теорема No Free Lunch при усреднении по всем целевым функциям?
[ ] Существует один универсально лучший алгоритм
[ ] Сложные модели всегда обобщают лучше простых
[x] Усреднённо по всем задачам качество любых двух алгоритмов одинаково
[ ] Кросс-валидация заменяет доменное знание
> Усреднённо по всем целевым функциям (при равномерном распределении) ожидаемое качество любых алгоритмов равно — превосходство всегда оплачено индуктивным смещением под конкретный класс задач.
---
Почему перепараметризованная сеть ($p \gg n$) с SGD хорошо обобщает, несмотря на double descent?
[ ] Потому что её VC-размерность равна числу параметров
[ ] Потому что она недообучается на трейне
[x] SGD неявно выбирает интерполирующее решение с наименьшей нормой — самое гладкое
[ ] Потому что шум $\sigma^2$ исчезает при $p \gg n$
> Среди бесконечного множества интерполирующих решений SGD тяготеет к решению минимальной нормы; эффективная ёмкость определяется нормой/гладкостью, а не подсчётом весов.
```

Разминка.

1. Для класса "пороги на прямой" $h(x) = \mathbf{1}[x > \theta]$ докажи, что VC-размерность равна 1: покажи, что одну точку можно раздробить, а две — нет.
2. Выведи, во сколько раз нужно увеличить выборку, чтобы вдвое уменьшить член $\varepsilon$ в оценке для конечного класса гипотез. Объясни, откуда берётся ответ "вчетверо".

Основное.

3. Возьми датасет оттока (любой публичный churn-датасет или синтетику по образцу из кода выше). Построй `validation_curve` для глубины дерева и для коэффициента $L_2$-регуляризации логистической регрессии. Найди оптимум по валидации, объясни форму кривых в терминах bias-variance.
4. Реализуй bootstrap-оценку смещения и разброса (как в коде) для трёх моделей разной сложности на одной задаче. Построй таблицу bias², variance, их сумма против реальной тестовой ошибки. Прокомментируй расхождение с неустранимым шумом.
5. Эмпирически воспроизведи проклятие размерности: для размерностей $d = 2, 10, 50, 200$ сгенерируй равномерные точки и измерь отношение (max − min) расстояний к min. Покажи, что оно стремится к нулю.

Челлендж.

6. Воспроизведи double descent на простой полносвязной сети (PyTorch): зафиксируй маленький датасет, плавно увеличивай ширину скрытого слоя через точку интерполяции, строй тестовую ошибку. Найди пик у $p \approx n$ и второй спуск при $p \gg n$. Объясни, почему дно второго спуска может быть ниже первого минимума.

Мини-проект. Возьми реальную задачу из своей предметной области (прогноз оттока, LTV или антифрод).
