Эмпирическая иллюстрация bias-variance и кривой обобщения. Стек: Python 3.12, scikit-learn 1.6, numpy 2.x.

```python
import numpy as np
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import validation_curve

rng = np.random.default_rng(42)

def true_f(x):
    return np.sin(2 * np.pi * x)

n = 200
X = rng.uniform(0, 1, size=(n, 1))
y = true_f(X.ravel()) + rng.normal(0, 0.3, size=n)  # сигнал + шум sigma=0.3

depths = range(1, 16)
train_scores, val_scores = validation_curve(
    DecisionTreeRegressor(random_state=0),
    X, y,
    param_name="max_depth",
    param_range=depths,
    scoring="neg_mean_squared_error",
    cv=5,
)

train_mse = -train_scores.mean(axis=1)
val_mse = -val_scores.mean(axis=1)

for d, tr, va in zip(depths, train_mse, val_mse):
    print(f"depth={d:2d}  train_mse={tr:.3f}  val_mse={va:.3f}")

best = depths[int(np.argmin(val_mse))]
print(f"\nОптимальная глубина по валидации: {best}")
print(f"Неустранимый шум sigma^2 = {0.3**2:.3f} — нижняя граница val_mse")
```

Train MSE монотонно падает (ёмкость растёт, подгонка под шум). Val MSE имеет U-образный минимум — это bias-variance компромисс в действии, а его дно упирается в $\sigma^2 = 0.09$.

Оценка VC-подобной устойчивости через bootstrap-разброс предсказаний:

```python
from sklearn.base import clone

def variance_estimate(model, X, y, x_query, n_boot=200):
    preds = np.empty((n_boot, len(x_query)))
    for b in range(n_boot):
        idx = rng.integers(0, len(X), len(X))  # bootstrap-выборка
        m = clone(model).fit(X[idx], y[idx])
        preds[b] = m.predict(x_query)
    return preds.mean(axis=0), preds.var(axis=0)

x_query = np.linspace(0, 1, 50).reshape(-1, 1)

for depth in [2, 8, 14]:
    mean_p, var_p = variance_estimate(
        DecisionTreeRegressor(max_depth=depth, random_state=0), X, y, x_query
    )
    bias2 = np.mean((true_f(x_query.ravel()) - mean_p) ** 2)
    print(f"depth={depth:2d}  bias^2={bias2:.3f}  variance={var_p.mean():.3f}")
```

Малая глубина — высокое смещение, низкий разброс. Большая глубина — наоборот. Это разложение из теории, измеренное напрямую.

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude, Cursor, Copilot) полезны как партнёр по выводу формул и проверке интуиции. Сильные сценарии: попросить вывести bias-variance разложение шаг за шагом и самому проверить каждый переход; сгенерировать контрпример к ложному утверждению; объяснить, почему VC-размерность линейного классификатора равна $p+1$ с геометрической картинкой.

    Пример промпта: "Выведи bias-variance разложение для квадратичной потери, показав каждый шаг раскрытия матожидания и почему перекрёстный член зануляется. Затем объясни, какое предположение о шуме здесь критично и что сломается без него."

    Где врут и что перепроверять: LLM путают VC-размерность с числом параметров (для нейросетей это совсем не одно и то же); часто дают устаревшую картину "больше параметров = переобучение", игнорируя double descent; могут уверенно привести неверную константу в оценке обобщения. Любую формулу сверяй с первоисточником (ESL, Shalev-Shwartz & Ben-David). Численные оценки VC и Радемахера почти всегда требуют ручной проверки — модель склонна выдумывать точные значения там, где известны лишь границы.

!!! note "Стек 2026"

    Актуально: scikit-learn 1.6 (`validation_curve`, `learning_curve` для эмпирического изучения обобщения); numpy 2.x; для теоретических экспериментов с double descent — PyTorch 2.x с явным контролем ширины сети. Концепции из Shalev-Shwartz & Ben-David (PAC-learning, Радемахер) — текущий стандарт изложения теории.

    Устарело относительно старых курсов: тезис "число параметров больше числа примеров означает гарантированное переобучение" — опровергнут double descent для перепараметризованных сетей. Изолированное использование VC-размерности как единственной меры сложности уступило место сложности Радемахера и data-dependent оценкам. Совет "всегда снижай размерность перед обучением" неуниверсален: современные модели с правильным индуктивным смещением работают в исходной размерности.

!!! warning "Ловушки"

    Утечка данных (data leakage). Самая дорогая ошибка: признак, косвенно содержащий целевую переменную (например, сумма последнего вывода средств при прогнозе оттока). Эмпирический риск близок к нулю, истинный — катастрофа. Теория тут бессильна, потому что нарушено само определение задачи.

    Подбор гиперпараметров по тесту. Если ты крутишь $\lambda$ или глубину по тестовой выборке, тест перестаёт оценивать обобщение — ты переобучаешься на нём. Нужен отдельный валидационный сплит или вложенная кросс-валидация.

    Слом i.i.d. в проде. Модель обучена на одном распределении, прод подаёт другое (новый трафик, сезон, бонусная акция). Все гарантии обобщения аннулируются. Мониторь дрейф распределения признаков.

    Сеньорская ловушка: вера в число параметров. Опытные инженеры всё ещё на автомате говорят "слишком много параметров — переобучится". Для перепараметризованных сетей с SGD это неверно. Эффективная ёмкость определяется нормой решения и индуктивным смещением, а не подсчётом весов.

    Сеньорская ловушка: кросс-валидация при зависимых данных. Стандартная k-fold CV предполагает i.i.d. При временных рядах или сгруппированных данных (несколько строк на игрока) случайный сплит даёт оптимистичную оценку из-за утечки между фолдами. Нужны TimeSeriesSplit или GroupKFold.
