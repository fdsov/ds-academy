Базовый разбор временного ряда: проверка стационарности и baseline-прогноз.

```python
import pandas as pd
import numpy as np
from statsmodels.tsa.stattools import adfuller, kpss
from statsmodels.tsa.arima.model import ARIMA

revenue = pd.read_csv("daily_ggr.csv", parse_dates=["date"], index_col="date")["ggr"]

def stationarity_report(series):
    adf_p = adfuller(series.dropna())[1]
    # KPSS обратная гипотеза: высокий p => стационарен
    kpss_p = kpss(series.dropna(), regression="c", nlags="auto")[1]
    verdict = "стационарен" if adf_p < 0.05 and kpss_p > 0.05 else "нестационарен"
    return {"adf_p": round(adf_p, 4), "kpss_p": round(kpss_p, 4), "verdict": verdict}

print(stationarity_report(revenue))
print(stationarity_report(revenue.diff()))  # часто первая разность уже стационарна

# Корректная валидация: только прошлое предсказывает будущее
split = int(len(revenue) * 0.8)
train, test = revenue[:split], revenue[split:]

model = ARIMA(train, order=(2, 1, 2)).fit()
forecast = model.forecast(steps=len(test))

naive = np.repeat(train.iloc[-1], len(test))  # baseline: завтра = сегодня
mape_arima = np.mean(np.abs((test - forecast) / test))
mape_naive = np.mean(np.abs((test - naive) / test))
print(f"ARIMA MAPE: {mape_arima:.3f} | Naive MAPE: {mape_naive:.3f}")
```

Walk-forward валидация (защита от утечки из будущего) и градиентный бустинг по признакам — продакшн-стандарт.

```python
from sklearn.model_selection import TimeSeriesSplit
from lightgbm import LGBMRegressor
from sklearn.metrics import mean_absolute_error

df = revenue.to_frame("ggr")
for lag in [1, 7, 14]:
    df[f"lag_{lag}"] = df["ggr"].shift(lag)
df["roll_7"] = df["ggr"].shift(1).rolling(7).mean()  # shift(1) критичен: без него утечка
df["dow"] = df.index.dayofweek
df = df.dropna()

X, y = df.drop(columns="ggr"), df["ggr"]
tscv = TimeSeriesSplit(n_splits=5)
maes = []
for tr_idx, va_idx in tscv.split(X):
    m = LGBMRegressor(n_estimators=300, learning_rate=0.05, verbose=-1)
    m.fit(X.iloc[tr_idx], y.iloc[tr_idx])
    maes.append(mean_absolute_error(y.iloc[va_idx], m.predict(X.iloc[va_idx])))
print(f"Walk-forward MAE: {np.mean(maes):.1f} +/- {np.std(maes):.1f}")
```

Риск-метрики стратегии и когортный LTV.

```python
def sharpe(returns, rf=0.0, periods=252):
    excess = returns - rf / periods
    return np.sqrt(periods) * excess.mean() / excess.std(ddof=1)

def max_drawdown(equity):
    peak = np.maximum.accumulate(equity)
    return ((peak - equity) / peak).max()

returns = pd.Series(np.random.normal(0.0006, 0.012, 500))  # подставь реальные доходности
equity = (1 + returns).cumprod()
print(f"Sharpe: {sharpe(returns):.2f} | MaxDD: {max_drawdown(equity):.1%}")

# Когортный retention -> грубая оценка LTV
events = pd.read_csv("player_events.csv", parse_dates=["reg_date", "active_date"])
events["day"] = (events["active_date"] - events["reg_date"]).dt.days
cohort = events.groupby([events["reg_date"].dt.to_period("M"), "day"])["player_id"].nunique()
retention = cohort.unstack(0).div(cohort.unstack(0).iloc[0]).fillna(0)
arpu_active_day = 4.2  # средняя маржа в активный день, из данных
ltv_estimate = retention.mean(axis=1).sum() * arpu_active_day
print(f"Оценка LTV (площадь под retention x ARPU): {ltv_estimate:.2f}")
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude, Cursor, Copilot) сильны в этом домене как ускоритель рутины и спарринг-партнёр по логике. Хорошие задачи: сгенерировать каркас feature engineering для временного ряда, объяснить разницу между ADF и KPSS, отревьюить бэктест на предмет look-ahead bias, накидать варианты признаков для антифрода.

    Пример промпта: «Вот мой код бэктеста торговой стратегии на pandas [код]. Найди все места, где возможна утечка из будущего (look-ahead bias) или survivorship bias. Для каждого укажи строку, объясни механизм утечки и предложи корректную версию. Не предлагай улучшений доходности — только корректность валидации.»

    Где LLM врут и что перепроверять обязательно. Первое — статистические нюансы: LLM уверенно путает, в какую сторону формулируется нулевая гипотеза в KPSS, и может посоветовать `shuffle=True` для временного ряда. Любое утверждение про валидацию проверяй сам. Второе — финансовые формулы аннуализации (множитель $\sqrt{252}$ против $\sqrt{12}$) часто перевираются. Третье — LLM не знает твоих данных: он не увидит, что твой «доход» — это power law с китами, и предложит среднее. Четвёртое — он генерирует правдоподобно выглядящий бэктест, который тихо переобучен. Правило: LLM пишет черновик и ловит синтаксические грабли, но методологическую корректность подтверждаешь ты.

!!! warning "Ловушки"

    Утечка из будущего через нормализацию. `scaler.fit(весь_датасет)` до сплита заливает статистики теста в трейн. Фит только на трейне, всегда.

    Перемешивание временного ряда. `train_test_split(shuffle=True)` рвёт временную структуру и даёт случайную тестовую точку из прошлого — классическая катастрофа, на которой горят и джуны, и сеньоры в спешке.

    Переобучение стратегии перебором. Прогнал 1000 комбинаций параметров, взял лучшую — нашёл шум. Чем больше испытаний, тем выше нужный порог Sharpe, чтобы поверить. Формулируй гипотезы из экономической логики до взгляда на данные.

    Среднее в power law. «Средний LTV» в гемблинге не описывает никого: киты тянут среднее вверх, медиана в разы ниже. Сегментируй (масс vs VIP), смотри распределение, а не точку.

    Accuracy на дисбалансе. 99.5% accuracy в антифроде = модель просто говорит «все честные». Используй PR-AUC, precision/recall и матрицу стоимости ошибок.

    Прогноз LTV без даунсайда. Оптимизация только на GGR без responsible gaming — регуляторный и репутационный риск, который однажды обнулит проект штрафом.

    Аннуализация невпопад. Sharpe на дневных данных умножаешь на $\sqrt{252}$, на месячных — на $\sqrt{12}$. Перепутал множитель — Sharpe завышен в разы, решение принято на иллюзии.
