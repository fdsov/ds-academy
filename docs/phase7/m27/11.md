# M27.11 · Как встроить этику в воркфлоу

Этика, которую делают «когда-нибудь потом отдельным проектом», не делается никогда. Её нужно встроить в pipeline как обязательные gate, по аналогии с тестами в CI.

- **На этапе данных.** Datasheet обязателен. Проверка репрезентативности подгрупп. Документирование известных смещений. Явная фиксация защищённых атрибутов (даже если их не используем как фичи — они нужны для измерения fairness на выходе).
- **На этапе обучения.** Фиксация выбранного определения fairness и обоснование. По возможности — методы митигации (reweighing, ограничения при обучении, постобработка порогов по группам).
- **Перед релизом (gate).** Fairness-отчёт с метриками по подгруппам, не только агрегат. Model card. Проверка на соответствие регуляторной категории. Для значимых автоматических решений — наличие human-in-the-loop и механизма обжалования.
- **В проде.** Мониторинг fairness и приватности во времени (дрифт ломает справедливость). Логирование решений для аудируемости. Периодический повторный аудит.
