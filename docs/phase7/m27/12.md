Измерим fairness на синтетическом примере антифрод-скоринга. Стек 2026: `fairlearn` для метрик и митигации справедливости, `scikit-learn` для модели. Для дифференциальной приватности в обучении — `opacus` (для PyTorch) или `diffprivlib`; для интерпретации — `shap`.

```python
import numpy as np
import pandas as pd
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.model_selection import train_test_split
from fairlearn.metrics import (
    MetricFrame, selection_rate,
    true_positive_rate, false_positive_rate,
    demographic_parity_difference, equalized_odds_difference,
)
from sklearn.metrics import accuracy_score

rng = np.random.default_rng(42)
n = 20000
# region — защищённый атрибут; в данных база фрода различается по региону (историческая предвзятость)
region = rng.integers(0, 2, size=n)
base_fraud_rate = np.where(region == 0, 0.08, 0.05)
y = (rng.random(n) < base_fraud_rate).astype(int)

# фичи коррелируют и с фродом, и с регионом (redundant encoding защищённого атрибута)
deposit_velocity = rng.normal(0, 1, n) + 0.6 * y + 0.4 * region
night_activity = rng.normal(0, 1, n) + 0.5 * y + 0.3 * region
X = pd.DataFrame({"deposit_velocity": deposit_velocity, "night_activity": night_activity})

X_tr, X_te, y_tr, y_te, a_tr, a_te = train_test_split(
    X, y, region, test_size=0.3, random_state=0
)

model = GradientBoostingClassifier(random_state=0).fit(X_tr, y_tr)
y_pred = model.predict(X_te)

# метрики с разбивкой по защищённой группе — главное, чего НЕ видно в агрегате
mf = MetricFrame(
    metrics={
        "accuracy": accuracy_score,
        "selection_rate": selection_rate,
        "TPR": true_positive_rate,
        "FPR": false_positive_rate,
    },
    y_true=y_te, y_pred=y_pred, sensitive_features=a_te,
)
print("Агрегат:\n", mf.overall)
print("\nПо группам:\n", mf.by_group)

# единичные числа-«разрывы» для gate в CI
print("\nDemographic parity diff:", demographic_parity_difference(y_te, y_pred, sensitive_features=a_te))
print("Equalized odds diff:", equalized_odds_difference(y_te, y_pred, sensitive_features=a_te))
```

Митигация порогом по группам (постобработка) — выравниваем equalized odds, осознанно жертвуя частью агрегатной точности:

```python
from fairlearn.postprocessing import ThresholdOptimizer

postproc = ThresholdOptimizer(
    estimator=model,
    constraints="equalized_odds",
    prefit=True,
    predict_method="predict_proba",
)
postproc.fit(X_tr, y_tr, sensitive_features=a_tr)
y_fair = postproc.predict(X_te, sensitive_features=a_te)

mf_fair = MetricFrame(
    metrics={"TPR": true_positive_rate, "FPR": false_positive_rate},
    y_true=y_te, y_pred=y_fair, sensitive_features=a_te,
)
print("После митигации по группам:\n", mf_fair.by_group)
print("Equalized odds diff после:", equalized_odds_difference(y_te, y_fair, sensitive_features=a_te))
```

Идея k-анонимности на квазиидентификаторах (диагностика, а не продакшен-анонимайзер):

```python
def k_anonymity_violations(df: pd.DataFrame, quasi_ids: list[str], k: int) -> pd.DataFrame:
    sizes = df.groupby(quasi_ids).size().rename("group_size").reset_index()
    return sizes[sizes["group_size"] < k]

players = pd.DataFrame({
    "age_band": rng.choice(["18-25", "26-35", "36-45", "46+"], 1000),
    "region": rng.choice(["A", "B", "C"], 1000),
    "vip_tier": rng.choice(["bronze", "silver", "gold", "platinum"], 1000),
})
viol = k_anonymity_violations(players, ["age_band", "region", "vip_tier"], k=5)
print(f"Комбинаций квазиидентификаторов с размером < 5: {len(viol)}")
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude, Cursor, Copilot) полезны на этой теме в трёх ролях. Первое: генерация черновика model card и datasheet по шаблону — рутинная структурированная работа, где модель экономит часы. Второе: «адвокат дьявола» по fairness — попроси перечислить, как именно твоя постановка может дискриминировать и какие прокси защищённого атрибута есть в фичах. Третье: объяснение регуляторных требований простым языком как отправная точка.

    Пример промпта: «Вот описание антифрод-модели для гемблинга и список её 18 признаков [...]. Выступи в роли fairness-аудитора. (1) Какие из признаков могут быть redundant encoding защищённых атрибутов (возраст, регион, соц-эконом статус)? (2) Какое определение fairness уместно для решения о блокировке выплаты и почему? (3) Какие guardrail-метрики я обязан добавить в A/B на промо? Для каждого пункта дай конкретику, без общих слов.»

    Где LLM врут и что перепроверять обязательно: конкретные пункты, сроки и пороги регуляций (GDPR, EU AI Act) модель путает и устаревает — сверяй с первоисточником или юристом, никогда не цитируй закон со слов LLM в проде. Численные «факты» вроде «87% идентифицируются по трём полям» проверяй по оригинальным исследованиям. И главное: LLM не знает базовых ставок и распределений твоих данных — реальные fairness-метрики считаются только на твоих данных кодом, модель тут лишь подсказчик гипотез, а не источник чисел.

!!! note "Стек 2026"

    Актуально сейчас: `fairlearn` — де-факто стандарт для fairness-метрик и митигации в Python-экосистеме; `shap` для локальных объяснений; дифференциальная приватность через `opacus` (DP-SGD для PyTorch) и `diffprivlib`. Регуляторно — EU AI Act действует поэтапно (запреты с февраля 2025, режим GPAI с августа 2025; высокорисковые требования отложены до декабря 2027 / августа 2028), риск-категоризация стала практическим чек-листом, а не теорией. Model cards и datasheets — ожидаемая норма, а во многих компаниях обязательный артефакт релиза.

    Что устарело относительно старых курсов: подход «удалили защищённый признак — значит fair» давно опровергнут (redundant encoding). «Одна метрика fairness на все случаи» — нет, теорема о невозможности заставляет выбирать осознанно. «Анонимизировали = безопасно» — устарело, стандарт сместился к дифференциальной приватности с явным бюджетом. «Этика — отдельный этап в конце» сменилась встроенными gate в CI/CD.

!!! warning "Ловушки"

    - **Агрегатная метрика прячет дискриминацию.** Accuracy 0.92 в среднем может скрывать TPR 0.9 в одной группе и 0.6 в другой. Всегда считай метрики с разбивкой по подгруппам — на этом горят даже сеньоры, гордые общим ROC-AUC.
    - **Fairness through unawareness.** Уверенность, что удаление защищённого признака делает модель справедливой. Признак протекает через прокси; справедливость измеряется на выходе.
    - **Метка как истина.** Забыть, что $\tilde{Y}$ — смещённый прокси (пойманные мошенники, обратившиеся за самоисключением). Оптимизация под кривую метку даёт кривую модель при идеальных «метриках».
    - **Калибровка приравнивается к справедливости.** «Мы откалиброваны» не закрывает вопрос равного FPR — это COMPAS наоборот. Назови явно, какое определение ты выбрал.
    - **Post-hoc объяснение как истина.** SHAP поверх чёрного ящика — приближение, иногда нестабильное. Для высокорисковых решений рассматривай интерпретируемую по построению модель.
    - **Приватность одного файла.** Деанонимизация идёт через связывание с внешними данными. Гарантию формулируй против атакующего с фоновым знанием.
    - **A/B без guardrail по вреду.** Оптимизация депозита/time-on-site без метрик markers of harm — прямой путь к эксплуатации проблемных игроков и отзыву лицензии.
