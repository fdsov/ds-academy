- **Quant-сигнал с честным walk-forward** — стратегия с риск-метриками (Sharpe, MaxDD) и демонстрацией, как look-ahead bias рисует фиктивную прибыль (M28.5).

Ключевые понятия, которые ты применишь на практике: стационарность и её тест (ADF/KPSS), наивный baseline как планка, walk-forward без утечки из будущего, `shift(1)` против look-ahead bias, аннуализация Sharpe множителем $\sqrt{252}$, T-shape.

Цель не «получить красивую кривую», а **поймать себя на граблях, на которых горят сеньоры**: перемешивание ряда, утечка через нормализацию, переобучение перебором, перепутанный множитель аннуализации.

```python
import numpy as np
import pandas as pd

rng = np.random.default_rng(28)
n = 730  # два года дневных данных
dates = pd.date_range("2024-01-01", periods=n, freq="D")

trend = np.linspace(8000, 14000, n)
weekly = 1500 * np.sin(2 * np.pi * dates.dayofweek / 7)        # выходные выше
weekend = np.where(dates.dayofweek >= 5, 2500, 0)             # суб/вс бонус
month_end = np.where(dates.day >= 27, 1800, 0)                # выплаты зарплат
noise = rng.normal(0, 900, n)
ggr = np.clip(trend + weekly + weekend + month_end + noise, 1000, None)

revenue = pd.Series(ggr, index=dates, name="ggr")
revenue.to_csv("daily_ggr.csv")
print(revenue.describe().round(0))
```

«Что получилось»: ряд с восходящим трендом (значит нестационарный по среднему), стабильной недельной сезонностью и всплесками в конце месяца. Это типичный бизнес-ряд, на котором ARIMA и Prophet — baseline, а бустинг по признакам — рабочая лошадка прода.

## Ход работы

### Шаг 1: проверь стационарность

**Зачем.** В M28.4 стационарность — центральное понятие: модель, обученная на старом режиме, не работает на новом, если «характер» ряда уплывает. Прежде чем прогнозировать, надо понять, нужна ли дифференциация. Смотрим оба теста (ADF + KPSS), потому что их нулевые гипотезы зеркальны.

```python
from statsmodels.tsa.stattools import adfuller, kpss

def stationarity_report(series):
    s = series.dropna()
    adf_p = adfuller(s)[1]                                  # H0: нестационарен
    kpss_p = kpss(s, regression="c", nlags="auto")[1]      # H0: стационарен
    verdict = "стационарен" if adf_p < 0.05 and kpss_p > 0.05 else "нестационарен"
    return {"adf_p": round(adf_p, 4), "kpss_p": round(kpss_p, 4), "verdict": verdict}

print("Исходный:", stationarity_report(revenue))
print("Разность:", stationarity_report(revenue.diff()))
```

**Что получилось.** На исходном ряду ADF не отвергает нестационарность (высокий p) из-за тренда — вердикт «нестационарен». Первая разность $\nabla X_t = X_t - X_{t-1}$ убирает тренд, и обычно вердикт переключается на «стационарен». Это и есть «I» в ARIMA: $d=1$.

!!! question "Проверь себя"

    1. Почему мы смотрим ADF и KPSS вместе, а не один из них?
    2. Что физически делает `.diff()` с трендом ряда?

??? success "Ответы"

    1. У тестов зеркальные нулевые гипотезы: ADF — H0 «нестационарен», KPSS — H0 «стационарен». Согласие обоих (ADF отверг нестационарность И KPSS не отверг стационарность) даёт уверенность; противоречие — сигнал к осторожности.
    2. Считает приращение между соседними точками. Линейный тренд превращается в примерно постоянное среднее, поэтому уплывающее `E[X_t]` стабилизируется — ряд приближается к стационарному.

### Шаг 2: разбей по времени и поставь baseline

**Зачем.** В M28.4 жёсткое правило: никогда не сдавай форкаст без сравнения с наивным baseline. И никакого `shuffle=True` — сплит только хронологический, иначе будущее утечёт в трейн. Baseline задаёт планку, ниже которой сложность не оправдана.

```python
horizon = 30
train, test = revenue[:-horizon], revenue[-horizon:]

naive = np.repeat(train.iloc[-1], horizon)                       # завтра = сегодня
seasonal_naive = train.iloc[-7:].values.tolist() * (horizon // 7 + 1)
seasonal_naive = np.array(seasonal_naive[:horizon])              # повтор недели

def mape(y, yhat):
    return float(np.mean(np.abs((y.values - yhat) / y.values)))

print(f"Naive MAPE:          {mape(test, naive):.3f}")
print(f"Seasonal-naive MAPE: {mape(test, seasonal_naive):.3f}")
```

**Что получилось.** Сезонный наив (прогноз = значение неделю назад) заметно бьёт плоский наив, потому что в ряду сильная недельная сезонность. Это уже планка: любая «умная» модель обязана её перебить, иначе она бесполезна.

### Шаг 3: ARIMA как baseline-классика

**Зачем.** M28.4: ARIMA редко выигрывает на проде, но это незаменимый инструмент мышления и вторая планка. `order=(2,1,2)` — AR(2), одна дифференциация (из Шага 1), MA(2).

```python
from statsmodels.tsa.arima.model import ARIMA

model = ARIMA(train, order=(2, 1, 2)).fit()
arima_fc = model.forecast(steps=horizon).values
print(f"ARIMA MAPE: {mape(test, arima_fc):.3f}")
```

**Что получилось.** ARIMA ловит тренд и краткосрочную автокорреляцию, но базовая (несезонная) ARIMA плохо держит недельный паттерн — она может проиграть сезонному наиву. Это честный результат: он показывает, что без явных сезонных признаков классика тут не король. Вывод модуля — для сезонности нужен SARIMA или признаки календаря, что мы и сделаем бустингом.

### Шаг 4: LightGBM по лагам с walk-forward

**Зачем.** Главный продакшн-стандарт 2026 из M28.4 и M28.10: градиентный бустинг по инженерным признакам (лаги, скользящие, календарь) + валидация `TimeSeriesSplit`. Критичная деталь — `shift(1)` в скользящем среднем: без него признак «знает» текущий день и даёт утечку.

```python
from sklearn.model_selection import TimeSeriesSplit
from lightgbm import LGBMRegressor
from sklearn.metrics import mean_absolute_error

df = revenue.to_frame("ggr")
for lag in [1, 7, 14, 28]:
    df[f"lag_{lag}"] = df["ggr"].shift(lag)
df["roll_7"] = df["ggr"].shift(1).rolling(7).mean()   # shift(1) спасает от утечки
df["roll_28"] = df["ggr"].shift(1).rolling(28).mean()
df["dow"] = df.index.dayofweek
df["day"] = df.index.day
df = df.dropna()

X, y = df.drop(columns="ggr"), df["ggr"]
tscv = TimeSeriesSplit(n_splits=5)
maes = []
for tr, va in tscv.split(X):
    m = LGBMRegressor(n_estimators=400, learning_rate=0.05, num_leaves=31, verbose=-1)
    m.fit(X.iloc[tr], y.iloc[tr])
    maes.append(mean_absolute_error(y.iloc[va], m.predict(X.iloc[va])))
print(f"Walk-forward MAE: {np.mean(maes):.0f} +/- {np.std(maes):.0f}")
```

**Что получилось.** Walk-forward гоняет 5 расширяющихся окон «учись на прошлом — проверяйся на будущем», поэтому MAE честный, без заглядывания вперёд. Бустинг с лагами `lag_7`, `lag_14` и календарём `dow`/`day` видит недельную сезонность и конец месяца — он должен уверенно бить и наив, и ARIMA. **Артефакт 1 готов**: ты можешь предъявить таблицу MAPE/MAE и обоснованный вывод, что сложность оправдана.

!!! question "Проверь себя"

    1. Зачем `shift(1)` перед `.rolling(7).mean()`?
    2. Чем `TimeSeriesSplit` принципиально отличается от обычного `KFold`?

??? success "Ответы"

    1. Без сдвига скользящее среднее за 7 дней включает сегодняшний `ggr` — целевое значение, которого в момент прогноза ещё нет. Это look-ahead bias: признак «подсматривает» ответ, на проде он недоступен, и метрика обвалится.
    2. `KFold` берёт случайные фолды (будущее попадает в трейн) — для ряда это утечка. `TimeSeriesSplit` всегда обучается на прошлом и проверяется на следующем по времени блоке, сохраняя причинность.

### Шаг 5: quant-сигнал и цена утечки

**Зачем.** Вторая ветка направления (M28.5): сигнал с честным walk-forward и риск-метриками. Здесь главный урок — насколько сильно look-ahead bias раздувает фиктивный Sharpe. Сделаем оба варианта: честный сигнал (торгуем по вчерашнему сигналу) и «читерский» (по сегодняшней доходности).

```python
prices = 100 * np.exp(np.cumsum(rng.normal(0.0003, 0.012, 600)))
ret = pd.Series(prices, index=pd.date_range("2024-01-01", periods=600)).pct_change().dropna()

def sharpe(r, periods=252):
    return np.sqrt(periods) * r.mean() / r.std(ddof=1)

def max_drawdown(equity):
    peak = np.maximum.accumulate(equity)
    return float(((peak - equity) / peak).max())

signal = (ret.rolling(5).mean() > 0).astype(int)        # momentum-сигнал
honest = signal.shift(1) * ret                          # решаем ВЧЕРА, торгуем сегодня
cheat = signal * ret                                    # утечка: сигнал по сегодня

for name, strat in [("честный", honest.dropna()), ("с утечкой", cheat.dropna())]:
    eq = (1 + strat).cumprod()
    print(f"{name:10s} Sharpe={sharpe(strat):5.2f}  MaxDD={max_drawdown(eq):.1%}")
```

**Что получилось.** Честный вариант даёт скромный (часто околонулевой) Sharpe — это реалистично для низкого отношения сигнал/шум из M28.5. «С утечкой» Sharpe взлетает в разы, потому что `signal * ret` использует знак сегодняшней доходности для торговли сегодня же — невозможно вживую. **Артефакт 2 готов**: сигнал с риск-метриками плюс наглядная демонстрация, почему `.shift(1)` отделяет правду от иллюзии.

!!! question "Проверь себя"

    1. Почему `signal.shift(1) * ret` корректно, а `signal * ret` — нет?
    2. Если поменять `periods=252` на `12`, как исказится Sharpe и почему это опасно?

??? success "Ответы"

    1. Решение о позиции принимается по информации, доступной на конец предыдущего дня (`shift(1)`). `signal * ret` строит сигнал из текущей доходности и ей же торгует — заглядывание в будущее, дающее нереализуемую прибыль.
    2. Sharpe масштабируется множителем $\sqrt{K}$. Для дневных данных верно $\sqrt{252}\approx15.9$; ошибочный $\sqrt{12}\approx3.46$ занизит/исказит результат в ~4.6 раза. Перепутанный множитель ведёт к решению на иллюзии — завышенному или заниженному риск-скорректированному доходу.

### Шаг 6: персональный план углубления

**Зачем.** M28.3 и M28.8: специализация — сознательный сдвиг в зону малого предложения $S$. Финальный артефакт — документ на одну страницу. Прогони два кандидата через формулу и зафиксируй выбор.

Оцени по шкале 1-5 спрос $D$, импакт $\text{Impact}$ и (обратно) предложение $S$, посчитай $V \propto \dfrac{D \cdot \text{Impact}}{S}$:

```python
import pandas as pd
plan = pd.DataFrame({
    "D": [4, 5], "Impact": [3, 5], "S": [5, 2]},          # S: 5=много людей, 2=мало
    index=["обобщённый ML-инженер", "iGaming-аналитик/антифрод"])
plan["V"] = (plan["D"] * plan["Impact"] / plan["S"]).round(2)
print(plan)
```

Затем запиши план по шаблону (три фильтра + 90 дней):

```text
Вертикаль: <time series / quant / gambling / NLP / CV / recsys>
Фильтр 1 — нечестное преимущество: <доступ к данным / доменный опыт>
Фильтр 2 — деньги: <прямой P&L-эффект домена>
Фильтр 3 — интерес: <почему не выгорит за 5-10 лет>

90 дней (правило 70/20/10):
- Читаю: <Hyndman fpp3 / Lopez de Prado / PyMC-Marketing>
- Строю: <форкаст GGR в прод / антифрод-классификатор / LTV-модель>
- Метрика успеха: <MAE форкаста < X / PR-AUC антифрода > Y / Sharpe честной стратегии>
```

**Что получилось.** Колонка `V` численно показывает, что узкий домен с малым $S$ выигрывает за счёт знаменателя. **Артефакт 3 готов** — план привязан к твоему преимуществу, а не к моде.
