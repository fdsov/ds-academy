# M0.5 · Роли в команде данных

### Интуиция

Часто всё валят в кучу "ну, человек с данными". На деле это четыре разные профессии с разными входами и выходами. Грубая аналогия с рестораном: Data Engineer строит и обслуживает кухню и поставки продуктов, Data Analyst готовит конкретные блюда по запросу зала, Data Scientist изобретает новые рецепты и проверяет их на дегустациях, ML Engineer ставит лучшие рецепты на конвейер, чтобы их готовили тысячи раз в день стабильно.

### Формально

Различаем роли по трём осям: что на входе, что на выходе, за что отвечает.

- Data Engineer. Вход: сырые источники (логи ставок, платёжный шлюз, события клиента). Выход: чистые, надёжные, доступные данные (хранилище, пайплайны, витрины). Отвечает за то, чтобы данные вообще существовали, были свежими и корректными. Стек: SQL, Python, Spark, Airflow/Dagster, dbt, потоковые системы (Kafka), облачные хранилища.
- Data Analyst. Вход: вопрос бизнеса ("почему упал депозит в Бразилии?"). Выход: ответ с цифрами, дашборд, отчёт. Отвечает за корректную интерпретацию и коммуникацию. Стек: SQL, BI (Metabase, Superset, Tableau), Python для анализа, статистика на уровне A/B и когорт.
- Data Scientist. Вход: открытая задача с неопределённостью ("можно ли предсказать отток игрока?"). Выход: модель, эксперимент, количественный вывод с оценкой неопределённости. Отвечает за корректность метода и валидность вывода. Стек: статистика, ML, экспериментирование, Python (pandas/polars, scikit-learn, причинный вывод).
- ML Engineer. Вход: рабочая модель/прототип. Выход: модель в продакшене — стабильная, быстрая, мониторимая, переобучаемая. Отвечает за инженерное качество ML-системы. Стек: Python, MLOps (контейнеры, оркестрация, feature store, мониторинг дрифта), иногда низкоуровневая оптимизация.

Границы размыты, и в маленькой команде один человек носит несколько шляп. Но осознавать, в какой роли ты сейчас, критично: в роли аналитика твоя ошибка стоит одного отчёта, в роли ML-инженера твоя ошибка крутится в проде 24/7 и масштабирует ущерб.

!!! note "Где здесь Data Scientist по-настоящему"

    Сердцевина именно DS-роли — это не "обучить модель", а отвечать на вопрос в условиях неопределённости так, чтобы вывод выдержал критику. Модель — лишь один из инструментов. Эксперимент, причинный анализ, аккуратная оценка — тоже DS. Поэтому научный метод (следующая секция) и есть ядро профессии.

    3. Почему ошибка ML Engineer в среднем дороже ошибки Data Analyst?
    4. В чём сердцевина роли Data Scientist, если это не "обучить модель"?

??? success "Ответы"

    1. Что на входе, что на выходе, за что отвечает.
    2. Data Engineer: вход — сырые источники (логи, платёжный шлюз), выход — чистые надёжные доступные данные (хранилища, пайплайны, витрины). Data Analyst: вход — конкретный вопрос бизнеса, выход — ответ с цифрами/дашборд/отчёт.
    3. Ошибка аналитика стоит одного отчёта, а ошибка ML Engineer крутится в проде 24/7 и масштабирует ущерб на каждый прогон.
    4. Отвечать на вопрос в условиях неопределённости так, чтобы вывод выдержал критику; модель — лишь один из инструментов наряду с экспериментом и причинным анализом.
