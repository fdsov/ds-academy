Базовый инструмент линейной алгебры и численного анализа в Python по-прежнему NumPy (версия 2.x на июнь 2026, с переходом на свободный потоковый интерпретатор Python 3.13+). Для символьного дифференцирования — SymPy, для автоматического — PyTorch.

```python
import numpy as np

# Векторы как профили двух игроков: [депозиты, ср.ставка, дни_актив, GGR]
a = np.array([5, 120, 14, 340], dtype=float)
b = np.array([6, 110, 16, 380], dtype=float)

# Скалярное произведение и косинусная близость (поиск похожих игроков)
dot = a @ b                      # оператор @ — матричное/скалярное умножение
cos = dot / (np.linalg.norm(a) * np.linalg.norm(b))
print(f"dot={dot:.1f}, cosine={cos:.4f}")  # cosine близко к 1 — игроки похожи

# Нормы
print(np.linalg.norm(a, ord=2), np.linalg.norm(a, ord=1))
```

```python
# Матричное умножение: прогноз линейной модели для батча игроков
X = np.array([[5, 120, 14],
              [6, 110, 16],
              [2,  30,  3]], dtype=float)   # 3 игрока, 3 признака
w = np.array([0.5, 0.01, 2.0])              # веса модели
y_hat = X @ w                                # форма (3,) — прогноз на каждого
print(y_hat)

# Ранг и обусловленность: диагностика мультиколлинеарности
print("rank:", np.linalg.matrix_rank(X))
print("cond:", np.linalg.cond(X))           # огромное число -> почти сингулярна
```

```python
# Собственные значения/векторы ковариационной матрицы — основа PCA
Xc = X - X.mean(axis=0)                      # центрируем признаки
cov = (Xc.T @ Xc) / (len(X) - 1)            # ковариационная матрица
eigvals, eigvecs = np.linalg.eigh(cov)       # eigh — для симметричных матриц
order = np.argsort(eigvals)[::-1]            # по убыванию дисперсии
print("доля объяснённой дисперсии:", eigvals[order] / eigvals.sum())
```

```python
# Градиентный спуск своими руками: минимизируем MSE линейной регрессии
rng = np.random.default_rng(42)
X = rng.normal(size=(500, 3))
true_w = np.array([2.0, -1.0, 0.5])
y = X @ true_w + rng.normal(scale=0.1, size=500)

w = np.zeros(3)
eta = 0.1
for step in range(200):
    grad = (2 / len(X)) * X.T @ (X @ w - y)   # градиент MSE по весам
    w -= eta * grad                           # шаг против градиента
print("восстановленные веса:", w.round(3))    # близко к [2, -1, 0.5]
```

```python
# Автоматическое дифференцирование и цепное правило в PyTorch
import torch

x = torch.tensor(1.5, requires_grad=True)
y = torch.sin(x ** 2)        # вложенная функция: цепное правило внутри
y.backward()                 # backprop вычисляет dy/dx автоматически
# вручную: cos(x^2) * 2x
print(x.grad.item(), (torch.cos(x**2) * 2 * x).item())  # совпадают
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude, Cursor, Copilot) сильны в линейной алгебре и анализе как в репетиторе и генераторе численных проверок. Используй их, чтобы превратить формулу в работающий NumPy-код и тут же проверить интуицию экспериментом.

    Пример рабочего промпта: «Объясни геометрически, почему ковариационная матрица симметрична, и напиши NumPy-скрипт, который численно подтверждает, что её собственные векторы ортогональны. Прокомментируй только неочевидные шаги.»

    Где они врут и что перепроверять обязательно: LLM регулярно путают строки и столбцы при транспонировании, теряют коэффициент $2$ или знак в производных функций потерь, и уверенно выдают неверную размерность результата матричного умножения. Они также могут «доказать» ложное утверждение красивым текстом. Железное правило: любое математическое утверждение от LLM проверяй численным экспериментом на NumPy (подставь конкретные числа и сравни), а формулу градиента — через `torch.autograd` или конечные разности. Если LLM и автоград расходятся — прав автоград.

!!! note "Стек 2026"

    Актуально: NumPy 2.x (новый промоушн типов, ускорение на free-threaded Python 3.13+), PyTorch 2.x с `torch.compile` для быстрого автоградиента, JAX для исследовательского кода с `grad`/`vmap`, SymPy для символьной математики. Для PCA и разложений — `scikit-learn` и `numpy.linalg`/`scipy.linalg` (под капотом LAPACK).

    Устарело относительно старых курсов: ручное обращение матриц через `np.linalg.inv` для решения систем (используй `np.linalg.solve` или разложения); `np.matrix` (давно deprecated, всегда используй `np.array` и оператор `@`); вычисление PCA через явную ковариацию и `eig` на больших данных (предпочтительно SVD через `np.linalg.svd` или `TruncatedSVD` — численно стабильнее).

!!! warning "Ловушки"

    - Путаница строк и столбцов: `X.shape = (n_samples, n_features)` — объекты по строкам. Перепутаешь оси при умножении — получишь молчаливо неверный результат вместо ошибки, если размерности случайно совпали.
    - Явное обращение матрицы `np.linalg.inv(X.T @ X)` — численно нестабильно при мультиколлинеарности. Сеньоры горят на этом в продакшене: модель «работает», но коэффициенты бессмысленны из-за плохой обусловленности.
    - Забытый коэффициент в градиенте MSE (множитель $2/n$) или неверный знак — модель «учится наоборот» и расходится. Всегда сверяй аналитический градиент с конечными разностями.
    - Косинусная близость на нецентрированных или ненормированных данных может вводить в заблуждение: признак с огромным масштабом (GGR в тысячах) задавит остальные. Масштабируй признаки перед сравнением.
    - Слишком большой learning rate $\eta$ — лосс взрывается в NaN; слишком маленький — кажется, что «не учится». Прежде чем менять модель, проверь $\eta$.
    - Уверенность, что найденный минимум — глобальный. Для невыпуклых задач (любая нейросеть) это не так. Несколько запусков с разной инициализацией — норма, а не паранойя.
