# M2.15 · f-strings

Интуиция. f-строка — это шаблон, в который прямо внутри вставляются значения переменных. Современный, читаемый способ форматирования.

Формально. Строка с префиксом `f`, выражения внутри `{}` вычисляются и подставляются. Поддерживает форматные спецификаторы после `:` и режим отладки `=`.

```python
ltv = 1234.5678
market = "RU"
print(f"LTV={ltv:.2f} в {market}")       # LTV=1234.57 в RU
print(f"{ltv:,.0f}")                       # 1,235 — разделитель тысяч
print(f"{ltv=}")                           # ltv=1234.5678 — режим отладки
conv = 0.1834
print(f"конверсия {conv:.1%}")             # конверсия 18.3%
```

## Как это читать «на пальцах»

f-строка состоит из двух частей: обычного текста и «дырок» `{...}`, куда Python подставляет
вычисленное значение. Всё, что вне скобок, печатается как есть. Всё, что внутри скобок, —
это полноценное выражение Python: переменная, арифметика, вызов функции, обращение к словарю.

```python
deposits = 47
revenue = 18230.5
print(f"Сегодня {deposits} депозитов на сумму ${revenue:,.2f}")
# Сегодня 47 депозитов на сумму $18,230.50

avg = revenue / deposits
print(f"Средний депозит: ${avg:.2f}")          # Средний депозит: $387.88
print(f"Округлённо: {revenue / deposits:.0f}") # выражение прямо в скобках: Округлённо: 388
```

Важно: вычисление происходит в момент создания строки. Если потом изменить `deposits`,
уже напечатанная строка не поменяется — внутри неё лежит «снимок» значения.

## Анатомия форматного спецификатора

После двоеточия внутри `{}` идёт мини-язык формата. Общая форма:

`{значение:[заполнитель][выравнивание][знак][ширина][,или_][.точность][тип]}`

Не нужно помнить всё — достаточно нескольких рабочих кирпичей, которые в аналитике
встречаются постоянно:

```python
x = 1234.5678

print(f"{x:.2f}")     # 1234.57   — 2 знака после точки (округление)
print(f"{x:,.2f}")    # 1,234.57  — разделитель тысяч + 2 знака
print(f"{x:10.2f}")   #    1234.57 — ширина поля 10, выравнивание вправо
print(f"{x:<10.2f}")  # 1234.57    — выравнивание влево
print(f"{x:+.2f}")    # +1234.57   — всегда показывать знак
print(f"{x:.0f}")     # 1235       — без дробной части
```

Проценты — отдельный частый случай. Спецификатор `%` умножает на 100 и дописывает знак `%`:

```python
conv = 0.1834
print(f"{conv:.1%}")  # 18.3%  (0.1834 * 100, один знак после точки)
print(f"{conv:.0%}")  # 18%
```

Типичная ошибка: написать `{conv*100:.1f}%` и получить тот же результат руками. Это работает,
но `:.1%` короче и не даёт забыть про умножение.

## Режим отладки `=`

Если поставить `=` сразу после выражения, Python напечатает и сам текст выражения, и его
значение. Это спасает при отладке — не нужно дублировать имя переменной вручную.

```python
ltv = 1234.5678
cac = 410.0
print(f"{ltv=}")            # ltv=1234.5678
print(f"{ltv - cac=:.2f}")  # ltv - cac=824.57  — работает и с выражением, и со спецификатором
```

Раньше писали `print("ltv =", ltv)`, и при копировании легко было получить рассинхрон между
подписью и переменной. `{ltv=}` устраняет этот класс багов.

## Разобранный пример из продуктовой аналитики

Соберём строку-сводку по когорте игроков так, как её отдают в дашборд или в чат команды:

```python
ftd = 312          # first-time depositors
ngr = 54870.40     # net gaming revenue, $
retention_d7 = 0.231
arpu = ngr / ftd

summary = (
    f"Когорта: FTD={ftd}, "
    f"NGR=${ngr:,.2f}, "
    f"ARPU=${arpu:,.2f}, "
    f"R7={retention_d7:.1%}"
)
print(summary)
# Когорта: FTD=312, NGR=$54,870.40, ARPU=$175.87, R7=23.1%
```

Здесь видно сразу три приёма: разделитель тысяч для денег (`:,.2f`), процент для retention
(`:.1%`) и вычисление ARPU прямо при подстановке. Длинную f-строку разбили на несколько
соседних строковых литералов в скобках — Python склеит их автоматически.

## Пограничные случаи и типичные ошибки

Фигурные скобки как литерал. Если нужен сам символ `{` или `}`, его удваивают:

```python
print(f"{{не выражение}} {ltv:.0f}")  # {не выражение} 1235
```

Кавычки внутри `{}`. До Python 3.12 нельзя было использовать тот же тип кавычек, что и у
строки. Безопасный приём — заранее вынести значение в переменную:

```python
row = {"market": "RU"}
print(f"рынок {row['market']}")  # одинарные внутри двойных — ок
```

Забытый префикс `f`. Самая частая ошибка: `print("LTV={ltv}")` напечатает буквально
`LTV={ltv}`, без подстановки. Если в выводе видны скобки — проверь, есть ли `f` перед кавычкой.

Бэкслеш внутри выражения. До Python 3.12 нельзя было ставить `\` прямо в `{...}`. Обходится
переменной или f-строкой над уже готовым значением.

`None` и нечисловые значения. Спецификатор `:.2f` сломается, если значение не число:

```python
value = None
# print(f"{value:.2f}")  # TypeError: unsupported format string passed to NoneType
print(f"{value}")        # None — без числового спецификатора печатается как есть
```

Поэтому числовые форматы применяют только к тому, что гарантированно число. Для возможных
`None` сначала подставляют значение по умолчанию.

!!! warning "f-строка форматирует для показа, а не для расчётов"

    `f"{ltv:.2f}"` возвращает строку `"1234.57"`, а не число. Округлённое значение нужно
    только для вывода человеку. Если потом считать на основе этого текста, придётся обратно
    парсить через `float(...)`. Для вычислений округляй числами (`round(ltv, 2)`), а f-строку
    используй только на последнем шаге — при печати.

## Проверь себя

Что напечатает `print(f"{0.0731:.1%}")`? Ответ: `7.3%` — спецификатор `%` умножает на 100 и
оставляет один знак после точки.
