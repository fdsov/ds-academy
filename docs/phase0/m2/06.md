# M2.6 · Comprehensions: list, dict, set, generator

Это визитная карточка питоничного кода. Где новичок пишет цикл с `append`, профессионал пишет одну строку.

Интуиция. Comprehension — это «собери новую коллекцию из старой по правилу». Читается как фраза на английском: «возьми x для каждого x из данных, где выполнено условие».

Формально. Выражение `[выражение for элемент in итерабельное if условие]`. Эквивалентно циклу с накоплением, но быстрее (оптимизировано в интерпретаторе) и читается одним взглядом.

```python
# list comprehension: суммы выигрышных ставок
wins = [b.amount for b in bets if b.outcome == "win"]

# dict comprehension: id игрока -> его LTV
ltv = {p.id: p.deposits - p.withdrawals for p in players}

# set comprehension: уникальные страны
countries = {p.country for p in players}

# generator expression: ленивая сумма, без построения списка в памяти
total_ggr = sum(b.stake - b.payout for b in all_bets)
```

Разница между скобками принципиальна. Квадратные `[]` строят список целиком в памяти. Круглые `()` создают генератор — он не хранит результат, а выдаёт элементы по одному по требованию. На миллиарде ставок `sum([... for ...])` съест всю память, а `sum(... for ...)` пройдёт потоком за константную память.

!!! warning "Не превращай comprehension в нечитаемое"

    Вложенные comprehensions с двумя-тремя условиями — это уже не питонично, а ребус. Если строка не читается за один взгляд, разверни в обычный цикл. Питоничность — про ясность, а не про «в одну строку любой ценой».

!!! question "Проверь себя"

    1. В чём принципиальная разница между `[x for x in data]` и `(x for x in data)` по памяти?
    2. Почему `sum(b.stake for b in all_bets)` на миллиарде ставок предпочтительнее, чем `sum([b.stake for b in all_bets])`?
    3. Когда comprehension стоит развернуть обратно в обычный цикл?

??? success "Ответы"

    1. Квадратные скобки строят список целиком в памяти (память = размер результата). Круглые создают генератор: элементы выдаются по одному по требованию, результат не хранится.
    2. Версия с генератором проходит данные потоком за константную память; версия со списком сначала материализует весь список и съест всю память.
    3. Когда читаемость страдает: вложенные comprehensions с двумя-тремя условиями превращаются в ребус. Питоничность — про ясность, а не «в одну строку любой ценой».
