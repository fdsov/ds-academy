# M2.19 · Практика

### Быстрый квиз

Выбери вариант — ответ подсветится сразу.

```quiz
Что делает присваивание `b = a` для списка?
[ ] Создаёт независимую копию списка
[x] Связывает второе имя с тем же самым объектом
[ ] Копирует только верхний уровень вложенности
[ ] Создаёт неизменяемую версию списка
> Присваивание никогда не копирует объект — оно лишь приклеивает ещё один ярлык к тому же списку, поэтому изменение через `b` видно и через `a`.
---
Для чего из перечисленного уместно использовать оператор `is`?
[ ] Для сравнения двух чисел
[ ] Для сравнения двух строк
[x] Для сравнения с `None`
[ ] Для сравнения содержимого списков
> `is` сравнивает идентичность объекта. Для чисел и строк он «случайно» срабатывает из-за кеширования CPython; корректно его применять только для `None`, `True`, `False`.
---
Почему ключом `dict` или элементом `set` может быть только неизменяемый объект?
[ ] Потому что изменяемые объекты занимают слишком много памяти
[x] Потому что коллекция полагается на хеш, а он не должен меняться под ней
[ ] Потому что Python запрещает вкладывать списки друг в друга
[ ] Потому что изменяемые объекты медленнее создаются
> Поиск в хеш-таблице опирается на стабильный хеш. У изменяемого объекта содержимое и хеш можно поменять, что сломало бы поиск, поэтому ключи и элементы множества должны быть неизменяемыми.
---
Какова средняя асимптотика проверки `x in coll` для списка и для множества?
[ ] Для обоих O(1)
[ ] Для обоих O(n)
[x] Для списка O(n), для множества O(1)
[ ] Для списка O(1), для множества O(n)
> Список перебирает элементы линейно — O(n). Множество хранит элементы по хешу и сразу проверяет нужную ячейку — в среднем O(1). Поэтому `x in some_list` в цикле по миллиону строк превращается в O(n^2).
---
В чём принципиальная разница между `[x for x in data]` и `(x for x in data)`?
[ ] Никакой разницы, это синонимы
[x] Квадратные строят список в памяти целиком, круглые создают ленивый генератор
[ ] Круглые скобки работают только с числами
[ ] Квадратные скобки быстрее на любых данных
> `[]` материализует весь результат в памяти, `()` создаёт генератор, который выдаёт элементы по одному по требованию. На огромном логе генератор проходит данные за константную память.
---
Почему `def add_bet(bet, history=[])` — опасный код?
[ ] Список нельзя передавать аргументом
[ ] Python запрещает изменяемые аргументы
[x] Дефолт вычисляется один раз при определении и живёт между вызовами
[ ] Аргумент `bet` затирает `history`
> Значение по умолчанию вычисляется один раз при определении функции, поэтому один и тот же список накапливает данные между вызовами. Правильно — дефолт `None`, а коллекцию создавать внутри функции.
---
Чем функция с `yield` отличается от функции с `return` при вызове?
[ ] Она сразу выполняет весь код и возвращает список
[x] Она не выполняется, а возвращает объект-генератор
[ ] Она работает только с числами
[ ] Она автоматически кеширует результат
> Функция с `yield` при вызове не исполняет тело, а возвращает генератор. Код идёт до следующего `yield` только на каждом `next()`, замораживая состояние между ними — это и есть ленивые вычисления.
```

Разминка.

1. Дан список ставок `bets = [50, 120, 30, 200, 75, 30, 120]`. Через comprehension и `set` найди уникальные суммы. Через `Counter` найди три самые частые суммы.
2. Напиши функцию `ltv(deposits, withdrawals)` с аннотациями типов, возвращающую float. Добавь дефолт `withdrawals=0` корректно.
3. Объясни на примере с реальными числами, почему `a = [1,2]; b = a; b.append(3)` меняет `a`, а `a = [1,2]; b = a[:]; b.append(3)` — нет.

Основное.

4. Есть два множества id: `yesterday` и `today`. Посчитай D1-retention (доля вчерашних, вернувшихся сегодня), новых игроков и ушедших — операциями над множествами.
5. Напиши генератор `read_deposits(path)`, который лениво читает JSONL-лог и отдаёт только события типа `deposit`. Посчитай суммарный объём депозитов через `sum(... for ...)` без загрузки файла целиком.
6. Напиши декоратор `@retry(times=3)`, который при исключении повторяет вызов функции до `times` раз, потом пробрасывает последнюю ошибку. Подсказка: декоратор с аргументом — это три уровня вложенных функций.
7. Сделай `@dataclass(frozen=True)` класс `Cohort(market, month)` и проверь, что объекты можно класть в `set` и использовать ключами `dict`.

Челлендж.

8. Реализуй контекстный менеджер `db_transaction`, который при выходе без ошибки «коммитит» (печатает COMMIT), а при исключении «откатывает» (печатает ROLLBACK) и пробрасывает ошибку. Используй `@contextmanager` и `try/except/raise`.
9. У тебя лог в 10 млн строк ставок. Напиши конвейер из генераторов: парсинг → фильтр по рынку → извлечение суммы → нарастающий итог через `itertools.accumulate`. Докажи (замером памяти), что он не материализует весь лог.

Мини-проект. Собери модуль `retention.py`, который: читает `events.jsonl` ленивым генератором (dataclass-событие); считает D1, D7, D30 retention по когортам месяца регистрации; группирует через `defaultdict` и `set`; возвращает `dict[str, dict[int, float]]` (когорта → день → retention); полностью типизирован, проходит `ruff check` и `mypy` без ошибок; имеет блок `if __name__ == "__main__"`. Сравни время работы наивной (списки + `in`) и оптимизированной (множества) версий на сгенерированных данных.
