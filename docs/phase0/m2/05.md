# M2.5 · Срезы (slicing)

Интуиция. Срез — это «вырезать кусок последовательности». Как ножницами: с какого элемента начать, на каком закончить, с каким шагом.

Формально. Синтаксис `seq[start:stop:step]`. Включает `start`, исключает `stop` (полуинтервал). Отрицательные индексы считают с конца. Пропущенные части берут значения по умолчанию: начало, конец, шаг 1.

```python
bets = [10, 20, 30, 40, 50, 60]
bets[1:4]     # [20, 30, 40] — со 2-го по 4-й
bets[:3]      # [10, 20, 30] — первые три
bets[-2:]     # [50, 60] — последние два
bets[::2]     # [10, 30, 50] — каждый второй
bets[::-1]    # [60, 50, 40, 30, 20, 10] — разворот
```

Срез списка создаёт новый список (поверхностную копию). Это идиома для безопасного копирования: `copy = bets[:]`. На строках срезы работают так же, но возвращают новую строку (строки неизменяемы).

## Почему `stop` исключается — интуиция «на пальцах»

Полуинтервал `[start, stop)` придуман не для того, чтобы вас путать. У него есть три практических удобства, ради которых на него и завязан весь Python:

1. Длина среза считается вычитанием: `bets[1:4]` содержит ровно `4 - 1 = 3` элемента. Не нужно прибавлять или вычитать единицу — это убирает класс ошибок «off-by-one».
2. Срезы стыкуются без зазора и без перекрытия: `bets[:3]` и `bets[3:]` вместе дают весь список, элемент с индексом 3 не теряется и не дублируется. Это удобно для разбиения выборки на train/test: `data[:k]` и `data[k:]`.
3. `bets[i:i]` всегда пустой — нулевой по длине срез в любой точке.

Представьте индексы не «на» элементах, а «между» ними — как метки на линейке:

```text
  10   20   30   40   50   60
 0    1    2    3    4    5    6
-6   -5   -4   -3   -2   -1
```

Срез `bets[1:4]` берёт всё между меткой 1 и меткой 4 → `[20, 30, 40]`. С такой картинкой отрицательные индексы и пропуски перестают быть магией: `bets[-2:]` — это «от метки -2 до конца».

## Разбор шага и направления

`step` задаёт направление и «плотность» обхода. Положительный шаг идёт слева направо, отрицательный — справа налево. Когда шаг отрицательный, меняются и значения по умолчанию для `start`/`stop`: начало становится концом последовательности, конец — её началом. Поэтому `bets[::-1]` разворачивает список, а явное `bets[5:0:-1]` пропустит первый элемент (`stop=0` исключается):

```python
bets[5:0:-1]   # [60, 50, 40, 30, 20] — без 10, потому что stop=0 не включается
bets[5::-1]    # [60, 50, 40, 30, 20, 10] — пропущенный stop доходит до самого начала
bets[4:1:-1]   # [50, 40, 30] — от индекса 4 к индексу 1 (не включая 1)
```

Типичная ловушка: попытаться развернуть через `bets[len(bets):0:-1]` или `bets[-1:0:-1]` — оба теряют первый элемент. Запомните идиому: разворот — это `bets[::-1]`, и точка.

## Срез никогда не выходит за границы

В отличие от индексации одним числом (`bets[10]` бросит `IndexError`), срез молча обрезается по фактической длине. Это спасает от падений, но иногда маскирует баг:

```python
bets[2:100]    # [30, 40, 50, 60] — лишний хвост просто отброшен, ошибки нет
bets[100:]     # [] — пустой срез, тоже без ошибки
bets[5]        # IndexError: list index out of range
```

Практический вывод: «взять первые 10 строк» через `rows[:10]` безопасно даже если строк всего 3 — вернётся 3. А вот `rows[9]` для проверки «есть ли 10-я» — упадёт.

## Поверхностная копия: где это кусается

`copy = bets[:]` создаёт новый список, но копирует только верхний уровень — ссылки на вложенные объекты остаются общими. Для плоского списка чисел это полная независимость. Для списка списков — нет:

```python
sessions = [[10, 20], [30, 40]]
shallow = sessions[:]
shallow[0].append(99)
sessions      # [[10, 20, 99], [30, 40]] — оригинал изменился!
```

Если нужна полная независимость вложенных структур — `copy.deepcopy`. Для одномерных данных (ставки, суммы депозитов, временные метки) поверхностной копии достаточно.

## Пример из продуктовой аналитики

Допустим, `daily_revenue` — список дневной выручки казино в долларах за две недели, свежие даты в конце:

```python
daily_revenue = [
    1200, 1500, 1100, 1800, 2200, 3100, 2900,   # неделя 1 (пн–вс)
    1300, 1600, 1250, 1900, 2400, 3300, 3050,   # неделя 2 (пн–вс)
]

last_week   = daily_revenue[-7:]                 # последние 7 дней
prev_week   = daily_revenue[-14:-7]              # предыдущие 7 дней
weekends    = daily_revenue[5::7]                # каждый 6-й и... нет — см. ниже
```

Подсчёт «выходные дни» через шаг требует аккуратности: суббота/воскресенье идут парами, один `step` их не выберет. Чаще берут два отдельных среза `daily_revenue[5::7]` (субботы) и `daily_revenue[6::7]` (воскресенья). А вот сравнение недель срезами — чистый случай:

```python
wow_growth = (sum(last_week) - sum(prev_week)) / sum(prev_week)
# (16850 - 13800) / 13800 ≈ 0.221 → рост выручки неделя-к-неделе ≈ +22.1%
```

Здесь видно, почему стыкуемость срезов важна: `daily_revenue[-14:-7]` и `daily_revenue[-7:]` покрывают ровно две недели без пересечения, поэтому неделя не «считается дважды».

!!! warning "Частые ошибки"

    - `bets[1:4]` это 3 элемента, а не 4 — `stop` не включается.
    - Разворот списка — только `bets[::-1]`; конструкции с явным `stop=0` теряют первый элемент.
    - Срез не падает на выходе за границы, индексация одним числом — падает.
    - `copy = bets[:]` копирует только верхний уровень; вложенные списки остаются общими.

!!! note "Проверь себя"

    Что вернёт `daily_revenue[::-2]` для списка из 14 элементов? Сколько элементов в результате и какой элемент будет первым?
