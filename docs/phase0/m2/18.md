Собирающий пример: потоковый расчёт D7-retention по логу событий, на чистом Python со стандартной библиотекой.

```python
from __future__ import annotations
import json
from collections import defaultdict
from datetime import datetime, timedelta
from pathlib import Path
from dataclasses import dataclass


@dataclass(frozen=True)
class Event:
    player_id: int
    ts: datetime
    kind: str


def read_events(path: Path):
    """Ленивое чтение лога: одна запись в памяти за раз."""
    with path.open() as f:
        for line in f:
            raw = json.loads(line)
            yield Event(
                player_id=raw["player_id"],
                ts=datetime.fromisoformat(raw["ts"]),
                kind=raw["type"],
            )


def d7_retention(path: Path) -> float:
    first_seen: dict[int, datetime] = {}
    active_days: dict[int, set] = defaultdict(set)

    for e in read_events(path):
        if e.player_id not in first_seen:
            first_seen[e.player_id] = e.ts
        active_days[e.player_id].add(e.ts.date())

    retained = 0
    for pid, reg in first_seen.items():
        target = (reg + timedelta(days=7)).date()
        if target in active_days[pid]:
            retained += 1

    cohort = len(first_seen)
    return retained / cohort if cohort else 0.0


if __name__ == "__main__":
    rate = d7_retention(Path("events.jsonl"))
    print(f"D7 retention: {rate:.1%}")
```

Здесь работает почти всё из модуля: dataclass, генератор для ленивого чтения, контекстный менеджер `with`, `defaultdict` и `set` для уникальных дней, аннотации типов, f-string, идиома `__main__`.

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude, Cursor, Copilot) сильны в Python-рутине: сгенерировать comprehension, написать декоратор по описанию, объяснить чужой код, предложить рефакторинг цикла в векторную форму. Это ускоряет в разы.

    Пример хорошего промпта: «Вот функция, которая считает retention циклом по списку с проверкой `in` на 2 млн игроков и работает 20 минут. Перепиши, заменив списки на множества/словари, объясни итоговую асимптотику и где именно было $O(n^2)$. Сохрани аннотации типов и стиль PEP 8.»

    Где LLM врут и что перепроверять. Первое — асимптотика и память: модель уверенно скажет «оптимально», хотя оставит скрытый $O(n^2)$ или материализует список там, где нужен генератор; всегда сам проверяй сложность на горячем пути. Второе — изменяемые дефолты и побочные эффекты: модель спокойно сгенерирует `def f(x, acc=[])`; читай сигнатуры. Третье — устаревший стек: по обучающим данным LLM тянет `os.path`, `black`+`flake8`, `poetry` вместо `pathlib`, `ruff`, `uv`. Четвёртое — тонкости копирования (поверхностная против глубокой) и `is` vs `==`. Правило: LLM пишет черновик, ты держишь модель данных и сложность в голове.

!!! note "Стек 2026"

    Актуально сейчас: `uv` для окружений и пакетов (вытеснил pip/venv/poetry), `ruff` для линта и форматирования (вытеснил black + flake8 + isort), `pyright`/`mypy` для типов, dataclasses и `@dataclass(frozen=True)` вместо ручных конструкторов, `pathlib` вместо `os.path`, f-strings вместо `.format()` и `%`, встроенные дженерики `list[int]` и `str | None` вместо `typing.List`, `typing.Optional`. Polars всё активнее теснит pandas на больших данных, но pandas остаётся стандартом.

    Устарело относительно старых курсов: `%`-форматирование и `.format()`, `os.path.join`, ручное открытие/закрытие файлов без `with`, `from typing import List, Dict, Optional`, классы с написанными вручную `__init__`/`__repr__` там, где хватает dataclass, и совет «pip install в глобальное окружение».

!!! warning "Ловушки"

    - Изменяемый аргумент по умолчанию (`def f(x, acc=[])`) — живёт между вызовами. Дефолт `None`, коллекцию создавай внутри.
    - Сравнение чисел/строк через `is` — работает «случайно» из-за кеширования, ломается непредсказуемо. `is` только для `None`/`True`/`False`.
    - `b = a` для списка/словаря — это второй ярлык, не копия. Меняешь один — меняется оба. Нужна копия — `a.copy()`, для вложенных — `copy.deepcopy`.
    - `x in some_list` внутри цикла — $O(n^2)$. На больших данных переводи в set.
    - `[... for ...]` вместо `(... for ...)` там, где результат сразу агрегируется (`sum`, `any`, `max`) — лишняя материализация и память.
    - `except: pass` — глотает баги. Лови конкретный тип, логируй.
    - Поверхностная копия вложенной структуры: `outer.copy()` копирует только верхний уровень, внутренние списки общие.
    - Float-арифметика денег: `0.1 + 0.2 != 0.3`. Для финансов используй `decimal.Decimal` или считай в копейках целыми.
    - Мутация коллекции во время итерации по ней — приводит к пропускам и ошибкам. Итерируй по копии или строй новую.
