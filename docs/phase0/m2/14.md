# M2.14 · Аннотации типов (type hints)

Интуиция. Аннотации — это подписи к коду: «эта функция принимает список чисел и возвращает float». Python их не проверяет в рантайме, но они нужны людям и инструментам: IDE подсказывает, статический анализатор ловит ошибки до запуска.

Формально. Синтаксис `def f(x: int) -> str`. С Python 3.9+ встроенные дженерики пишутся как `list[int]`, `dict[str, float]`. Проверяются статически инструментами `mypy` или `pyright`, в рантайме игнорируются.

```python
def aggregate_ltv(players: list[Player], market: str | None = None) -> dict[str, float]:
    result: dict[str, float] = {}
    for p in players:
        if market is None or p.market == market:
            result[p.id] = p.ltv()
    return result
```

В DS аннотации особенно ценны на границах: сигнатуры функций, которые принимают DataFrame и что-то возвращают. Они документируют контракт лучше любого комментария и не устаревают незаметно, потому что их проверяет линтер.
