# M2.10 · Декораторы

Интуиция. Декоратор — это обёртка вокруг функции, которая добавляет поведение, не трогая саму функцию. Как чехол на телефон: телефон тот же, но добавилась защита. Типичное применение: логирование, замер времени, кеширование, проверка прав.

Формально. Декоратор — это функция, которая принимает функцию и возвращает новую функцию. Синтаксис `@decorator` над определением — синтаксический сахар для `func = decorator(func)`.

```python
import functools
import time

def timed(func):
    @functools.wraps(func)            # сохраняет имя и docstring оригинала
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__}: {time.perf_counter() - start:.3f}s")
        return result
    return wrapper

@timed
def compute_retention(events):
    ...                               # тяжёлый расчёт
    return result
```

Самый полезный встроенный декоратор для аналитика — `functools.lru_cache`. Он кеширует результаты функции: вызвал с теми же аргументами — мгновенно вернул запомненное. Это спасает при повторных дорогих запросах к одним и тем же когортам.

```python
from functools import lru_cache

@lru_cache(maxsize=None)
def cohort_size(market: str, month: str) -> int:
    return expensive_db_query(market, month)   # второй вызов с теми же аргументами — без запроса
```
