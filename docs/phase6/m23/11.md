# M23.11 · DuckDB и революция small data

**На пальцах.** Долго считалось: много данных — нужен кластер (Spark, отдельные сервера). Но оказалось, что у большинства компаний «большие данные» — это десятки гигабайт, которые помещаются на один мощный ноутбук. DuckDB — это «SQLite для аналитики»: встраиваемая колоночная база, которая молотит Parquet-файлы прямо у тебя в процессе, без серверов и кластеров.

**Формально.** **DuckDB** — встраиваемая (in-process) колоночная OLAP-СУБД. Запускается внутри Python-процесса, читает Parquet/CSV/Iceberg напрямую (в т.ч. с S3), векторизованный движок даёт скорость, сравнимую с распределёнными системами на данных до сотен ГБ. К 2026 DuckDB вытеснил Spark из огромного класса задач, где раньше поднимали кластер «на всякий случай». Рядом в той же нише small/medium data стоит **Polars** — in-process колоночный DataFrame-движок на Rust с ленивым выполнением; выбор между ними чаще про стиль (SQL у DuckDB против DataFrame-API у Polars), а не про возможности.

```python
import duckdb

# Считаем GGR по гео прямо из Parquet на S3, без кластера
con = duckdb.connect()
ggr = con.execute("""
    select geo,
           sum(bet_amount - payout) as ggr,
           count(distinct player_id) as players
    from read_parquet('s3://lake/bets/bet_date=2026-06-*/**/*.parquet')
    group by geo
    order by ggr desc
""").df()
print(ggr)
```

!!! tip "Правило выбора движка 2026"

    Спроси сначала: «помещается ли это в DuckDB на одной машине?» Чаще всего — да. Spark/распределённое — только когда данные реально не влезают (терабайты) или нужен распределённый ML. Преждевременный Spark — самая дорогая ошибка джуна-инженера.

!!! question "Проверь себя"

    1. Что такое DuckDB одной фразой и чем он принципиально отличается от Spark по способу запуска?
    2. Какой главный вопрос задаёт «правило выбора движка 2026» перед тем, как тянуть Spark?
    3. Когда распределённый движок всё-таки оправдан?

??? success "Ответы"

    1. DuckDB — встраиваемая (in-process) колоночная OLAP-СУБД, «SQLite для аналитики»; запускается внутри Python-процесса и читает Parquet/CSV/Iceberg напрямую, тогда как Spark — это отдельный распределённый кластер.
    2. «Помещается ли это в DuckDB на одной машине?» — чаще всего да; преждевременный Spark считается самой дорогой ошибкой джуна-инженера.
    3. Когда данные реально не влезают на одну машину (терабайты) или нужен распределённый ML.
