# M23.9 · Форматы и партиционирование

**На пальцах.** CSV — это как хранить таблицу текстом: читать целиком, медленно, без сжатия. Parquet — это умный бинарный формат, который хранит данные по колонкам, знает типы, сжимает и позволяет читать только нужное. Партиционирование — это раскладывать данные по папкам (например, по дате), чтобы запрос за один день не сканировал всю историю.

**Формально.**

- **Apache Parquet** — колоночный бинарный формат со встроенной схемой, типами, сжатием и статистикой по блокам (min/max), что даёт **predicate pushdown** — движок пропускает блоки, заведомо не подходящие под фильтр. Стандарт хранения в lake/lakehouse.
- **Партиционирование** — физическое разбиение датасета по значению колонки, обычно по дате: `bets/bet_date=2026-06-15/part-0.parquet`. Запрос с фильтром по дате читает только нужные партиции (**partition pruning**), а не весь датасет.

!!! warning "Партиционирование можно испортить"

    Слишком мелкие партиции (по часу, по player_id) рождают миллионы крошечных файлов — это «small files problem», и метаданными движок захлёбывается сильнее, чем экономит на чтении. Правило: партиция должна быть «увесистой» (сотни МБ — единицы ГБ). Для ставок партиционируй по дню, максимум по дню+гео, но не по игроку.
