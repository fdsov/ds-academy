# M22.6 · Контейнеризация: Docker базово

### Интуиция: упаковать всё окружение в коробку

Классическая беда: «у меня на ноутбуке работает, на сервере — нет». Причина — разные версии Python, библиотек, системных пакетов. Docker решает это, упаковывая твой код вместе со всем окружением (интерпретатор, зависимости, системные либы) в один переносимый образ — «коробку», которая запускается одинаково где угодно.

### Формально: образы и контейнеры

Образ (image) — неизменяемый снимок файловой системы с твоим приложением и зависимостями, собранный по инструкции из Dockerfile. Контейнер (container) — запущенный экземпляр образа, изолированный процесс. Образ собирают один раз (`docker build`), запускают многократно (`docker run`).

Минимальный Dockerfile для ML-сервиса фиксирует базовый образ Python, копирует зависимости, ставит их, копирует код и модель, объявляет команду запуска. Главное MLOps-правило: образ должен быть детерминированным — закрепляй версии всех зависимостей (lock-файл), иначе пересборка через месяц даст другое окружение и другие предсказания. Это прямая связь с воспроизводимостью.
