# M22.4 · Сериализация моделей

### Интуиция: как заморозить модель в файл

Обученная модель живёт в оперативной памяти Python-процесса как объект. Чтобы перенести её на сервер, нужно «заморозить» этот объект в файл, а потом «разморозить» обратно. Это и есть сериализация (сохранение) и десериализация (загрузка).

### Формально: три инструмента

pickle — стандартный механизм Python для сериализации произвольных объектов в байты. Работает почти со всем, но имеет два серьёзных недостатка: он привязан к версиям Python и библиотек (модель, сохранённая в scikit-learn 1.4, может не загрузиться в 1.6), и он небезопасен — при загрузке pickle-файла исполняется произвольный код. Никогда не загружай pickle из недоверенного источника.

joblib — надстройка над pickle, оптимизированная под большие numpy-массивы (а значит — под веса ML-моделей). Это де-факто стандарт для сохранения scikit-learn моделей: быстрее и компактнее на больших массивах. Те же ограничения по безопасности и совместимости версий, что у pickle.

ONNX (Open Neural Network Exchange) — открытый формат для обмена моделями между фреймворками. Модель конвертируется в граф вычислений, независимый от Python и от библиотеки обучения. Плюсы: можно запускать через onnxruntime на C++, в браузере, на мобильных, без тяжёлых зависимостей; быстрее на инференсе; безопаснее (нет исполнения произвольного кода). Минус: не всё конвертируется идеально, и кастомные преобразования требуют усилий.

!!! warning "Версионная ловушка pickle"

    Самая частая прод-авария новичка: модель обучили в одной версии sklearn, а сервис собрали с другой. При загрузке — невнятная ошибка или, хуже, тихо неправильные предсказания. Всегда фиксируй версии библиотек рядом с моделью (в том же артефакте или в lock-файле) и проверяй загрузку в окружении, идентичном продовому.

Правило выбора простое: внутри одного Python-стека для табличных моделей — joblib. Когда нужна кросс-платформенность, низкая латентность или деплой вне Python — ONNX.

!!! question "Проверь себя"

    1. Чем joblib предпочтительнее pickle для scikit-learn моделей?
    2. Назови два главных недостатка pickle и joblib.
    3. В каких случаях стоит выбрать ONNX вместо joblib?

??? success "Ответы"

    1. joblib оптимизирован под большие numpy-массивы (веса моделей) — сохраняет быстрее и компактнее.
    2. Привязка к версиям Python и библиотек (несовместимость при загрузке) и небезопасность — при десериализации исполняется произвольный код.
    3. Когда нужна кросс-платформенность, низкая латентность инференса или деплой вне Python.

!!! example "Мини-задание"

    Возьми любую обученную sklearn-модель, сохрани её через `joblib.dump` и через `pickle.dump`. Сравни размер двух файлов и время загрузки (`%timeit joblib.load` против `pickle.load`). Затем в `requirements.txt` зафиксируй точную версию sklearn (`scikit-learn==...`) — это и есть защита от версионной ловушки.
