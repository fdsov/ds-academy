Подбор порога под прибыль и оценка эффекта на бутстрапе.

```python
import numpy as np
from sklearn.metrics import roc_curve

def profit_at_threshold(y_true, scores, t, v_tp, c_fp, c_fn, v_tn=0.0):
    pred = scores >= t
    tp = np.sum((pred == 1) & (y_true == 1))
    fp = np.sum((pred == 1) & (y_true == 0))
    fn = np.sum((pred == 0) & (y_true == 1))
    tn = np.sum((pred == 0) & (y_true == 0))
    return tp * v_tp + fp * c_fp + fn * c_fn + tn * v_tn

def best_threshold(y_true, scores, **costs):
    grid = np.linspace(0.01, 0.99, 99)
    profits = [profit_at_threshold(y_true, scores, t, **costs) for t in grid]
    i = int(np.argmax(profits))
    return grid[i], profits[i]

t_star, profit = best_threshold(
    y_val, p_val, v_tp=40, c_fp=-320, c_fn=-40
)
print(f"Порог под прибыль: {t_star:.2f}, ожидаемая прибыль: {profit:.0f}")
```

Оценка ATE по A/B с бутстрап-доверительным интервалом.

```python
import numpy as np

def ate_bootstrap(y_treat, y_ctrl, n_boot=10000, seed=42):
    rng = np.random.default_rng(seed)
    ate = y_treat.mean() - y_ctrl.mean()
    boot = np.empty(n_boot)
    for b in range(n_boot):
        t = rng.choice(y_treat, size=len(y_treat), replace=True)
        c = rng.choice(y_ctrl, size=len(y_ctrl), replace=True)
        boot[b] = t.mean() - c.mean()
    lo, hi = np.percentile(boot, [2.5, 97.5])
    return ate, (lo, hi)

ate, ci = ate_bootstrap(retained_treat, retained_ctrl)
print(f"Эффект на удержание: {ate:+.3f}, 95% ДИ [{ci[0]:+.3f}, {ci[1]:+.3f}]")
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude, Cursor, Copilot) сильны на этой теме в трёх местах: набросать матрицу стоимости из словесного описания бизнес-процесса, сгенерировать бойлерплейт для перебора порога и бутстрапа, и переформулировать технический вывод в язык бизнеса.

    Пример промпта: «У меня антифрод-классификатор. Ложная блокировка честного игрока стоит ~320 долларов потерянного LTV, пропуск фродера ~40 долларов. Выведи формулу оптимального порога, посчитай его для этих чисел и объясни, почему он не равен 0.5. Затем напиши функцию на Python, которая перебирает порог по сетке и максимизирует ожидаемую прибыль на валидации.»

    Где врут и что перепроверять: LLM регулярно путает наблюдаемый эффект с инкрементальным — может предложить оценить ROI retention-модели как «процент оставшихся среди затронутых» без контрольной группы. Это грубая каузальная ошибка, всегда требуй holdout/A/B. Также модели любят брать порог 0.5 по умолчанию и оптимизировать F1 вместо прибыли — проверяй, что в коде максимизируется именно денежная функция. И они путают LTV-формулы (с дисконтированием и без) — сверяй с определением, которое используешь ты.

!!! note "Стек 2026"

    Актуально: causalml и EconML для uplift и causal-инференса; sklearn 1.6+ с `CalibratedClassifierCV` (isotonic/sigmoid) для калибровки; фреймворки фиче-флагов и экспериментов (GrowthBook, Statsig, Eppo) для прод-A/B с автоматическим расчётом значимости и guardrails; DoWhy для проверки каузальных допущений. Holdout и канареечный rollout — индустриальный стандарт, поддержан почти всеми платформами фиче-флагов.

    Устарело относительно старых курсов: выбор порога по F1 как умолчание; presentation эффекта модели без контрольной группы («было X%, стало Y%»); ручные A/B без планирования размера выборки; вера в то, что высокий offline-AUC автоматически означает бизнес-эффект. Также устарела практика «сначала ML, потом думаем зачем» — современный подход baseline-first (правило/эвристика, затем модель только при доказанном upside).

!!! warning "Ловушки"

    - Порог 0.5 по умолчанию. Дефолтный порог почти никогда не оптимален по прибыли. На нём горят и джуны, и сеньоры, забывшие про асимметрию стоимостей.
    - Денежные расчёты на некалиброванной модели. Умножать ранжирующий скор бустинга на деньги как вероятность — самообман. Сначала калибровка и Brier.
    - Наблюдаемое вместо инкрементального. Отчитываться «среди затронутых остались 60%» без holdout — каузальная катастрофа. Ты не знаешь, сколько осталось бы без воздействия.
    - Таргетинг churn-моделью вместо uplift. Сольёшь бюджет на sure things и lost causes. Деньги работают только на persuadables.
    - Рандомизация на уровне сессии при сетевых эффектах. Если игроки взаимодействуют (рефералы, общие лиги), нужна гео/кластерная рандомизация, иначе утечка между группами смажет эффект.
    - Train/serving skew. Фича в проде считается не так, как в обучении. Shadow mode обязателен, чтобы поймать это до запуска.
    - Подглядывание в A/B (peeking). Остановить тест, как только увидел значимость, — раздувает false positive. Размер выборки и срок фиксируются заранее (см. M23).
    - Игнор стоимости владения моделью. ML — это вечный мониторинг и переобучение. Если правило даёт 80% за 5% стоимости, модель не нужна.
