# M12.1 · Зачем этот модуль

Представь, что ты три недели крутил когортный анализ, нашёл, что игроки из канала А удерживаются на 7-й день в два раза хуже игроков из канала Б. Ты приходишь к продакту и говоришь: «retention D7 по каналу А — 11.3%, по каналу Б — 22.6%, разница статистически значима, p меньше 0.01». Продакт кивает, ничего не понимает, и решение не принимается. А теперь ты показываешь две retention-кривые на одном графике: одна резко проваливается, вторая держится. Продакт смотрит три секунды и говорит «всё, режем закупку по каналу А». Это и есть цена визуализации — не украшение, а пропускная способность канала «данные → решение».

Без этого навыка ломается главное: твой анализ не доходит до того, кто принимает решение. Можно быть гением статистики и при этом сливать всю ценность работы на последней миле — на картинке, которую никто не понял или, хуже, которую неправильно поняли. Обрезанная ось, перепутанная палитра, график-спагетти из двадцати линий — и аудитория уносит из твоей презентации ложный вывод. В гемблинге и финансах, где решения двигают миллионы, цена такой ошибки прямая и денежная.

Этот модуль учит двум вещам сразу: технике (как из данных получить корректную картинку на актуальном стеке 2026) и риторике (как из картинки собрать историю, которая меняет решение). Первое без второго даёт красивые бесполезные дашборды. Второе без первого — манипуляцию. Нам нужны оба.
