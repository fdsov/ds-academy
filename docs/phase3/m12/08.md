# M12.8 · Инструменты 2026: какой когда

### Интуиция

Четыре основных инструмента в Python, и выбор не про «какой моднее», а про сценарий: статичная картинка для отчёта/статьи, статистический график за две строки, или интерактив для дашборда и разведки.

Полезнее всего держать в голове две оси, по которым инструменты расходятся:

- **Статика против интерактива.** Нужна ли пользователю возможность навести курсор, зумить, переключать серии? Если результат — PNG в PDF-отчёте или картинка в статье, интерактив только мешает (и не выживает в печати). Если результат — дашборд, где аналитик «копается» в данных, без hover и зума никак.
- **Императив против декларатива.** Императивно ты описываешь *как рисовать* (нарисуй ось, поставь точку, добавь подпись) — это matplotlib. Декларативно ты описываешь *что кодируешь* (x — это дата, y — это GGR, цвет — это страна), а движок сам решает, как это нарисовать — это altair и seaborn.objects. Декларативный код короче и воспроизводимее, императивный даёт абсолютный контроль.

Эти две оси объясняют, почему инструментов именно четыре: каждый занимает свой угол. matplotlib — статика + императив (полный контроль). seaborn — статика + полу-декларатив (быстрый EDA). plotly — интерактив + императив-ish (богатый, тяжёлый). altair — интерактив + декларатив (чистый, лёгкий, но с лимитом по объёму).

### Карта инструментов

- matplotlib — фундамент и низкоуровневый движок. Полный контроль над каждым пикселем, любой статичный экспорт (PNG, SVG, PDF для публикаций). Многословен. На нём построены seaborn и pandas.plot. Знать обязательно — это «ассемблер» визуализации, к нему спускаешься, когда надо подкрутить то, что высокоуровневая обёртка не даёт.
- seaborn — статистическая надстройка над matplotlib. Boxplot, violin, ECDF, heatmap, парные графики, регрессии — в одну строку, с приличными дефолтами. Новый интерфейс seaborn.objects (`so`) реализует грамматику графики в стиле ggplot. Рабочая лошадь EDA.
- plotly — интерактив: зум, hover-подсказки, переключение серий, 3D. Идеален для дашбордов (ядро Dash) и для разведки больших данных, где надо «потрогать». Тяжелее в вебе, чем altair.
- altair — декларативная грамматика графики поверх Vega-Lite. Пишешь, что кодируешь, не как рисуешь. Чистый, воспроизводимый, отлично связывает графики (выбор на одном фильтрует другой). Идеален, когда мыслишь грамматикой; слабее на очень больших данных (по умолчанию лимит 5000 строк).

Практическое правило 2026: EDA и статистика — seaborn; финальная статика в отчёт или статью — matplotlib (часто через seaborn с доводкой); интерактивный дашборд или презентация для стейкхолдера — plotly или altair.

### Один и тот же график на четырёх инструментах

Чтобы разница из абстрактной стала осязаемой — посмотрим, как нарисовать удержание игроков по дням (retention curve) на каждом инструменте. Данные одни: `df` с колонками `day` (номер дня, 0..30) и `retention` (доля вернувшихся, 0..1).

```python
import matplotlib.pyplot as plt

# matplotlib — императив: рисуем шаг за шагом
fig, ax = plt.subplots(figsize=(7, 4))
ax.plot(df["day"], df["retention"], marker="o")
ax.set_xlabel("День")
ax.set_ylabel("Доля вернувшихся")
ax.set_title("Retention curve")
ax.set_ylim(0, 1)
fig.savefig("retention.png", dpi=150)  # чистый статичный экспорт
```

```python
import seaborn as sns

# seaborn — короче, дефолты приличные из коробки
sns.lineplot(data=df, x="day", y="retention", marker="o")
```

```python
import plotly.express as px

# plotly — та же строка, но в выводе будет hover и зум
fig = px.line(df, x="day", y="retention", markers=True,
              title="Retention curve")
fig.show()  # интерактивный HTML-виджет
```

```python
import altair as alt

# altair — декларатив: описываем кодировку, не рисование
alt.Chart(df).mark_line(point=True).encode(
    x="day:Q",
    y="retention:Q",
)
```

Обрати внимание: matplotlib просит явно создать `fig, ax` и расставить подписи руками — зато ровно ты решаешь, где что. seaborn и altair выводят почти то же одной логической операцией. plotly при той же лаконичности отдаёт интерактивный виджет, а не картинку — на чём и строится его ниша.

### Развёрнутая интуиция «на пальцах»

Думай про это как про инструменты в мастерской, а не про конкурентов:

- matplotlib — это **верстак и набор отвёрток**. Им можно сделать всё, но за каждую мелочь платишь строчками кода. seaborn и pandas.plot — это, по сути, насадки на этот же верстак.
- seaborn — это **пресет**: ты говоришь «boxplot вот по этим колонкам», и он сам считает квартили, рисует усы, подбирает цвета. Когда нужно увидеть распределение депозитов по сегментам за 10 секунд — это он.
- plotly — это **витрина с подсветкой**: навёл мышь — увидел точную цифру, зумнул в аномальный день, выключил лишнюю серию кликом по легенде. Цена — тяжёлый JS в браузере.
- altair — это **рецепт блюда, а не само блюдо**: ты описываешь связь данных и визуальных каналов, а Vega-Lite готовит. Один рецепт легко переиспользовать и связать с другим (выбор на одном графике фильтрует второй).

### Пограничные случаи и типичные ошибки

- **«Возьму plotly, потому что красиво» для отчёта в PDF.** Интерактив в печати мёртв: hover и зум не переносятся в статичный документ, а вес HTML-виджета раздувает ноутбук. Для финальной статики — matplotlib/seaborn. Plotly умеет экспортировать в PNG (через `kaleido`), но тогда ты платишь за интерактив, которым не пользуешься.
- **altair на сыром большом датасете.** По умолчанию altair откажется рисовать больше 5000 строк (`MaxRowsError`) — это защита, а не баг: Vega-Lite встраивает данные прямо в спецификацию графика, и 200k строк превратятся в гигантский HTML. Правильно — сначала агрегировать (по дням, по сегментам), а не поднимать лимит вслепую через `alt.data_transformers.disable_max_rows()`.
- **Спор «seaborn или matplotlib».** Это ложная дилемма: seaborn *возвращает* matplotlib-объекты (`Axes`), поэтому типичный production-флоу — нарисовать seaborn'ом, а потом доточить осями/аннотациями через matplotlib. Они не альтернативы, а слои.
- **plotly без явного указания, что данных много.** На сотнях тысяч точек обычный `Scatter` подвешивает браузер; для этого есть `Scattergl` (WebGL-рендер). Знать про это — часть выбора «plotly для разведки больших данных».
- **Перепутать «интерактив» и «анимация».** Все четыре могут анимировать, но интерактив (реакция на действия пользователя) — это про plotly/altair. Для отчёта он не нужен.

### Иллюстрация из продуктовой аналитики

Один сценарий — три разных правильных выбора, в зависимости от аудитории:

- **Ты сам, разведка.** Пришёл срез по 30 странам и GGR по дням. Бросаешь `sns.relplot` с фасетами по странам — за одну строку видишь, где провал, не заботясь о красоте. Это seaborn.
- **Продакт хочет «потрогать».** Нужно, чтобы менеджер сам навёл мышь на пик 14-го числа и увидел, что это был всплеск FTD после кампании. Делаешь `plotly.express.line` с hover — он сам читает цифры. Это plotly.
- **Совет директоров, слайд в PDF.** Нужен один чистый график GGR за квартал, с подписанной осью в долларах ($\text{GGR} = \text{ставки} - \text{выплаты}$), без интерактива, который всё равно умрёт в печати. Рисуешь matplotlib (можно начать с seaborn и доточить). Это matplotlib.

Подпись оси в долларах — частая мелочь, ради которой и спускаются в matplotlib:

```python
import matplotlib.ticker as mticker

ax.yaxis.set_major_formatter(
    mticker.FuncFormatter(lambda v, _: f"${v/1000:.0f}k")
)
# 250000 -> "$250k" на оси — то, что seaborn из коробки не даст
```
