# M12.9 · EDA-воркфлоу: как смотреть на новые данные

Разведочный анализ (Exploratory Data Analysis, термин Тьюки) — это не хаотичное «построю-ка графиков», а дисциплина. Рабочая последовательность:

1. Каждую числовую переменную по отдельности — гистограмма или ECDF. Ищешь форму, скос, бимодальность, выбросы, подозрительные пики (например, скопление на круглых суммах депозита).
2. Каждую категориальную — bar частот. Ищешь дисбаланс, редкие категории, опечатки («RU», «ru», «Russia»).
3. Пары переменных — scatter для число-число, boxplot/violin для число-категория, heatmap корреляций для всего числового сразу.
4. Срез во времени — line ключевых метрик, чтобы поймать тренды, сезонность, аномальные дни (выкатка, сбой трекинга, бонусная атака).
5. Small multiples — фасеты по сегменту (страна, канал, платформа), чтобы увидеть, скрывает ли агрегат разнонаправленные группы (парадокс Симпсона).

Цель EDA — не красивые картинки, а гипотезы и аномалии до моделирования. Подробнее про EDA как этап пайплайна — см. модуль про подготовку данных (M10-M11).
