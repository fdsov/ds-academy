# M12.4 · Грамматика графики

### Интуиция

Долгое время думали, что есть «типы графиков»: вот бар-чарт, вот лайн-чарт, вот скаттер — выбери из меню. Это тупик: типов бесконечно, и для нового вопроса нужного в меню нет. Прорыв Лиланда Уилкинсона (книга The Grammar of Graphics, 1999) в том, что любой статистический график — это не атомарный «тип», а конструкция из нескольких независимых слоёв. Меняешь слой — получаешь другой график, не выходя за систему. Это как от «набора готовых фраз» перейти к грамматике языка: конечные правила, бесконечные предложения.

### Формально

График задаётся отображением переменных данных на визуальные свойства. Основные компоненты грамматики:

- Данные — таблица в «длинном» формате: одна строка = одно наблюдение, столбцы = переменные.
- Эстетики (aesthetics, mapping) — какая переменная управляет какой визуальной осью: $x$, $y$, цвет, размер, форма, прозрачность.
- Геометрия (geom) — какими графическими примитивами рисуем: точки, линии, столбцы, площади.
- Статистические преобразования (stat) — что считается перед отрисовкой: подсчёт частот для гистограммы, сглаживание для тренда, агрегация среднего.
- Шкалы (scales) — как значения переменной переводятся в пиксели/цвета: линейная, логарифмическая, временная шкала; конкретная палитра.
- Фасеты (facets) — разбиение на сетку маленьких подграфиков по категориальной переменной (small multiples).
- Координаты — декартовы, полярные (круговая диаграмма — это, по сути, stacked bar в полярных координатах).

Формула одной строкой: график = данные + отображение(эстетики) + геометрия + статистика + шкалы + фасеты. Эта модель лежит в основе ggplot2, а в Python — в библиотеке altair (через Vega-Lite) и в plotnine. Даже когда ты пишешь на matplotlib, мыслить полезно именно так: «какая переменная на $x$, какая на цвет, какая геометрия, какая шкала».

??? example "Один датасет — три графика сменой одного слоя"

    Возьмём таблицу `(дата, канал, депозиты)`.

    - Геометрия line, $x$=дата, $y$=депозиты, цвет=канал → динамика депозитов по каналам.
    - Геометрия bar, $x$=канал, $y$=сумма(депозиты), stat=sum → сравнение каналов.
    - Геометрия area + position=stack → состав суммарных депозитов по каналам во времени.

    Данные те же. Менялись только geom, stat и position. Это и есть грамматика в действии.

!!! question "Проверь себя"

    1. Перечисли основные компоненты грамматики графики (хотя бы пять слоёв).
    2. Чем «грамматика графики» принципиально отличается от подхода «выбери тип из меню»?
    3. Что такое «длинный» формат данных и почему он удобен для грамматики?
    4. Круговая диаграмма в терминах грамматики — это какая геометрия в каких координатах?

??? success "Ответы"

    1. Данные, эстетики (mapping), геометрия (geom), статистические преобразования (stat), шкалы (scales), фасеты (facets), координаты.
    2. Грамматика задаёт конечный набор независимых слоёв, комбинации которых дают бесконечно много графиков; «меню типов» ограничено заранее заготовленным списком и не покрывает новые вопросы.
    3. Длинный формат: одна строка = одно наблюдение, столбцы = переменные. Он позволяет напрямую отображать переменные на эстетики (x, y, цвет, фасет) без ручной перекройки таблицы.
    4. Это, по сути, stacked bar (геометрия столбца с накоплением) в полярных координатах.
