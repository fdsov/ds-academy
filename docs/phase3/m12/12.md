```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

rng = np.random.default_rng(42)

# Депозиты: лог-нормальное распределение, типичное для денег в гемблинге
deposits = rng.lognormal(mean=3.0, sigma=1.1, size=5000)

fig, axes = plt.subplots(1, 3, figsize=(15, 4))

# Гистограмма в линейной шкале — всё прижато к нулю
axes[0].hist(deposits, bins=50, color="#4C72B0")
axes[0].set_title("Депозиты, линейная шкала")

# Та же гистограмма в логарифмической — читаемый колокол
axes[1].hist(deposits, bins=50, color="#4C72B0")
axes[1].set_xscale("log")  # согласуем шкалу с природой денег
axes[1].set_title("Депозиты, лог-шкала")

# ECDF — честный график распределения без выбора бина
sns.ecdfplot(deposits, ax=axes[2], color="#C44E52")
axes[2].set_xscale("log")
axes[2].set_title("ECDF депозитов (лог-x)")

plt.tight_layout()
plt.savefig("deposits_dist.png", dpi=150, bbox_inches="tight")
```

Когортная retention-heatmap на seaborn — рабочая лошадь продуктовой аналитики:

```python
# cohorts: индекс — неделя регистрации, столбцы — возраст когорты (нед.),
# значения — доля вернувшихся (retention)
cohorts = pd.DataFrame(
    {age: np.clip(0.9 ** age + rng.normal(0, 0.03, 8) - i * 0.02, 0, 1)
     for age in range(8)},
    index=pd.date_range("2026-03-02", periods=8, freq="W"),
)

plt.figure(figsize=(9, 5))
sns.heatmap(
    cohorts, annot=True, fmt=".0%",
    cmap="viridis",          # colorblind-safe sequential
    cbar_kws={"label": "Retention"},
)
plt.title("Деградация когорт: свежие строки бледнее")
plt.xlabel("Возраст когорты, недель")
plt.ylabel("Неделя регистрации")
plt.tight_layout()
```

Интерактивный сравнительный график на altair (грамматика графики в явном виде):

```python
import altair as alt

df = pd.DataFrame({
    "day": list(range(31)) * 2,
    "retention": np.concatenate([0.5 * 0.85 ** np.arange(31),
                                  0.5 * 0.93 ** np.arange(31)]),
    "channel": ["A"] * 31 + ["B"] * 31,
})

chart = (
    alt.Chart(df)
    .mark_line(point=True)                      # геометрия
    .encode(
        x=alt.X("day:Q", title="День жизни"),   # эстетика x
        y=alt.Y("retention:Q", axis=alt.Axis(format="%")),
        color=alt.Color("channel:N",            # эстетика цвета (категориальная)
                        scale=alt.Scale(scheme="set2")),
        tooltip=["channel", "day", alt.Tooltip("retention:Q", format=".1%")],
    )
    .properties(title="Retention-кривые по каналам", width=600, height=350)
)
chart.save("retention_curves.html")
```

!!! tip "AI-копилот: где нейросети помогают"

    LLM (Claude, Cursor, Copilot) сильны в синтаксисе визуализации, который никто не держит в голове: точные аргументы matplotlib, переключение шкал, форматирование осей, конвертация графика между seaborn, plotly и altair. Это экономит часы рутины.

    Хороший промпт конкретен про данные и намерение, а не «построй красивый график»:

    > «У меня pandas DataFrame с колонками registration_week, cohort_age_days, retention_rate. Построй когортную heatmap на seaborn: строки — недели регистрации, столбцы — возраст когорты, цвет — retention. Палитра viridis, аннотации в процентах, заголовок-вывод. Объясни выбор палитры.»

    Где LLM врут и что перепроверять обязательно:

    - Устаревший API. Модель может выдать `seaborn.distplot` (deprecated) или старые plotly-вызовы. Сверяй с актуальной докой 2026, особенно для seaborn.objects и altair v6.
    - Молчаливые искажения. LLM спокойно сгенерит bar-чарт с обрезанной осью или dual-axis, потому что «так в примерах». Принципы честности осей и выбора палитры держишь ты — модель их не гарантирует.
    - Палитра наугад. По умолчанию часто радужная/jet-подобная, не colorblind-safe и искажающая порядок. Всегда требуй viridis/cividis для величин и явно проверяй.
    - Выдуманные параметры. Модель уверенно зовёт несуществующий аргумент — запускай код, читай traceback, не доверяй на слово.

!!! note "Стек 2026"

    Актуально сейчас:

    - matplotlib остаётся фундаментом; viridis — дефолтная палитра именно из-за colorblind-safe и сохранения порядка в ч/б.
    - seaborn.objects (интерфейс `so`) — грамматика графики прямо в seaborn, постепенно вытесняет «функцию на тип графика».
    - altair на Vega-Lite 6 и plotly — основной интерактив; Streamlit — стандарт для быстрых дата-приложений на Python.
    - ECDF и violin вытеснили голую гистограмму как инструмент честного сравнения распределений.

    Устарело относительно старых курсов:

    - `seaborn.distplot` deprecated — используй `histplot`, `kdeplot`, `ecdfplot`.
    - Радужная палитра jet — антипаттерн (искажает порядок, не colorblind-safe), хотя десятилетиями была дефолтом.
    - 3D-графики и круговые диаграммы как «серьёзный аналитический инструмент» — практически вышли из профессионального обихода.
    - Excel-стиль «график ради графика» с тенями и градиентами противоречит data-ink принципу.

!!! warning "Ловушки"

    - Обрезанная ось $y$ на bar/area. Превращает разницу в 2% в драматичный обрыв. На столбцах ось значений всегда от нуля.
    - Dual-axis (две оси $y$) — на этом горят сеньоры. Совмещение двух серий на разных осях создаёт ложную «корреляцию»: подвинул масштаб — и линии «совпали». Почти всегда честнее два отдельных графика (small multiples) или нормировка к индексу.
    - График-спагетти. 15 линий одного цвета — нечитаемо. Подсвети 1-2 нужные, остальные приглуши в серый, либо разнеси по фасетам.
    - Радужная/неупорядоченная палитра для упорядоченной величины. Глаз не считывает «какой цвет больше». Только sequential для величин.
    - Overplotting на scatter. Десятки тысяч точек сливаются в чёрное пятно, скрывая плотность. Лечится alpha, hexbin, 2D-density.
    - Цвет как единственный код смысла. 8% аудитории не различит красное и зелёное. Дублируй формой/штриховкой/подписью.
    - Парадокс Симпсона в агрегате. Общий тренд может быть обратным трендам внутри сегментов. Всегда проверяй разбивкой (фасеты), прежде чем делать вывод по агрегату.
    - Pie из 8 кусков. Углы не сравниваются. Замени на отсортированный bar долей.
